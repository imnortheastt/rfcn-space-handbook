## Why this matters

The O-RU is the only part of the RAN that actually touches RF. It converts digital IQ samples to radio waves and vice versa. Everything else is digital processing.

In O-RAN, the RU becomes interchangeable. Understanding O-RU capabilities and constraints is essential for multi-vendor deployments and site planning.

</section>

## How it works

### Open Fronthaul protocol stack

```
Fronthaul Protocol Layers:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  ┌─────────────────────────────────────────────────┐    │
│  │ eCPRI (enhanced CPRI)                           │    │
│  │ - Message types: IQ data, C-plane, etc.        │    │
│  │ - Section headers for PRB allocation           │    │
│  └─────────────────────────────────────────────────┘    │
│                       │                                  │
│  ┌─────────────────────────────────────────────────┐    │
│  │ Ethernet (or UDP/IP over Ethernet)             │    │
│  │ - VLAN tagging for plane separation            │    │
│  │ - PCP for priority                             │    │
│  └─────────────────────────────────────────────────┘    │
│                       │                                  │
│  ┌─────────────────────────────────────────────────┐    │
│  │ Timing (S-Plane)                                │    │
│  │ - PTP (IEEE 1588) for synchronization          │    │
│  │ - SyncE for frequency                          │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  Bandwidth requirements (100 MHz, Cat-B, 64T64R):       │
│  - U-Plane: ~25 Gbps peak                              │
│  - C-Plane: ~100 Mbps                                  │
│  - Total: 25G or 100G Ethernet typical                 │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### Timing requirements

```
O-RU Timing Constraints:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Timing Parameter        │ Requirement                  │
│  ────────────────────────┼──────────────────────────────│
│                          │                              │
│  Frequency accuracy      │ ±50 ppb (normal)            │
│  (SyncE / GPS)           │ ±16 ppb (high accuracy)     │
│                          │                              │
│  Phase accuracy (TDD)    │ ±1.5 µs at air interface    │
│                          │ ±3.0 µs at antenna          │
│                          │                              │
│  Fronthaul delay         │ T1a (min/max) per O-RAN     │
│  (C-Plane → U-Plane)     │ Typically 200-500 µs        │
│                          │                              │
│  Symbol alignment        │ CP length tolerance         │
│                          │                              │
│  Why critical:                                          │
│  - TDD: UL/DL collision if timing wrong                │
│  - FDD: Inter-cell interference                        │
│  - MIMO: Beamforming coherence requires phase sync     │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### IQ compression

```
Fronthaul IQ Compression:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Problem: 64T64R @ 100 MHz = 50+ Gbps uncompressed     │
│                                                          │
│  Solution: Compress IQ samples                          │
│                                                          │
│  ┌─────────────────────────────────────────────────┐    │
│  │ Block Floating Point (BFP):                     │    │
│  │                                                  │    │
│  │ 16-bit I, Q (raw)     → 9-bit I, Q + exponent   │    │
│  │                                                  │    │
│  │ Exponent shared across PRB (12 subcarriers)     │    │
│  │ Compression ratio: ~50%                         │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  ┌─────────────────────────────────────────────────┐    │
│  │ Modulation Compression:                         │    │
│  │                                                  │    │
│  │ Uses modulation type info from C-Plane          │    │
│  │ QPSK: 2 bits/symbol, 256QAM: 8 bits/symbol     │    │
│  │ Higher compression for lower modulation         │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  Trade-off: Compression adds quantization noise         │
│  (affects EVM, throughput at cell edge)                │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## Apply

**O-RU selection criteria:**

| Factor | Small cell | Macro (4T4R) | Massive MIMO |
|--------|------------|--------------|--------------|
| Category | A | A or B | B |
| Power | 1-5W | 20-80W | 200W+ |
| Antenna | 2T2R | 4T4R | 64T64R |
| Fronthaul | 10G | 25G | 25G-100G |
| Weight | < 5 kg | 10-20 kg | 30-50 kg |

**Debugging O-RU:**

```bash
# Check fronthaul link
ethtool eth0 | grep "Speed\|Link"

# PTP sync status
pmc -u -b 0 'GET TIME_STATUS_NP'

# O-RU alarms via NETCONF
netconf-cli get --filter "<alarms/>"

# eCPRI statistics
ecpri-cli show stats

# VSWR check
oru-cli show vswr --port all
```

**Common O-RU parameters:**

| Parameter | Typical Range | Notes |
|-----------|---------------|-------|
| TX power | 2-200W per port | Class defines max |
| Frequency bands | 600 MHz - 40 GHz | Per O-RU model |
| Bandwidth | 5-100 MHz | Per carrier |
| SCS | 15, 30, 60, 120 kHz | Band dependent |
| EVM | < 8% (256QAM) | RF quality metric |

<section class="lesson-pitfalls">

## Pitfalls

1. **Timing sync loss.** If PTP fails, TDD O-RU must stop transmitting. Ensure redundant timing sources.

2. **VSWR alarm.** High VSWR indicates antenna/feeder issue. Can damage PA if ignored.

3. **Thermal throttling.** O-RU reduces power in high temperature. Site cooling is critical.

4. **Fronthaul jitter.** Late packets are dropped. Use traffic shaping and priority queuing.

5. **IQ compression artifacts.** Aggressive compression degrades EVM. Monitor cell edge performance.

6. **Cat-A/B mismatch.** O-DU must know O-RU category. Wrong assumption breaks beamforming.

7. **Carrier aggregation timing.** Multiple carriers require tight phase alignment. Verify per-carrier sync.

8. **Firmware incompatibility.** O-RAN spec evolves. O-DU and O-RU firmware versions must be compatible.

</section>

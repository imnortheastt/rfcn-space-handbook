## Why this matters

In traditional RAN, the base station did everything — RF, signal processing, scheduling, RRC. O-RAN splits this into components that can come from different vendors.

The O-CU handles the "upper layers" — RRC for control plane, PDCP for user plane. Understanding O-CU is essential for multi-vendor RAN deployments.

## Apply

**O-CU deployment options:**

| Option | CU-CP | CU-UP | Use Case |
|--------|-------|-------|----------|
| Integrated | Same node | Same node | Simple deployment |
| Separated | Centralized | Edge | Low latency UP |
| Pooled | Shared | Per-edge | Control plane efficiency |
| Redundant | Active-standby | Load-balanced | High availability |

**Debugging O-CU:**

```bash
# Wireshark: F1AP messages
f1ap

# E1AP messages
e1ap

# Check F1 setup
grep "F1SetupRequest\|F1SetupResponse" /var/log/cu-cp/*.log

# Bearer context operations
grep "BearerContext" /var/log/cu-cp/*.log

# PDCP statistics
pdcp-cli show stats --drb
```

**Key metrics:**

| Metric | Typical | Alert |
|--------|---------|-------|
| F1 latency (CP) | < 10 ms | > 50 ms |
| F1 latency (UP) | < 5 ms | > 20 ms |
| E1 latency | < 5 ms | > 20 ms |
| RRC setup time | < 200 ms | > 500 ms |
| PDCP throughput | Line rate | Drops |

<section class="lesson-pitfalls">

## Pitfalls

1. **F1 transport reliability.** F1 uses SCTP (CP) and UDP (UP). Network issues between CU and DU directly impact service.

2. **E1 split complexity.** Separating CU-CP and CU-UP adds latency. Keep E1 link short.

3. **UE context synchronization.** CU-CP, CU-UP, and DU must have consistent UE context. Sync failures cause drops.

4. **PDCP reordering buffer.** CU-UP buffers out-of-order packets. Buffer overflow causes loss.

5. **Handover forwarding.** During handover, source CU-UP forwards to target. Forwarding path must be fast.

6. **ROHC context loss.** Header compression context must be maintained. Context mismatch causes decompression failure.

7. **Multi-DU coordination.** One CU-CP can serve multiple DUs. F1 setup and management overhead increases.

8. **Security key handling.** CU-CP derives and distributes keys. Key mismatch breaks ciphering.

## Why this matters

Prepaid customers pay before service — run out of credit, service stops. Postpaid customers use first, pay later — service continues, bill arrives monthly.

The network handles these completely differently. Understanding online vs offline charging explains why prepaid users get throttled instantly while postpaid users don't.

</section>

## Intuition

### Online vs offline comparison

```
Charging Modes Side-by-Side:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Online Charging (Prepaid):                             │
│  ┌─────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │  Balance: $10.00                                │    │
│  │           ↓                                     │    │
│  │  [Request quota] → CHF checks balance          │    │
│  │           ↓                                     │    │
│  │  [Grant 100 MB] ← quota issued                 │    │
│  │           ↓                                     │    │
│  │  [Use 80 MB] → threshold reached               │    │
│  │           ↓                                     │    │
│  │  [Request more] → deduct $0.80, grant more     │    │
│  │           ↓                                     │    │
│  │  [Balance: $0] → session blocked/throttled     │    │
│  │                                                  │    │
│  │  Real-time • Quota-based • Balance deduction   │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  Offline Charging (Postpaid):                           │
│  ┌─────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │  Credit Limit: $500                             │    │
│  │           ↓                                     │    │
│  │  [Notify session start] → CHF records          │    │
│  │           ↓                                     │    │
│  │  [Use 500 MB] → no real-time check             │    │
│  │           ↓                                     │    │
│  │  [Report periodically] → CHF accumulates       │    │
│  │           ↓                                     │    │
│  │  [Session ends] → CHF creates CDR              │    │
│  │           ↓                                     │    │
│  │  [CDR → Billing] → Invoice generated           │    │
│  │                                                  │    │
│  │  Delayed • CDR-based • Billed later            │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### When to use each

```
Charging Mode Selection:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Use Online When:                                       │
│  ├── Prepaid subscription                              │
│  ├── Real-time credit control needed                   │
│  ├── Sponsored data (third-party pays)                 │
│  ├── Usage-based premium services                      │
│  └── Roaming with credit limit                         │
│                                                          │
│  Use Offline When:                                      │
│  ├── Postpaid subscription                             │
│  ├── Unlimited plans                                   │
│  ├── Enterprise accounts (bulk billing)               │
│  ├── Flat-rate services                                │
│  └── No balance check needed                           │
│                                                          │
│  Use Both (Hybrid):                                     │
│  ├── Postpaid with spending limit                     │
│  ├── Prepaid with CDR for detailed billing            │
│  └── Mixed services on same subscription              │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

│                       │────────────────────►│
│                       │                      │

│                       │◄────────────────────│
│                       │                      │
│ Grant: 100 MB        │                      │
│◄──────────────────────│                      │
│                       │                      │

│                       │                      │

```
When Quota Runs Out:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Terminate:                                             │
│  ┌─────────────────────────────────────────────────┐    │
│  │ - Immediately release session                   │    │
│  │ - User loses connectivity                       │    │
│  │ - Harsh but clear                               │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  Redirect:                                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │ - Keep session, redirect HTTP to top-up page   │    │
│  │ - User can add credit                          │    │
│  │ - UPF applies redirect rules                   │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  Restrict Access:                                       │
│  ┌─────────────────────────────────────────────────┐    │
│  │ - Keep session with limited AMBR (e.g., 1 Mbps)│    │
│  │ - User can still browse slowly                 │    │
│  │ - Less disruptive                              │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  Notification Only:                                     │
│  ┌─────────────────────────────────────────────────┐    │
│  │ - Continue service, notify user                │    │
│  │ - Used with spending limits                    │    │
│  │ - PCF notified, may trigger policy change      │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
└─────────────────────────────────────────────────────────┘

```
Combined Online + Offline:

┌─────────────────────────────────────────────────────────┐

│  Scenario: Postpaid with spending limit                 │
│                                                          │
│  ┌─────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │  Monthly limit: $100                            │    │
│  │                                                  │    │
│  │  Online component:                              │    │

│  │  - Block if limit exceeded                     │    │
│  │                                                  │    │
│  │  Offline component:                             │    │
│  │  - Generate CDR for detailed billing           │    │

## Apply

**Charging mode by scenario:**

| Scenario | Mode | Why |
|----------|------|-----|
| Prepaid SIM | Online | Real-time credit control |
| Unlimited postpaid | Offline | No balance check needed |
| Postpaid with cap | Hybrid | Limit + detailed billing |
| Sponsored data | Online | Third-party pays per byte |
| IoT device | Offline | Bulk billing, low overhead |
| Roaming | Online | Credit control abroad |

**Debugging charging issues:**

```bash
# Wireshark: online vs offline
http2.header.path contains "chargingData" && json.value.string contains "ONLINE"
http2.header.path contains "chargingData" && json.value.string contains "OFFLINE"

# Check quota grants
grep "grantedUnit" /var/log/smf/*.log

# Final unit action
grep "finalUnitAction" /var/log/chf/*.log

# CDR output
tail -f /var/spool/chf/cdr/*.cdr
```

**Common parameters:**

| Parameter | Online | Offline |
|-----------|--------|---------|
| Quota | Granted per request | Not applicable |
| Reporting | On threshold/time | Periodic/final |
| CDR | Optional | Always generated |
| Latency impact | Higher | Lower |
| Failure handling | Service blocked | Service continues |

<section class="lesson-pitfalls">

## Pitfalls

1. **CHF latency.** Online charging adds latency to every quota request. Optimize CHF response time.

2. **Over-usage between updates.** User may exceed quota between SMF updates. Allow tolerance margin.

3. **Quota exhaustion during call.** Voice call cut mid-sentence is poor UX. Grant extra quota for active calls.

4. **CDR loss.** Offline CDRs must be reliably stored. Lost CDR = lost revenue. Use persistent storage.

5. **Wrong charging mode.** Applying online to postpaid (unnecessary) or offline to prepaid (no control) causes issues.

6. **Redirect loop.** If redirect page itself counts against quota, user can't top up. Whitelist top-up URLs.

7. **Roaming complexity.** V-CHF vs H-CHF depends on roaming agreement. Ensure correct routing.

8. **Session-level vs flow-level.** Same session may have flows with different charging modes. Track separately.

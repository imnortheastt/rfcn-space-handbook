## Why this matters

PCF is the brain that decides what each user can do on the network. Can this user start a video call? What QoS should they get? Have they exceeded their data limit?

Without PCF, every subscriber would get the same treatment — no differentiation, no enforcement, no business model.

</section>

## Intuition

### What PCF controls

```
PCF Policy Domains:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Session Management (SM) Policy:                        │
│  ├── QoS decisions (5QI, AMBR, GBR)                    │
│  ├── Session rules (what flows are allowed)            │
│  ├── Charging method selection                          │
│  └── Traffic steering (UL classifier, SSC mode)        │
│                                                          │
│  Access and Mobility (AM) Policy:                       │
│  ├── Service area restrictions                         │
│  ├── RAT/frequency selection priority                  │
│  ├── UE route selection policy (URSP)                  │
│  └── Network slice selection assistance                │
│                                                          │
│  UE Policy:                                             │
│  ├── Route selection rules sent to UE                  │
│  └── Operator-specific UE configuration                │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

              ┌─────────┐
              │   AF    │ (Application Function)

          ┌────────────────┐

          └───────┬────────┘
        ┌─────────┼─────────┐
        │         │         │

   Npcf_AMPolicy  Npcf_SMPolicy  Charging

### From PCRF to PCF

```
4G vs 5G Policy:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  4G (EPC):                                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │  PCRF (Policy & Charging Rules Function)        │    │
│  │  - Gx interface to PGW (Diameter)               │    │
│  │  - Rx interface to AF (Diameter)                │    │
│  │  - Sp interface to SPR (subscriber data)        │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  5G (5GC):                                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │  PCF (Policy Control Function)                   │    │
│  │  - N7 to SMF (HTTP/2)                           │    │
│  │  - N5 to AF (HTTP/2)                            │    │
│  │  - N15 to AMF (HTTP/2)                          │    │
│  │  - Nudr to UDR (subscriber data)                │    │
│  │  - SBA, service-based                           │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  Key changes:                                           │
│  - Diameter → HTTP/2                                    │
│  - Separate AM and SM policy                            │
│  - UDR replaces SPR                                     │
│  - Network slice awareness                              │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

┌─────────────────────────────────────────────────────────┐

## Apply

**Common policy scenarios:**

| Scenario | PCF Action | SMF Effect |
|----------|------------|------------|
| Premium subscriber | Higher AMBR, priority 5QI | Better QoS flows |
| Data cap reached | AMBR = 1 Mbps | Throttling at UPF |
| VoLTE call start | Add GBR rule | Dedicated voice bearer |
| Roaming | Apply visited PLMN policy | Restricted services |
| Enterprise slice | Slice-specific rules | Dedicated QoS |

**Debugging PCF:**

```bash
# Wireshark: PCF service operations
http2 && sbi && pcf

# N7 policy create
http2.header.path contains "sm-policies"

# N5 AF authorization
http2.header.path contains "app-sessions"

# Check policy decision
grep "PccRule" /var/log/pcf/*.log
```

**Key API operations:**

| Interface | Operation | Purpose |
|-----------|-----------|---------|
| N7 (SMF) | SMPoliciesPost | Create SM policy |
| N7 (SMF) | SMPoliciesUpdate | Modify policy |
| N15 (AMF) | AMPoliciesPost | Create AM policy |
| N5 (AF) | PolicyAuthorizationCreate | AF requests QoS |

</section>

<section class="lesson-pitfalls">

## Pitfalls

1. **Missing policy association.** SMF must create SM policy before session completes. No association = default/reject.

2. **UDR unavailable.** PCF can't fetch subscriber policy if UDR is down. Need fallback rules.

3. **Policy timing.** PCF update may arrive while session is being modified. Race conditions possible.

4. **AF authorization mismatch.** AF requests QoS that subscription doesn't allow. PCF must reject or downgrade.

5. **Stale policy.** Policy decision based on old data (subscription changed, location changed). Subscribe to events.

6. **Slice policy conflict.** Different slices may have different policy rules. Ensure correct slice context.

7. **Charging integration.** PCF relies on CHF for usage events. Missing integration = no usage-based policy.

8. **Local breakout.** Roaming scenarios need V-PCF/H-PCF interaction. Policy authority depends on roaming agreement.

</section>

## Why this matters

xApps are the intelligence engines of Near-RT RIC. They consume RAN data, apply algorithms (including ML), and issue control actions — all within the 10ms to 1s control loop.

Understanding xApp development is essential for building O-RAN automation and optimization solutions.

**MLB xApp example (Go):**

```go
package main

import (
    "gerrit.o-ran-sc.org/r/ric-plt/xapp-frame/pkg/xapp"
)

type MLBXapp struct {
    threshold int
}

func (m *MLBXapp) Consume(msg *xapp.RMRParams) error {
    switch msg.Mtype {
    case xapp.RIC_INDICATION:
        return m.handleIndication(msg)
    case xapp.A1_POLICY_REQ:
        return m.handlePolicy(msg)
    }
    return nil
}

func (m *MLBXapp) handleIndication(msg *xapp.RMRParams) error {
    // Decode KPM indication
    kpm := DecodeKpmIndication(msg.Payload)
    
    // Check cell load
    for _, cell := range kpm.Cells {
        if cell.PrbUtilDL > m.threshold {
            // Find target cell for load balancing
            target := m.findLeastLoadedNeighbor(cell)
            if target != nil {
                m.triggerHandover(cell, target)
            }
        }
    }
    return nil
}

func main() {
    mlb := &MLBXapp{threshold: 70}
    xapp.SetMessageCallback(mlb)
    xapp.Run()
}
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Blocking RMR callback.** Long processing in callback blocks message queue. Use async/goroutines.

2. **Missing subscription cleanup.** xApp crashes without unsubscribing. Subscription manager handles orphans but check state.

3. **SDL access patterns.** Frequent small reads slow down. Batch reads, cache locally.

4. **Control rate too high.** Sending too many control messages causes E2 congestion. Implement rate limiting.

5. **E2SM encoding errors.** Mismatched ASN.1 encoding between xApp and E2 node. Verify E2SM version compatibility.

6. **Config not reloaded.** xApp doesn't pick up config changes. Implement config watch or restart.

7. **Memory leaks.** Large indication payloads not released. Profile memory usage, especially for ML xApps.

8. **Conflict with other xApps.** Multiple xApps controlling same parameters. Use conflict mitigation service.

## Why this matters

Near-RT RIC handles fast decisions. Non-RT RIC handles the strategic layer — training ML models, generating policies, and managing the RAN lifecycle.

Non-RT RIC is part of the SMO and provides the intelligence that guides Near-RT RIC behavior. Understanding both is essential for complete O-RAN deployment.

## Apply

**A1 policy types:**

| Policy Type | Purpose | Scope |
|-------------|---------|-------|
| QoS | Throughput/latency targets | Cell, slice, UE |
| Traffic Steering | Load balancing rules | Cell, TA |
| Resource | PRB allocation guidance | Cell, slice |
| Energy Saving | Power management | Cell, site |

**OSC Non-RT RIC components:**

```bash
# Deploy OSC Non-RT RIC
helm install nonrtric oran-sc/nonrtric

# Check rApp manager
kubectl get pods -n nonrtric | grep rappmanager

# Register A1 policy type
curl -X PUT http://a1-pms:8081/policytype \
  -H "Content-Type: application/json" \
  -d '{
    "policyTypeId": "ORAN_QoS_1.0",
    "policySchema": {...}
  }'

# Create A1 policy
curl -X PUT http://a1-pms:8081/policy \
  -H "Content-Type: application/json" \
  -d '{
    "policyId": "qos-001",
    "policyTypeId": "ORAN_QoS_1.0",
    "ricId": "ric-1",
    "policyData": {
      "scope": {"cellId": "cell-001"},
      "qosObjectives": {"minThroughput": 100000000}
    }
  }'

# Check policy status
curl http://a1-pms:8081/policy/qos-001/status
```

**rApp development:**

```python
# rApp example: Traffic prediction
from nonrtric_sdk import RAppClient

client = RAppClient()

# Get historical PM data
pm_data = client.get_pm_history(
    cell_id="cell-001",
    metrics=["prb_util", "throughput"],
    hours=168  # 1 week
)

# Train model
model = train_traffic_prediction(pm_data)

# Deploy to Near-RT RIC
client.deploy_model(
    model_id="traffic-pred-v1",
    model=model,
    target_xapp="traffic-steering-xapp"
)

# Create A1 policy based on prediction
if predicted_congestion():
    client.create_policy(
        policy_type="ORAN_TrafficSteering_1.0",
        policy_data={"action": "balance_load"}
    )
```

<section class="lesson-pitfalls">

## Pitfalls

1. **A1 policy not enforced.** Near-RT RIC receives policy but xApp ignores it. Check xApp A1 handling.

2. **Stale ML model.** Model trained on old data doesn't fit current conditions. Schedule regular retraining.

3. **Policy conflict.** Multiple rApps create conflicting policies. Use policy conflict detection.

4. **Data pipeline lag.** PM data arrives too late for timely decisions. Optimize data collection pipeline.

5. **Model too large.** Near-RT RIC can't load model within memory constraints. Compress or simplify model.

6. **Missing feedback.** No metrics on model performance after deployment. Implement monitoring.

7. **R1 API version mismatch.** rApp uses outdated R1 API. Check compatibility with Non-RT RIC version.

8. **Enrichment data stale.** External data (predictions) not updated frequently. Check enrichment provider.

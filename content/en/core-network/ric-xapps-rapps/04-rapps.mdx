## Why this matters

rApps are the intelligence engines of Non-RT RIC. They operate on timescales >1 second, performing analytics, training ML models, and generating policies that guide xApps.

Understanding rApp development enables building the "brain" that optimizes RAN behavior at the strategic level.

## Apply

**rApp development stack:**

| Component | Options |
|-----------|---------|
| Language | Python (ML), Go, Java |
| ML Framework | TensorFlow, PyTorch, scikit-learn |
| Model Format | ONNX (portable), SavedModel |
| Container | Docker, Kubernetes |
| Data Access | Kafka, REST APIs |

**Traffic prediction rApp example (Python):**

```python
from rapp_sdk import RAppFramework, R1Client
import pandas as pd
from sklearn.ensemble import GradientBoostingRegressor
import onnx

class TrafficPredictionRApp:
    def __init__(self, config):
        self.r1 = R1Client()
        self.model = None
        self.config = config
    
    def start(self):
        # Subscribe to PM data
        self.r1.subscribe_data(
            data_type="pm",
            filter={"counters": ["PRBUsageDL", "PRBUsageUL"]},
            callback=self.on_data
        )
        
        # Register as EI producer
        self.r1.register_ei_producer(
            info_type="traffic-prediction",
            schema=self.prediction_schema
        )
    
    def on_data(self, data):
        # Accumulate data for training
        self.data_buffer.append(data)
        
        if len(self.data_buffer) >= self.config['min_data_points']:
            self.train_model()
    
    def train_model(self):
        df = pd.DataFrame(self.data_buffer)
        X = self.extract_features(df)
        y = df['target_load']
        
        self.model = GradientBoostingRegressor()
        self.model.fit(X, y)
        
        # Export to ONNX and deploy
        onnx_model = self.to_onnx(self.model)
        self.r1.upload_model(
            model_id="traffic-pred-v2",
            artifact=onnx_model
        )
    
    def generate_predictions(self):
        predictions = self.model.predict(self.current_features)
        
        # Publish as enrichment info
        self.r1.publish_ei(
            info_type="traffic-prediction",
            data={"predictions": predictions.tolist()}
        )

if __name__ == "__main__":
    app = RAppFramework()
    rapp = TrafficPredictionRApp(app.config)
    app.run(rapp)
```

## Pitfalls

1. **Training on stale data.** PM data from weeks ago may not reflect current network. Implement data freshness checks.

2. **Policy conflict.** Multiple rApps generating conflicting policies. Use policy coordination service.

3. **Model drift undetected.** Predictions degrade over time. Implement monitoring and retraining triggers.

4. **Overloading A1.** Too many policy updates saturate A1 interface. Batch policy changes.

5. **EI latency too high.** Enrichment info arrives after xApp decision. Pre-compute and cache predictions.

6. **R1 service unavailable.** Non-RT RIC services restart. Implement retry with backoff.

7. **Large model deployment.** ONNX model too large for Near-RT RIC. Quantize or prune model.

8. **Schema mismatch.** Policy schema changes between versions. Version policy types carefully.

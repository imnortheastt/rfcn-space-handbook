<section class="lesson-hook">

## Why this matters

PDU sessions don't last forever. When you close an app, run out of data, or get deregistered, the session must be **released**. This frees IP addresses, UPF resources, and RAN bearers.

Proper release is essential — leaked sessions consume network resources indefinitely.

</section>

<section class="lesson-intuition">

## Intuition

### When release happens

```
PDU Session Release Triggers:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  UE-Initiated:                                          │
│  ├── User turns off mobile data                         │
│  ├── App no longer needs connectivity                   │
│  └── User switches to WiFi                              │
│                                                          │
│  Network-Initiated:                                     │
│  ├── Subscription terminated                            │
│  ├── Policy rejection (data cap, time limit)           │
│  ├── Inactivity timeout                                │
│  ├── UPF failure                                        │
│  └── Admin action (OAM)                                │
│                                                          │
│  Implicit (during deregistration):                      │
│  ├── UE powers off (switch-off deregistration)         │
│  └── Implicit deregistration (UE unreachable)          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### What gets cleaned up

```
Resources Released:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  UE Side:                                               │
│  ├── IP address released                                │
│  ├── QoS rules removed                                  │
│  └── DNS config cleared                                 │
│                                                          │
│  RAN Side:                                              │
│  ├── DRB (Data Radio Bearer) released                  │
│  └── QoS flow contexts deleted                          │
│                                                          │
│  SMF Side:                                              │
│  ├── PDU session context deleted                        │
│  └── PCF policy association released                    │
│                                                          │
│  UPF Side:                                              │
│  ├── PFCP session deleted                               │
│  ├── PDR/FAR/QER/URR rules removed                     │
│  ├── IP address returned to pool                       │
│  └── GTP-U tunnels torn down                           │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

<section class="lesson-apply">

## Apply

**Common release scenarios:**

| Scenario | Initiator | Cause |
|----------|-----------|-------|
| User toggles airplane mode | UE | switch-off deregistration |
| Data cap exceeded | Network (PCF) | policy violation |
| Prepaid balance exhausted | Network (CHF) | charging event |
| Inactivity timeout | Network (SMF) | timer expiry |
| Network maintenance | Network (OAM) | admin action |

**Debugging release:**

```bash
# Wireshark filter for release messages
nas-5gs.sm.message_type == 0xd1 || nas-5gs.sm.message_type == 0xd3 || nas-5gs.sm.message_type == 0xd4

# N4 session deletion
pfcp.message_type == 5 || pfcp.message_type == 6

# Check for orphaned sessions
grep "session.state" /var/log/smf/*.log | grep -v "released"
```

**Troubleshooting:**

| Issue | Check |
|-------|-------|
| Session not releasing | SMF stuck? N4 timeout? |
| IP not returned to pool | UPF deletion response received? |
| DRB not released | N2 message to gNB sent? |
| Usage not reported | URR configured? Final report received? |

</section>

<section class="lesson-pitfalls">

## Pitfalls

1. **Forgetting N4 deletion.** SMF must delete PFCP session. Missing this leaks UPF resources.

2. **No final usage report.** If URR was configured, UPF should send final usage. Missing this causes billing errors.

3. **RAN resources not released.** AMF must send N2 release to gNB. Missing this keeps DRB allocated.

4. **IP pool exhaustion.** If sessions don't release properly, IP pool depletes. Monitor pool utilization.

5. **PCF association not released.** SMF should notify PCF. Stale policy associations waste PCF resources.

6. **Ignoring release cause.** Different causes need different handling. "Reactivation requested" means UE should retry.

7. **Release during handover.** Don't release session just because gNB changes. Only path switches, session stays.

8. **Multi-homed sessions.** UE with both 3GPP and non-3GPP access — releasing one doesn't release the other.

</section>

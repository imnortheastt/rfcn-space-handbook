## Why this matters

When AMF needs an SMF, how does it know which SMF instances exist? What IP addresses? What slices they serve?

NRF answers this. Every NF registers with NRF at startup, provides heartbeats while running, and deregisters at shutdown. Without NRF, 5G's dynamic service architecture falls apart.

</section>

## Intuition

### NRF as service registry

```
NRF Role in 5G Core:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│                    ┌─────────────┐                      │
│                    │     NRF     │                      │
│                    │  ┌───────┐  │                      │
│                    │  │Registry│ │                      │
│                    │  └───────┘  │                      │
│                    └──────┬──────┘                      │
│                           │                             │
│    ┌──────────────────────┼──────────────────────┐     │
│    │                      │                      │     │
│    ▼                      ▼                      ▼     │
│  ┌───┐    Register     ┌───┐    Register     ┌───┐    │
│  │AMF│ ◄─────────────► │SMF│ ◄─────────────► │UDM│    │
│  └───┘    Heartbeat    └───┘    Heartbeat    └───┘    │
│    │      Discover        │      Discover        │     │
│    │                      │                      │     │
│    └──────────────────────┴──────────────────────┘     │
│                                                          │
│  NRF knows:                                             │
│  - Every NF instance (IP, port, FQDN)                  │
│  - Services each NF provides                           │
│  - Slices, DNNs, TAIs each NF serves                   │
│  - Current health status                               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### NF lifecycle

```
NF Lifecycle with NRF:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  1. Registration (startup)                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │ NF starts → PUT /nnrf-nfm/v1/nf-instances/{id} │    │
│  │           ← 201 Created                         │    │
│  │ NF now discoverable by other NFs               │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  2. Heartbeat (running)                                 │
│  ┌─────────────────────────────────────────────────┐    │
│  │ Every 30-60 seconds:                            │    │
│  │ PATCH /nnrf-nfm/v1/nf-instances/{id}           │    │
│  │ (update heartbeatTimer)                        │    │
│  │                                                  │    │
│  │ If NRF misses heartbeats → NF marked unhealthy │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  3. Deregistration (shutdown)                           │
│  ┌─────────────────────────────────────────────────┐    │
│  │ NF shutting down:                               │    │
│  │ DELETE /nnrf-nfm/v1/nf-instances/{id}          │    │
│  │ ← 204 No Content                                │    │
│  │ NF no longer discoverable                       │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

```
NF Registration Detail:

│                                 │

│ Body: NF Profile               │
│─────────────────────────────────►│
│                                 │
│                                 │ Validate profile
│                                 │ Store in registry
│                                 │
│ 201 Created                    │

│◄─────────────────────────────────│
│                                 │

│                                 │
│ Periodic heartbeat:            │
```

## Apply

**NF profile key fields by NF type:**

| NF Type | Key Profile Fields |
|---------|-------------------|
| SMF | smfInfo (sNssaiSmfInfoList, taiList, dnnList) |
| AMF | amfInfo (amfSetId, amfRegionId, guamiList, taiList) |
| UDM | udmInfo (supiRanges, gpsiRanges) |
| AUSF | ausfInfo (supiRanges, groupId) |
| PCF | pcfInfo (dnnList, supiRanges) |
| UPF | upfInfo (sNssaiUpfInfoList, interfaceUpfInfoList) |

**Debugging registration:**

```bash
# Check NF registration
curl -k "https://nrf.local:29510/nnrf-nfm/v1/nf-instances/{id}"

# List all registered NFs
curl -k "https://nrf.local:29510/nnrf-nfm/v1/nf-instances"

# Wireshark: registration/discovery
http2.header.path contains "nnrf-nfm" || http2.header.path contains "nnrf-disc"

# Check heartbeat failures
grep "heartbeat" /var/log/nrf/*.log | grep -i "fail\|timeout"
```

**Heartbeat timing:**

| Scenario | Heartbeat Timer | Recovery |
|----------|-----------------|----------|
| Normal | 30-60 seconds | — |
| High availability | 10-20 seconds | Faster failover |
| NRF miss tolerance | 2-3 missed | Mark NF unhealthy |
| Deregistration | Graceful DELETE | Immediate removal |

</section>

<section class="lesson-pitfalls">

## Pitfalls

1. **Heartbeat timeout.** If NF crashes without deregistering, NRF waits for heartbeat timeout. Configure shorter timers for HA.

2. **NRF single point of failure.** Deploy NRF in HA cluster. NFs should cache discovered NF profiles.

3. **Profile mismatch.** If NF profile doesn't match query criteria exactly, it won't be discovered. Check sNssai, DNN, TAI.

4. **Discovery cache staleness.** Consumer caches NF profiles. If producer changes, cache is stale. Subscribe to notifications.

5. **Load balancing.** NRF returns multiple instances. Consumer must implement selection logic (round-robin, least-load).

6. **Registration retry.** If initial registration fails, NF must retry with backoff. Don't flood NRF.

7. **Profile update.** When NF changes (new service, new IP), must re-register. Use PUT to replace or PATCH to update.

8. **UUID generation.** nfInstanceId must be unique. Use proper UUID generation, not sequential IDs.

## Why this matters

Before a phone can make calls, send data, or do anything useful on a 5G network, it must **register**. Registration tells the network: "I exist, authenticate me, and track where I am."

This procedure happens every time you power on your phone, move to a new area, or periodically while idle. Understanding registration is fundamental to debugging connectivity issues.

### Registration types

| Type | Trigger | Purpose |
|------|---------|---------|
| **Initial** | Power on, SIM insert | First-time or fresh registration |
| **Mobility** | Enter new RA/TA | Update location with network |
| **Periodic** | Timer expires | Confirm UE is still alive |
| **Emergency** | Emergency call | Limited registration without SIM |

## Apply

**Common registration parameters:**

| Parameter | Typical Value | Purpose |
|-----------|---------------|---------|
| T3512 | 54 min | Periodic registration timer |
| T3502 | 12 sec | Retry after registration reject |
| TAI list size | 16-64 TAIs | Reduce mobility updates |

**Troubleshooting registration:**

| Symptom | Possible Cause | Check |
|---------|---------------|-------|
| Reg reject (cause 3) | Illegal UE | SIM blocked/invalid |
| Reg reject (cause 5) | IMEI not accepted | Device blacklisted |
| Reg reject (cause 11) | PLMN not allowed | Roaming config |
| Reg timeout | AUSF/UDM unreachable | Core connectivity |
| Repeated reg | TAI list too small | Expand TAI list |

**Debugging commands:**

```bash
# Wireshark filter for NAS registration
nas-5gs.mm.message_type == 0x41 || nas-5gs.mm.message_type == 0x42

# tcpdump for N2 (NGAP)
tcpdump -i any port 38412 -w registration.pcap
```

<section class="lesson-pitfalls">

## Pitfalls

1. **SUCI vs 5G-GUTI confusion.** Initial registration uses SUCI (encrypted SUPI). Mobility/periodic use 5G-GUTI. Sending wrong ID type causes failure.

2. **TAI list too small.** If TAI list doesn't cover UE's movement pattern, excessive mobility updates occur.

3. **T3512 too short.** Aggressive periodic timer wastes UE battery and network resources.

4. **Missing NSSAI.** If UE doesn't request slices, default slice is used. May not match UE's service needs.

5. **Old AMF unreachable.** During mobility, new AMF contacts old AMF for context. If old AMF is down, context transfer fails.

6. **Authentication timeout.** Slow AUSF/UDM response causes registration timeout. Monitor SBI latency.

7. **SIM/credential mismatch.** K/OPc in UDR must match USIM. Any mismatch = auth failure = reg reject.

8. **Ignoring reject causes.** Different reject causes need different handling. Don't treat all rejects the same.

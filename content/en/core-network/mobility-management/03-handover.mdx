## Why this matters

You're in a video call on the train. The train moves at 300 km/h, passing through cell after cell. Your call doesn't drop. This is **handover** — the network seamlessly transfers your connection from one cell to another.

Handover is what makes mobility work. Without it, every cell boundary would drop your call.

</section>

## Intuition

### What handover accomplishes

```
Handover Goal:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  UE moving from Cell A to Cell B                        │
│                                                          │
│     Cell A              Cell B                          │
│   ┌────────┐          ┌────────┐                        │
│   │  gNB   │          │  gNB   │                        │
│   │  (src) │          │  (tgt) │                        │
│   └────┬───┘          └────┬───┘                        │
│        │                    │                            │
│        │    ┌──────┐       │                            │
│        │    │  UE  │───────┼───►                        │
│        │    │ ───► │       │                            │
│        │    └──────┘       │                            │
│        │                    │                            │
│   Weak signal          Strong signal                    │
│                                                          │
│  Requirements:                                          │
│  1. No call drop                                        │
│  2. Minimal interruption (< 50ms for voice)            │
│  3. No packet loss (or minimal)                        │
│  4. Preserve security context                          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### Handover types

| Type | Interface | When Used |
|------|-----------|-----------|
| **Xn handover** | Xn (gNB-gNB) | Same AMF, gNBs connected via Xn |
| **N2 handover** | N2 (via AMF) | No Xn, or AMF change required |
| **Inter-RAT** | N26/N2 | 5G ↔ 4G transitions |

### Xn vs N2 handover

```
Handover Path Selection:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Xn Handover (preferred when possible):                 │
│  ┌─────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │  gNB-src ◄────── Xn ──────► gNB-tgt            │    │
│  │     │                          │                │    │
│  │     └──────────────────────────┘                │    │
│  │          Direct, fast                           │    │
│  │                                                  │    │
│  │  AMF not in path until completion              │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  N2 Handover (when Xn not available):                   │
│  ┌─────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │  gNB-src ◄───N2───► AMF ◄───N2───► gNB-tgt     │    │
│  │                      │                          │    │
│  │                   Mediates                      │    │
│  │                                                  │    │
│  │  AMF coordinates entire handover               │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## Apply

**Handover parameters:**

| Parameter | Purpose | Typical Value |
|-----------|---------|---------------|
| A3 offset | Neighbor must be this much better | 2-6 dB |
| Time-to-trigger | How long A3 must hold | 40-640 ms |
| Hysteresis | Prevent ping-pong | 1-3 dB |
| T304 | HO completion timer | 50-200 ms |

**Troubleshooting handover:**

| Issue | Possible Cause | Check |
|-------|---------------|-------|
| Too late HO | A3 offset too high | Reduce offset |
| Too early HO | A3 offset too low | Increase offset |
| Ping-pong HO | Hysteresis too low | Increase hysteresis |
| HO failure | Target cell congested | Admission control |
| Data loss | Forwarding not working | Check Xn/tunnels |

**Wireshark filters:**

```bash
# NGAP handover messages
ngap.procedureCode == 1 || ngap.procedureCode == 0

# Xn handover
xnap.procedureCode == 0 || xnap.procedureCode == 1
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Ignoring measurement configuration.** UE only reports what you ask for. Misconfigured measurements = no handover.

2. **A3 offset too aggressive.** Too low offset causes ping-pong between cells. Too high causes late handover and drops.

3. **Xn not configured.** Without Xn interface, every handover goes through AMF (slower, more signaling).

4. **Data forwarding disabled.** During handover, buffered data at source gNB should forward to target. If disabled, packet loss.

5. **Timer T304 too short.** If UE can't complete handover before T304 expires, it declares HO failure and goes back to source.

6. **Not handling HO failure.** When handover fails, UE should return to source cell. Make sure re-establishment works.

7. **Inter-RAT complexity.** 5G↔4G handover involves MME/AMF interworking. N26 interface must be configured.

8. **Security context transfer.** Target cell needs security keys. If NH (Next Hop) chain breaks, handover fails.

## Why this matters

TS 23.502 is where 5G comes alive. While TS 23.501 defines WHAT exists, TS 23.502 defines HOW it works. Every signaling flow, every procedure, every state transition is here.

## How it works

### Registration procedure (Clause 4.2.2)

```
Initial Registration Flow:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  UE          gNB         AMF       AUSF/UDM    SMF      │
│  │            │           │           │          │       │
│  │──── RRC ───│           │           │          │       │
│  │  Setup     │           │           │          │       │
│  │            │           │           │          │       │
│  │──── Registration Request (via N1/N2) ────────│       │
│  │            │──────────→│           │          │       │
│  │            │           │           │          │       │
│  │            │           │── Nausf_UEAuth ────→│       │
│  │            │           │           │          │       │
│  │←── Authentication Request ─────────│          │       │
│  │            │           │           │          │       │
│  │──── Authentication Response ───────│          │       │
│  │            │           │           │          │       │
│  │            │           │── Nudm_UECM/SDM ───→│       │
│  │            │           │  (context, subs)     │       │
│  │            │           │           │          │       │
│  │←── Security Mode Command ──────────│          │       │
│  │            │           │           │          │       │
│  │──── Security Mode Complete ────────│          │       │
│  │            │           │           │          │       │
│  │            │           │── Nsmf (if PDU) ───────────→│
│  │            │           │           │          │       │
│  │←── Registration Accept ────────────│          │       │
│  │            │           │           │          │       │
│  │──── Registration Complete ─────────│          │       │
│  │            │           │           │          │       │
│                                                          │
│  Key Steps:                                             │
│  1. RRC connection establishment (RAN)                  │
│  2. Registration Request (NAS over N1)                  │
│  3. Authentication (AUSF/UDM)                           │
│  4. Security activation (NAS security)                  │
│  5. Context retrieval (UDM subscription)                │
│  6. PDU session (if requested, via SMF)                 │
│  7. Registration Accept (AMF to UE)                     │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### PDU session establishment (Clause 4.3.2)

```
PDU Session Establishment Flow:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  UE       gNB      AMF      SMF      UPF      PCF       │
│  │         │        │        │        │        │         │
│  │── PDU Session Establishment Request ────────│        │
│  │         │────────│        │        │        │         │
│  │         │        │        │        │        │         │
│  │         │        │── Nsmf_PDUSession_Create ───→│    │
│  │         │        │        │        │        │         │
│  │         │        │        │── Npcf_SMPolicy ───────→│ │
│  │         │        │        │  (get policy)   │        │
│  │         │        │        │        │        │         │
│  │         │        │        │── N4 Session ───│        │
│  │         │        │        │  Establishment  │        │
│  │         │        │        │        │        │         │
│  │         │        │←─ response with QoS ──────│        │
│  │         │        │        │        │        │         │
│  │         │←── N2 PDU Session Resource Setup ─│        │
│  │         │        │        │        │        │         │
│  │         │── N3 tunnel setup ───────│        │         │
│  │         │        │        │        │        │         │
│  │←── PDU Session Establishment Accept ────────│        │
│  │         │        │        │        │        │         │
│                                                          │
│  Key Steps:                                             │
│  1. UE requests PDU session (DNN, S-NSSAI)              │
│  2. AMF selects SMF (based on DNN, slice)               │
│  3. SMF gets policy from PCF                            │
│  4. SMF selects UPF (locality, capability)              │
│  5. SMF programs UPF via N4 (PFCP)                      │
│  6. AMF/gNB establish N3 tunnel                         │
│  7. UE receives IP address, QoS info                    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### Xn-based handover (Clause 4.9.1)

```
Xn Handover Flow:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  UE    Source gNB   Target gNB    AMF     SMF    UPF    │
│  │         │           │           │       │      │      │
│  │── Measurement Report ───│       │       │      │      │
│  │         │           │           │       │      │      │
│  │         │── Handover Request ──→│       │      │      │
│  │         │   (Xn-AP)    │        │       │      │      │
│  │         │           │           │       │      │      │
│  │         │←── Handover Request Ack ──│   │      │      │
│  │         │   (with DL forwarding)    │   │      │      │
│  │         │           │           │       │      │      │
│  │←── HO Command ──│   │           │       │      │      │
│  │  (via RRC)      │   │           │       │      │      │
│  │         │           │           │       │      │      │
│  │───── UE detaches ───│           │       │      │      │
│  │         │           │           │       │      │      │
│  │──────── Synchronize + HO Complete ──────│      │      │
│  │         │           │           │       │      │      │
│  │         │           │── Path Switch Request ─→│      │
│  │         │           │           │       │      │      │
│  │         │           │           │── N4 Mod ──→│      │
│  │         │           │           │  (new N3)   │      │
│  │         │           │           │       │      │      │
│  │         │           │←── Path Switch Ack ──────│      │
│  │         │           │           │       │      │      │
│  │         │←── UE Context Release ──│     │      │      │
│  │         │   (source cleanup)       │     │      │      │
│  │         │           │           │       │      │      │
│                                                          │
│  Key Characteristics:                                   │
│  - Direct signaling between gNBs (Xn interface)         │
│  - AMF involved only for path switch                    │
│  - SMF updates UPF with new N3 tunnel                   │
│  - Faster than N2-based handover                        │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### Service request (Clause 4.2.3)

```
Service Request Flow (UE-triggered):

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  UE (IDLE)     gNB         AMF        SMF        UPF    │
│  │              │           │          │          │      │
│  │── Service Request ──────→│          │          │      │
│  │   (NAS via RRC)          │          │          │      │
│  │              │           │          │          │      │
│  │              │           │── Context retrieval │      │
│  │              │           │   (if needed)       │      │
│  │              │           │          │          │      │
│  │              │←── N2 PDU Session Resource ────│      │
│  │              │    Setup Request     │          │      │
│  │              │           │          │          │      │
│  │              │── RRC Reconfiguration ──│       │      │
│  │←─────────────│           │          │          │      │
│  │              │           │          │          │      │
│  │── RRC Reconfig Complete ─│          │          │      │
│  │              │           │          │          │      │
│  │              │── N2 PDU Session Resource ────→│      │
│  │              │    Setup Response    │          │      │
│  │              │           │          │          │      │
│  │              │           │── N4 (update N3) ──→│      │
│  │              │           │          │          │      │
│  │←── Service Accept ───────│          │          │      │
│  │              │           │          │          │      │
│  │ (Now CONNECTED)          │          │          │      │
│                                                          │
│  Trigger: UE has data to send but is in IDLE mode       │
│  Result: RAN connection + N3 tunnel re-established      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**Reading procedure diagrams:**

```
How to Read MSCs (Message Sequence Charts):

1. Left-to-right = time progression
2. Vertical lines = entities (UE, NFs)
3. Arrows = messages/signals
4. Numbers = step sequence
5. Boxes = optional/conditional steps
6. Notes = additional conditions

Example notation:
  ────────→  = synchronous request
  - - - - →  = asynchronous/optional
  [condition] = optional step
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Reading only text.** MSC diagrams carry critical information. Study them.

2. **Skipping pre-conditions.** Each procedure has assumptions. Violating them = undefined behavior.

3. **Ignoring optional steps.** Steps in brackets [] may be mandatory in your scenario.

4. **Confusing procedures.** Registration ≠ Connection ≠ PDU Session. Know the difference.

5. **Missing error paths.** Success flows are shown first. Error handling is often in separate subclauses.

6. **Assuming one version.** Procedures evolve across releases. Check your target release.

7. **Not tracing service operations.** Nxxx_Service calls reference TS 29.xxx for details.

8. **Forgetting state.** Procedures depend on current UE/NF state. Track state machines.

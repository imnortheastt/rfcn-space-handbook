<section class="lesson-hook">

## Why this matters

AMF, SMF, and UPF are the "main characters" of 5G. But they need **supporting infrastructure** to function.

**NRF** enables service discovery. **NSSF** selects network slices. **PCF** enforces policies. **NEF** exposes APIs to external applications. Without these, the SBA architecture would collapse.

</section>

### The four supporting functions

```
Supporting NFs in 5G:

┌─────────────────────────────────────────────────────────┐

│  │ Service │   │  Slice  │   │ Policy  │   │  API    │ │
│  │Discovery│   │Selection│   │ Control │   │Exposure │ │
│  └────┬────┘   └────┬────┘   └────┬────┘   └────┬────┘ │
│       │             │             │             │       │
│       └─────────────┼─────────────┼─────────────┘       │
│                     │             │                      │
│                     ▼             ▼                      │
│              ┌───────────────────────┐                  │

```

| Function | One-line purpose |
|----------|------------------|
| **NRF** | "Where is the SMF that serves DNN X?" |
| **NSSF** | "Which slice should this UE use?" |
| **PCF** | "What QoS and rules apply to this session?" |
| **NEF** | "How can external apps talk to the 5G network?" |

## NRF — Network Repository Function

### What NRF does

NRF is the **service registry** for 5G SBA. Every NF registers its services with NRF; consumers discover services by querying NRF.

```
NRF Service Registry:

┌─────────────────────────────────────────────────────────┐
│                         NRF                              │
│                                                          │
│  Registered NFs:                                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │ NF: AMF-001                                      │    │
│  │   Type: AMF                                      │    │
│  │   Services: [Namf_Communication, Namf_Location]  │    │
│  │   Address: https://amf1.5gc.operator.com        │    │
│  │   PLMN: 310-260                                  │    │
│  │   Slices: [SST=1, SST=2]                        │    │
│  │   TAIs: [TAI1, TAI2, TAI3]                      │    │
│  ├─────────────────────────────────────────────────┤    │
│  │ NF: SMF-001                                      │    │
│  │   Type: SMF                                      │    │
│  │   Services: [Nsmf_PDUSession]                   │    │
│  │   DNNs: [internet, ims]                         │    │
│  │   ...                                            │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### NRF operations

```
NRF Services (Nnrf):

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Nnrf_NFManagement:                                     │
│  ├── NFRegister         (NF joins network)             │
│  ├── NFUpdate           (NF changes profile)           │
│  ├── NFDeregister       (NF leaves)                    │
│  └── NFStatusSubscribe  (watch for changes)            │
│                                                          │
│  Nnrf_NFDiscovery:                                      │
│  └── NFDiscover         (find NFs matching criteria)   │
│                                                          │
│  Nnrf_AccessToken:                                      │
│  └── AccessTokenRequest (OAuth2 token for NF-to-NF)    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

```

 │──────────────────────────►│
 │                           │
 │                      ┌────┴────┐
 │                      │ Search  │
 │                      │ registry│
 │                      └────┬────┘
 │                           │
 │  NFDiscover Response      │
 │  - SMF-001: https://...   │
 │  - SMF-002: https://...   │
 │◄──────────────────────────│
 │                           │
```

## NSSF — Network Slice Selection Function

### What NSSF does

NSSF helps select the appropriate **network slice** for a UE. Different slices serve different purposes (eMBB, URLLC, mMTC).

```
Network Slicing Concept:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│        Physical 5G Network                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │  ┌─────────────────────────────────────────┐   │    │
│  │  │ Slice 1 (SST=1): eMBB                   │   │    │
│  │  │ AMF-A, SMF-A, UPF-A                     │   │    │
│  │  └─────────────────────────────────────────┘   │    │
│  │                                                  │    │
│  │  ┌─────────────────────────────────────────┐   │    │
│  │  │ Slice 2 (SST=2): URLLC                  │   │    │
│  │  │ AMF-B, SMF-B, UPF-B (edge)              │   │    │
│  │  └─────────────────────────────────────────┘   │    │
│  │                                                  │    │
│  │  ┌─────────────────────────────────────────┐   │    │
│  │  │ Slice 3 (SST=3): mMTC                   │   │    │
│  │  │ AMF-C, SMF-C, UPF-C                     │   │    │
│  │  └─────────────────────────────────────────┘   │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  NSSF decides: "UE X gets Slice 1 + Slice 2"           │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## PCF — Policy Control Function

### What PCF does

PCF provides **policy rules** to AMF (access policies) and SMF (session policies). It decides QoS, charging rules, and traffic treatment.

```
PCF in the Policy Framework:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│                    ┌─────────┐                          │
│                    │   UDR   │ (policy data)            │
│                    └────▲────┘                          │
│                         │                                │
│                    ┌────┴────┐                          │
│                    │   PCF   │                          │
│                    └────┬────┘                          │
│                         │                                │
│           ┌─────────────┼─────────────┐                 │
│           │             │             │                  │
│           ▼             ▼             ▼                  │
│      ┌─────────┐   ┌─────────┐   ┌─────────┐           │
│      │   AMF   │   │   SMF   │   │   AF    │           │
│      │         │   │         │   │(via NEF)│           │
│      │ Access  │   │ Session │   │ App     │           │
│      │ Policy  │   │ Policy  │   │ Request │           │
│      └─────────┘   └─────────┘   └─────────┘           │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## NEF — Network Exposure Function

### What NEF does

NEF is the **API gateway** between external applications and the 5G core. It exposes network capabilities safely and securely.

```
NEF as API Gateway:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  External World              │      5G Core             │
│                              │                          │
│  ┌─────────┐                │                          │
│  │ IoT App │────┐           │                          │
│  └─────────┘    │           │                          │
│                  │           │                          │
│  ┌─────────┐    │    ┌──────┴─────┐    ┌─────────┐    │
│  │ Video   │────┼───►│    NEF     │───►│   PCF   │    │
│  │ Service │    │    │            │    └─────────┘    │
│  └─────────┘    │    │ - Validate │                    │
│                  │    │ - Translate│    ┌─────────┐    │
│  ┌─────────┐    │    │ - Authorize│───►│   UDM   │    │
│  │ Traffic │────┘    │            │    └─────────┘    │
│  │ Mgmt    │         └──────┬─────┘                    │
│  └─────────┘                │          ┌─────────┐    │
│                              │         │   AMF   │    │
│                              │         └─────────┘    │
│                              │                          │
└─────────────────────────────────────────────────────────┘
```

### NEF services

```
NEF Services (Nnef):

┌─────────────────────────────────────────────────────────┐

```

**NRF configuration:**

| Parameter | Purpose |
|-----------|---------|
| PLMN ID | Network identity |
| NRF peers | For NRF-to-NRF communication |
| OAuth2 config | Token-based authorization |

**NSSF configuration:**

| Parameter | Purpose |
|-----------|---------|

| AMF set info | Target AMFs per slice |

**PCF configuration:**

| Parameter | Purpose |
|-----------|---------|
| UDR endpoint | Policy data repository |
| Default policies | Fallback rules |
| Charging rules | Online/offline charging |

**NEF configuration:**

| Parameter | Purpose |
|-----------|---------|
| AF allowlist | Authorized external apps |
| API throttling | Rate limiting |
| Exposure scopes | What capabilities are exposed |

<section class="lesson-pitfalls">

## Pitfalls

1. **NRF as single point of failure.** NRF outage stops discovery. Deploy redundant NRF instances.

2. **NSSF complexity.** Slice selection logic can get complex. Start simple, add rules incrementally.

3. **PCF overload.** Every session queries PCF. Monitor and scale appropriately.

4. **NEF security gaps.** Exposing network APIs to external apps requires strict authorization. Audit AF access.

5. **Caching NRF results.** Stale discovery data causes routing to dead NFs. Set appropriate TTLs.

6. **Ignoring NSSAI consistency.** Requested, allowed, and configured NSSAI must align. Mismatches cause slice selection failures.

7. **PCF-UDR split brain.** Policy data in UDR and PCF runtime state must stay synchronized.

8. **NEF rate limiting.** External AFs can flood NEF with requests. Implement throttling from day one.

## Why this matters

AMF handles mobility. But when your phone needs to **send data** — browse the web, stream video, or upload files — that's **SMF** territory.

SMF manages **PDU sessions**: the data pipes between your device and the internet. It decides where your data goes, how it's treated (QoS), and controls the UPF that actually moves packets.

</section>

<section class="lesson-intuition">

## Intuition

### What SMF does

```
SMF's Role:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│                    ┌─────────┐                          │
│                    │   PCF   │ (policy)                 │
│                    └────▲────┘                          │
│                         │ Npcf                          │
│                         │                                │
│  ┌─────────┐      ┌─────┴─────┐      ┌─────────┐       │
│  │   AMF   │◄────►│    SMF    │◄────►│   UDM   │       │
│  └─────────┘ Nsmf └─────┬─────┘ Nudm └─────────┘       │
│                         │                                │
│                         │ N4 (PFCP)                      │
│                         │                                │
│                    ┌────▼────┐                          │
│                    │   UPF   │──────► Internet          │
│                    └─────────┘                          │
│                                                          │
│  SMF: Session control (signaling)                       │
│  UPF: Packet forwarding (data plane)                    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**Key insight:** SMF is control plane only. It tells UPF what to do but never touches user data.

### Key responsibilities

| Responsibility | What it means |
|----------------|---------------|
| **PDU session establishment** | Create data paths for UE |
| **UPF selection and control** | Choose UPF, install forwarding rules |
| **IP address allocation** | Assign IPv4/IPv6 to UE |
| **QoS management** | Configure QoS flows and rules |
| **Policy enforcement** | Apply PCF decisions |
| **Charging integration** | Interface with CHF for billing |
| **SSC mode handling** | Session continuity behavior |

### From 4G to 5G session management

```
4G (Combined):
┌─────────────────────────────────────────────────────────┐
│  MME: Session signaling (GTP-C to SGW/PGW)              │
│  SGW: User plane anchor                                  │
│  PGW: IP assignment, policy, charging                   │
└─────────────────────────────────────────────────────────┘

5G (Separated):
┌─────────────────────────────────────────────────────────┐
│  AMF: Mobility only (no session management)             │
│  SMF: Session signaling, QoS, IP allocation            │
│  UPF: Packet forwarding (decoupled from control)       │
└─────────────────────────────────────────────────────────┘
```

**Benefit:** Scale SMF for session capacity independently from AMF mobility capacity.

### N4 interface (PFCP)

SMF controls UPF via **PFCP (Packet Forwarding Control Protocol)**:

```
N4 (PFCP) Operations:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  SMF ──────────────────────────────────────────► UPF    │
│                     PFCP                                 │
│                                                          │
│  Session Management:                                    │
│  ├── Session Establishment Request/Response             │
│  ├── Session Modification Request/Response              │
│  └── Session Deletion Request/Response                  │
│                                                          │
│  Node Management:                                       │
│  ├── Association Setup/Release                          │
│  └── Heartbeat                                          │
│                                                          │
│  PFCP Session contains:                                 │
│  ┌─────────────────────────────────────────────────┐   │
│  │ PDR (Packet Detection Rule): match criteria     │   │
│  │ FAR (Forwarding Action Rule): what to do        │   │
│  │ QER (QoS Enforcement Rule): rate limits         │   │
│  │ URR (Usage Reporting Rule): billing             │   │
│  └─────────────────────────────────────────────────┘   │
│                                                          │
│  Transport: UDP port 8805                               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

5G QoS Model:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  PDU Session                                            │

│  │   └── QFI = 1                                        │
│  ├── QoS Flow 2 (5QI=9, video)                         │
│  │   └── QFI = 2                                        │

│                                                          │
│  SMF responsibilities:                                  │
│  ├── Create QoS flows based on PCF rules               │
│  ├── Assign QFI (QoS Flow Identifier)                  │
│  ├── Configure 5QI parameters                          │
│  └── Install QoS rules in UPF                          │
│                                                          │
└─────────────────────────────────────────────────────────┘

5QI Table (examples):
┌──────┬────────────────┬─────────┬───────────┐
│ 5QI  │ Resource Type  │ Delay   │ Use Case  │

```

### SSC modes (Session and Service Continuity)

```
SSC Modes:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  SSC Mode 1: IP address preserved                       │
│  ┌─────────────────────────────────────────────────┐   │
│  │ UE moves → Same UPF or anchor UPF maintained    │   │
│  │ IP address: unchanged                            │   │
│  │ Use case: TCP sessions, enterprise VPN          │   │
│  └─────────────────────────────────────────────────┘   │
│                                                          │
│  SSC Mode 2: Break-before-make                          │
│  ┌─────────────────────────────────────────────────┐   │
│  │ UE moves → Release old session, create new      │   │
│  │ IP address: changes                              │   │
│  │ Use case: Best effort traffic                   │   │
│  └─────────────────────────────────────────────────┘   │
│                                                          │
│  SSC Mode 3: Make-before-break                          │
│  ┌─────────────────────────────────────────────────┐   │
│  │ UE moves → New session established first        │   │
│  │ IP address: both available temporarily          │   │
│  │ Use case: Smooth handover for real-time apps   │   │
│  └─────────────────────────────────────────────────┘   │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### SMF selection

```
SMF Selection Criteria:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Inputs:                                                │
│  ├── DNN (Data Network Name) - like APN                │
│  ├── S-NSSAI (requested slice)                         │
│  ├── UE location (TAI)                                 │
│  └── Subscription info (UDM)                           │
│                                                          │
│  Selection factors:                                     │
│  ├── DNN support                                       │
│  ├── S-NSSAI support                                   │
│  ├── Serving area                                      │
│  ├── Load balancing                                    │
│  └── Subscription restrictions                          │
│                                                          │
│  Process:                                               │
│  1. AMF queries NRF for SMF supporting DNN/S-NSSAI     │

│  4. AMF sends CreateSMContext to selected SMF          │
│                                                          │

## Apply

**SMF configuration parameters:**

| Parameter | Purpose | Example |
|-----------|---------|---------|
| DNN list | Supported data networks | "internet", "ims", "enterprise" |
| IP pools | IPv4/IPv6 ranges | 10.0.0.0/8, fd00::/48 |
| S-NSSAI | Supported slices | SST=1, SST=2/SD=001 |
| UPF list | Known UPFs | UPF instances with N4 addresses |
| Default 5QI | Default QoS | 9 (best effort) |
| DNS servers | For UE | Primary, secondary DNS |

**Capacity metrics:**

| Metric | Description |
|--------|-------------|
| Active PDU sessions | Concurrent sessions |
| Session setup rate | Establishments/second |
| N4 transactions | PFCP messages/second |
| IP pool utilization | Used vs available IPs |

**Troubleshooting:**

| Issue | Check |
|-------|-------|
| Session fails | DNN configured, UPF reachable |
| No IP assigned | Pool exhaustion, DHCP timeout |
| QoS not applied | PCF policy, UPF rules installed |
| UPF unreachable | N4 heartbeat, PFCP association |
| Slow setup | N4 latency, NRF discovery |

<section class="lesson-pitfalls">

## Pitfalls

1. **SMF handles user traffic (wrong).** SMF is control plane only. If you're debugging data path issues, look at UPF.

2. **One SMF per DNN.** Multiple SMFs can serve the same DNN. Plan for load distribution.

3. **IP pool sizing.** Underestimate IP pool size and sessions will fail. Monitor utilization.

4. **Ignoring N4 heartbeat.** Lost PFCP association means lost UPF control. Tune heartbeat timers.

5. **QoS without PCF.** SMF can assign QoS, but PCF provides policy. Static config works for simple cases only.

6. **SSC mode mismatch.** Application needs and SSC mode must align. TCP over SSC mode 2 breaks connections.

7. **UPF selection complexity.** Edge deployments need careful UPF selection logic. Test mobility scenarios.

8. **Missing DNS in session.** UE needs DNS servers. Configure in SMF or PCF policy.

</section>

## Why this matters

When your phone powers on, registers with the network, moves between cells, or transitions from idle to connected — **AMF** handles it.

AMF is the **control plane anchor** for 5G. Every UE talks to AMF. Every gNB connects to AMF. Understanding AMF is understanding the heartbeat of 5G signaling.

</section>

<section class="lesson-intuition">

## Intuition

### What AMF does

```
AMF at the Center:

                         ┌─────────────────┐
                         │      UDM        │
                         │ (subscriber data)│
                         └────────▲────────┘
                                  │ Nudm
                                  │
┌─────────┐    N2/NGAP    ┌───────┴───────┐    Nsmf     ┌─────────┐
│  gNB    │◄─────────────►│      AMF      │◄───────────►│   SMF   │
└─────────┘               │               │             └─────────┘
                          │  (signaling   │
┌─────────┐    N1/NAS     │    hub)       │    Nausf    ┌─────────┐
│   UE    │◄─────────────►│               │◄───────────►│  AUSF   │
└─────────┘               └───────┬───────┘             └─────────┘
                                  │ Npcf
                                  │
                         ┌────────▼────────┐
                         │      PCF        │
                         │ (access policy) │
                         └─────────────────┘
```

**AMF is the UE's single contact point** for all control plane signaling.

### Key responsibilities

| Responsibility | What it means |
|----------------|---------------|
| **Registration** | UE attachment, authentication coordination |
| **Connection management** | Idle/Connected state, paging |
| **Mobility management** | Handover, TAI list management |
| **Access control** | Network slice selection, access restrictions |
| **NAS termination** | End-to-end NAS signaling with UE |
| **N2 termination** | Control plane signaling with gNB |

### From 4G MME to 5G AMF

```
4G MME vs 5G AMF:

┌─────────────────────────────────────────────────────────┐
│                      4G MME                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │ Registration │ Mobility │ Session │ Bearer     │    │
│  │              │          │ Mgmt    │ Mgmt       │    │
│  └─────────────────────────────────────────────────┘    │
│  All in one monolith                                    │
└─────────────────────────────────────────────────────────┘
                         │
                         ▼ Decomposition
┌─────────────────────────────────────────────────────────┐
│                      5G SBA                              │
│                                                          │
│  ┌───────────┐    ┌───────────┐    ┌───────────┐       │
│  │    AMF    │    │    SMF    │    │    UPF    │       │
│  │           │    │           │    │           │       │
│  │ Mobility  │    │ Session   │    │ User      │       │
│  │ Access    │    │ Mgmt      │    │ Plane     │       │
│  └───────────┘    └───────────┘    └───────────┘       │
│                                                          │
│  Session management split off to SMF                    │
└─────────────────────────────────────────────────────────┘
```

**Key difference:** AMF handles mobility and access; SMF handles sessions. Clean separation.

### AMF selection

When UE first connects, the network selects an AMF:

```
AMF Selection Factors:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  1. GUAMI in Initial UE Message (if present)            │
│     - UE provides previous AMF identifier               │
│     - Network tries to route to same AMF                │
│                                                          │
│  2. Requested NSSAI                                     │
│     - AMF must support UE's requested slices            │
│     - NSSF may assist in selection                      │
│                                                          │
│  3. Registration Area                                   │
│     - AMF serves specific TAIs                          │
│     - gNB routes to appropriate AMF                     │
│                                                          │
│  4. Load balancing                                      │
│     - Distribute UEs across AMF pool                    │
│     - Weight-based or round-robin                       │
│                                                          │
│  5. Operator policy                                     │
│     - VIP subscribers to specific AMF                   │
│     - Geographic considerations                          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## Apply

**AMF configuration parameters:**

| Parameter | Purpose | Typical value |
|-----------|---------|---------------|
| PLMN ID | Served PLMN | MCC-MNC (e.g., 310-260) |
| AMF Set ID | AMF pool grouping | 0-1023 |
| AMF Pointer | AMF within set | 0-63 |
| TAI List | Served tracking areas | List of TAIs |
| Supported NSSAIs | Slice support | SST/SD combinations |
| T3512 | Periodic registration timer | 54 min default |
| T3502 | Registration retry timer | 12 min default |

**AMF capacity planning:**

| Metric | Description |
|--------|-------------|
| Max UE contexts | Number of registered UEs |
| Registration rate | Registrations per second |
| Paging capacity | Pages per second |
| N2 connections | Number of gNBs |
| SBI throughput | HTTP/2 requests per second |

**Troubleshooting checklist:**

| Issue | Check |
|-------|-------|
| Registration fails | AUSF/UDM reachable, NRF registration |
| Paging fails | TAI list correct, gNB N2 status |
| Handover fails | Target AMF reachable, GUAMI routing |
| Slow performance | CPU/memory, SBI latency, N2 congestion |
| UE unreachable | CM state, last known TAI, paging timer |

<section class="lesson-pitfalls">

## Pitfalls

1. **Confusing AMF with MME.** AMF doesn't handle sessions — that's SMF. Don't expect bearer management in AMF.

2. **Ignoring TAI planning.** Poor TAI design causes excessive registration updates. Plan TAI lists for mobility patterns.

3. **Underestimating paging load.** IoT devices with frequent data can overwhelm paging. Consider always-connected strategies.

4. **Single AMF deployment.** AMF pools with load balancing are essential for resilience and scale.

5. **Forgetting inter-AMF mobility.** UE moving between AMFs requires context transfer. Test inter-AMF handover.

6. **NAS security misconfiguration.** Wrong algorithm support breaks authentication. Verify NAS security capability.

7. **N2 SCTP issues.** SCTP multi-homing and heartbeats require network configuration. Firewalls often block SCTP.

8. **Timer misalignment.** T3512 (periodic registration) should align with expected UE behavior and network load.

## Why this matters

When you insert your SIM and connect to a 5G network, two things happen: the network verifies you're a real subscriber (**authentication**), and it retrieves your subscription data (**who you are and what you're allowed to do**).

**AUSF** handles authentication. **UDM** manages subscriber data. Together, they are the trust foundation of 5G.

</section>

<section class="lesson-intuition">

## Intuition

### AUSF and UDM in the architecture

```
Authentication and Data Flow:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│                    ┌─────────┐                          │
│                    │   UDR   │ (database)               │
│                    └────▲────┘                          │
│                         │ Nudr                          │
│                         │                                │
│  ┌─────────┐      ┌─────┴─────┐      ┌─────────┐       │
│  │   AMF   │◄────►│    UDM    │◄────►│  AUSF   │       │
│  └─────────┘ Nudm │           │ Nausf└────▲────┘       │
│       │           │ Subscriber │           │            │
│       │           │ Data Mgmt  │           │            │
│       │           └───────────┘           │            │
│       │                                    │            │
│       └────────────────────────────────────┘            │
│              Authentication request                     │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**Flow:** AMF asks AUSF to authenticate → AUSF asks UDM for credentials → UDM gets from UDR → authentication completes.

### What each function does

| Function | Responsibility |
|----------|----------------|
| **AUSF** | Execute 5G-AKA/EAP-AKA', validate UE identity |
| **UDM** | Subscriber data, authentication credential generation, registration management |
| **UDR** | Database storage (stateless data repository) |

### From 4G HSS to 5G UDM/AUSF

```
4G HSS:
┌─────────────────────────────────────────────────────────┐
│                         HSS                              │
│  ┌─────────────────────────────────────────────────┐    │
│  │ Authentication │ Subscriber │ Location │ EPC   │    │
│  │ Vector Gen     │ Data       │ Updates  │ Specific│   │
│  └─────────────────────────────────────────────────┘    │
│  Monolithic, combined functions                         │
└─────────────────────────────────────────────────────────┘

5G UDM/AUSF:
┌─────────────────────────────────────────────────────────┐
│                                                          │
│  ┌─────────┐      ┌─────────┐      ┌─────────┐         │
│  │  AUSF   │      │   UDM   │      │   UDR   │         │
│  │         │      │         │      │         │         │
│  │  Auth   │◄────►│ Sub Data│◄────►│ Storage │         │
│  │  Only   │      │ Mgmt    │      │ Only    │         │
│  └─────────┘      └─────────┘      └─────────┘         │
│                                                          │
│  Separated, scalable, cloud-native                      │
└─────────────────────────────────────────────────────────┘
```

**Benefit:** Scale authentication independently from data management.

## Apply

**AUSF configuration:**

| Parameter | Purpose |
|-----------|---------|
| PLMN ID | Served network |
| UDM peer | UDM endpoint |
| SUPI ranges | Subscribers to serve |
| Key derivation | Algorithm selection |

**UDM configuration:**

| Parameter | Purpose |
|-----------|---------|
| PLMN ID | Home network |
| UDR endpoint | Data repository |
| SUPI ranges | Managed subscribers |
| Default NSSAI | Default slice config |

**Subscriber provisioning:**

```yaml
# Example UDM subscriber data
supi: "imsi-310260123456789"
gpsi:
  - "msisdn-+14155551234"
authentication:
  authMethod: "5G_AKA"
  encPermanentKey: "base64-encoded-K"
  encOpcKey: "base64-encoded-OPc"
accessAndMobilitySubscription:
  nssai:
    defaultSingleNssais:
      - sst: 1
      - sst: 2
        sd: "000001"
  ratRestrictions: []
  subscribedUeAmbr:
    uplink: "100 Mbps"
    downlink: "500 Mbps"
sessionManagement:
  - dnn: "internet"
    sscModes:
      defaultSscMode: "SSC_MODE_1"
    pduSessionTypes:
      defaultSessionType: "IPV4"
```

**Troubleshooting:**

| Issue | Check |
|-------|-------|
| Auth fails | K/OPc match between UE and UDR |
| SQN sync | Resync procedure, SQN in UDR |
| Subscription not found | SUPI provisioned, UDR reachable |
| Wrong slice | Default NSSAI configured |

<section class="lesson-pitfalls">

## Pitfalls

1. **AUSF stores subscriber data (wrong).** AUSF only executes authentication. UDM/UDR stores data.

2. **SQN synchronization issues.** If UE and network SQN diverge, auth fails. Implement resync procedure.

3. **K/OPc mismatch.** The permanent key must match exactly between USIM and UDR. Any mismatch = auth failure.

4. **Ignoring roaming.** UDM needs steering of roaming (SoR) configuration for visited networks.

5. **UDR as single point of failure.** UDR holds all subscription data. Plan for redundancy and backup.

6. **Hardcoding NSSAI.** Default/allowed NSSAI should come from subscription, not hardcoded in AMF/SMF.

7. **Missing UECM registration.** If UDM doesn't know where UE is registered, MT services fail.

8. **Authentication timeout.** AUSF/UDM latency affects registration time. Monitor SBI response times.

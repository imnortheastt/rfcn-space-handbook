## Why this matters

A traditional GEO communications satellite costs $200-500M and takes 3-5 years to build. A Starlink satellite costs under $250k and SpaceX builds 6 per day. That's a 1000x cost reduction and 100x speed increase. The secret isn't better technology — it's manufacturing philosophy. SpaceX, Planet, and OneWeb treat satellites like consumer electronics: standardized designs, COTS components, assembly lines, and accepting some unit failures instead of perfecting each one. Understanding mass production economics tells you which space businesses will win.

### The learning curve effect

```
Manufacturing Learning Curve:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Wright's Law: Every doubling of cumulative production             │
│                reduces unit cost by a fixed percentage (15-25%)    │
│                                                                      │
│  Cost per unit = C₁ × n^(-b)                                       │
│  where b = -log₂(learning rate)                                    │
│                                                                      │
│  EXAMPLE: Starlink production                                       │
│                                                                      │
│  Unit#     Cost      Learning Effect                               │
```

### Design for manufacturability

```
Design Principles that Enable Mass Production:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  1. FLAT-PACK ARCHITECTURE                                          │
│     • Starlink v1.5: flat plate design, no deployable antennas     │
│     • Fits 60 in Falcon 9 fairing (dense packing)                  │
│     • No complex mechanisms = fewer failure modes                   │
│                                                                      │
│  2. MODULAR SUBSYSTEMS                                              │
│     • Plug-and-play electronics boards                              │
│     • Standard interfaces (connectors, mounting)                    │
│     • Replace failed module, don't debug it                         │
│                                                                      │
│  3. REDUCED PART COUNT                                              │
│     ┌───────────────────────────────────────────────────┐          │
│     │  Traditional GEO satellite: ~500,000 parts        │          │
│     │  Starlink v1.0:              ~2,000 parts        │          │
│     │  Starlink v2:                ~1,500 parts        │          │
│     └───────────────────────────────────────────────────┘          │
│     • Each part = assembly step, test point, failure mode         │
│     • 250x fewer parts = 250x simpler production                   │
│                                                                      │
│  4. DESIGN FREEZE                                                   │
│     • Lock design after qualification                               │
│     • No mid-production engineering changes                        │
│     • Save improvements for next block/generation                  │
│     • "Kaizen" happens between generations, not during             │
│                                                                      │
│  5. SOFTWARE-DEFINED FUNCTIONS                                      │
│     • Same hardware, different firmware = different capability     │
│     • Field upgrades via uplink (no hardware change)               │
│     • Defer decisions from build-time to deploy-time              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Apply

**Calculate your manufacturing economics:**

```python
import numpy as np

def learning_curve_cost(first_unit_cost, unit_number, learning_rate=0.80):
    """
    Calculate unit cost based on Wright's Law learning curve.
    
    Args:
        first_unit_cost: Cost of the first unit produced
        unit_number: Which unit in the production sequence
        learning_rate: Cost ratio after doubling production (0.80 = 20% reduction)
    
    Returns:
        Cost of the nth unit
    """
    b = np.log(learning_rate) / np.log(2)
    return first_unit_cost * (unit_number ** b)

def cumulative_cost(first_unit_cost, total_units, learning_rate=0.80):
    """Calculate total cost of producing N units."""
    costs = [learning_curve_cost(first_unit_cost, n+1, learning_rate) 
             for n in range(total_units)]
    return sum(costs)

def average_unit_cost(first_unit_cost, total_units, learning_rate=0.80):
    """Calculate average cost per unit across production run."""
    return cumulative_cost(first_unit_cost, total_units, learning_rate) / total_units

def assembly_line_capacity(takt_time_min, stations, shifts_per_day, hours_per_shift=8):
    """
    Calculate daily production capacity.
    
    Args:
        takt_time_min: Time per station in minutes
        stations: Number of assembly stations
        shifts_per_day: Number of shifts
        hours_per_shift: Hours per shift
    
    Returns:
        Maximum daily output
    """
    minutes_per_day = shifts_per_day * hours_per_shift * 60
    return minutes_per_day / takt_time_min

def cots_vs_space_grade_savings(bill_of_materials, space_grade_multiplier=1000):
    """
    Calculate savings from COTS component strategy.
    
    Args:
        bill_of_materials: Dict of {component: cots_price}
        space_grade_multiplier: Average space-grade price multiple
    
    Returns:
        COTS total, space-grade equivalent total, savings
    """
    cots_total = sum(bill_of_materials.values())
    space_total = cots_total * space_grade_multiplier
    return cots_total, space_total, space_total - cots_total
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Underestimating NRE.** The first satellite costs 10-100x the hundredth. Budget for learning.

2. **Over-optimizing for unit cost.** If you only need 50 satellites, mass production loses to craft production.

3. **Supply chain single points of failure.** COTS means consumer lead times — a chip shortage stops your line.

4. **Ignoring test time.** Assembly may be 45 minutes; full functional test adds hours. Plan for it.

5. **Designing during production.** Every engineering change costs 10x on the factory floor. Freeze the design.

6. **Assuming linear scaling.** 10x production doesn't mean 10x facility — common resources become bottlenecks.

## Why this matters

Traditional aerospace optimizes for performance: maximum capability, minimum risk, cost be damned. NewSpace flips this: optimize for cost, accept "good enough" performance. A GEO satellite might use a $200k rad-hardened processor with 15-year guaranteed life. Starlink uses a $5 automotive chip, accepts 2% failure rate, and replaces dead satellites. Both are rational — for different business models. Understanding trade space analysis tells you how to make these decisions systematically, not by gut feel.

### The trade space concept

```

│  and their performance/cost outcomes.                               │
│                                                                      │
│                                                                      │
│  Performance                                                        │
│      ▲                                                              │
│      │              ★ Traditional (high perf, high cost)           │
│      │             /                                                │
│      │            /                                                 │
│      │           /   Pareto frontier                               │
│      │          /    (efficient designs)                           │
│      │         /                                                    │
│      │        ●━━━━━━━━━━━━●                                       │
│      │       /              ╲                                       │
│      │      /  Trade space   ╲                                     │
│      │     /   (all feasible  ╲                                    │
│      │    /    designs)        ★ NewSpace (good-enough, low cost) │

### The fundamental tradeoffs

```
Primary Engineering Tradeoffs:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  1. COST ↔ PERFORMANCE                                             │
│     ┌───────────────────────────────────────────────────────┐      │
│     │  More capable = more expensive                        │      │
│     │  Example: 10 Gbps link costs 3x more than 1 Gbps     │      │
│     │                                                        │      │
│     │  Decision: What performance do customers actually pay │      │
│     │            for? Don't gold-plate.                     │      │
│     └───────────────────────────────────────────────────────┘      │
│                                                                      │
│  2. COST ↔ RELIABILITY                                             │
│     ┌───────────────────────────────────────────────────────┐      │
│     │  Higher reliability = more testing, redundancy, cost  │      │
│     │  Example: Rad-hard chip ($200k) vs automotive ($5)   │      │
│     │                                                        │      │
│     │  Decision: Is fleet redundancy cheaper than unit      │      │
│     │            reliability? (Often yes for LEO)           │      │
│     └───────────────────────────────────────────────────────┘      │
│                                                                      │
│  3. COST ↔ SCHEDULE                                                │
│     ┌───────────────────────────────────────────────────────┐      │
│     │  Faster = more expensive (overtime, parallel dev)     │      │
│     │  Example: 6-month vs 18-month development cycle      │      │
│     │                                                        │      │
│     │  Decision: Does time-to-market matter more than      │      │
│     │            development cost? (Usually yes for startups)│     │
│     └───────────────────────────────────────────────────────┘      │
│                                                                      │
│  4. MASS ↔ COST                                                    │
│     ┌───────────────────────────────────────────────────────┐      │
│     │  Lighter = more expensive (exotic materials, machining)│     │
│     │  Example: Titanium ($50/kg) vs Aluminum ($3/kg)       │      │
│     │                                                        │      │
│     │  Decision: Compare mass savings × $/kg_launch vs      │      │
│     │            material premium. Do the math.             │      │
│     └───────────────────────────────────────────────────────┘      │
│                                                                      │
│  5. POWER ↔ MASS ↔ COST                                            │
│     ┌───────────────────────────────────────────────────────┐      │
│     │  More power = bigger solar arrays = more mass/cost    │      │
│     │  Example: 1 kW solar array ≈ 3 kg, 500 W payload     │      │
│     │                                                        │      │
│     │  Decision: What's the minimum power that delivers    │      │
│     │            the required capability?                   │      │

│                                                                      │
└─────────────────────────────────────────────────────────────────────┘

## How it works

### Design-to-cost methodology

```
Design-to-Cost Process:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  TRADITIONAL: Design → Cost → Shock → Redesign → Iterate           │
│  (Cost is discovered after design, always too high)                │
│                                                                      │
│  DESIGN-TO-COST: Target Cost → Design Constraints → Design         │
│  (Cost is an input, not an output)                                 │
│                                                                      │
│  STEP 1: Set target cost from business model                       │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Revenue model: $100/month × 1M subscribers = $100M/mo│         │
│  │  Required margin: 50%                                 │         │
│  │  Allowable cost: $50M/month                           │         │
│  │  Satellite fleet needed: 1,000 satellites             │         │
│  │  Constellation refresh: 5 years                       │         │
│  │  Satellites/year: 200                                 │         │
│  │  Total sat budget: $50M × 12 × 0.3 = $180M/year      │         │
│  │  Target cost per satellite: $900k                     │         │
│  │                                                        │         │
│  │  → This is your design constraint, not a wish.        │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  STEP 2: Allocate cost budget to subsystems                        │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Subsystem        │ % Budget │ Allocation             │         │
│  │  ──────────────────┼──────────┼────────────────        │         │
│  │  Payload (comms)   │    40%   │ $360k                  │         │
│  │  Avionics          │    15%   │ $135k                  │         │
│  │  Power system      │    15%   │ $135k                  │         │
│  │  Propulsion        │    10%   │ $90k                   │         │
│  │  Structure         │    10%   │ $90k                   │         │
│  │  Thermal           │     5%   │ $45k                   │         │
│  │  AIT (test/integ)  │     5%   │ $45k                   │         │
│  │  ──────────────────┼──────────┼────────────────        │         │
│  │  TOTAL             │   100%   │ $900k                  │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  STEP 3: Design each subsystem to its budget                       │
│  (If you can't hit the budget, trade performance or de-scope)     │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Quantitative trade study

```
How to Run a Trade Study:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  EXAMPLE: Choose a communications subsystem                        │
│                                                                      │
│  STEP 1: Define evaluation criteria and weights                    │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Criterion       │ Weight │ Rationale                 │         │
│  │  ─────────────────┼────────┼────────────────────────── │         │
│  │  Unit cost       │   35%  │ Business model driver    │         │
│  │  Data rate       │   25%  │ Customer value           │         │
│  │  Mass            │   15%  │ Launch cost impact       │         │
│  │  Power           │   10%  │ Solar array sizing       │         │
│  │  TRL (maturity)  │   10%  │ Schedule risk            │         │
│  │  Reliability     │    5%  │ Fleet handles failures   │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  STEP 2: Score each option (1-5 scale)                             │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Option        │Cost│Rate│Mass│Power│TRL │Rel │SCORE │         │
│  │  ──────────────┼────┼────┼────┼─────┼────┼────┼──────│         │
│  │  Option A: Ka  │  4 │  5 │  3 │   3 │  5 │  4 │ 4.05 │         │
│  │  Option B: Ku  │  5 │  3 │  4 │   4 │  5 │  4 │ 4.15 │ ← WIN  │
│  │  Option C: V   │  2 │  5 │  2 │   2 │  3 │  3 │ 2.85 │         │
│  │  Option D: Opt │  1 │  4 │  5 │   5 │  2 │  2 │ 2.65 │         │
│  │                                                        │         │
│  │  Score = Σ(weight × score) for each criterion         │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  STEP 3: Sensitivity analysis                                      │
│  "What if cost weight changes from 35% to 50%?"                   │
│  → Still Option B? If not, decision is sensitive to assumptions.  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

import numpy as np

def weighted_score(scores, weights):
    """
    Calculate weighted score for trade study.
    
    Args:
        scores: Dict of {criterion: score} (1-5 scale)
        weights: Dict of {criterion: weight} (sum to 1.0)
    
    Returns:
        Weighted total score
    """
    total = sum(scores[c] * weights[c] for c in scores)
    return total

def run_trade_study(options, criteria, weights):
    """
    Run a complete trade study.
    
    Args:
        options: Dict of {option_name: {criterion: score}}
        criteria: List of criterion names
        weights: Dict of {criterion: weight}
    
    Returns:
        Ranked list of (option_name, score)
    """
    results = []
    for name, scores in options.items():
        total = weighted_score(scores, weights)
        results.append((name, total))
    
    return sorted(results, key=lambda x: -x[1])

def sensitivity_analysis(options, criteria, base_weights, vary_criterion, vary_range):
    """
    Test how ranking changes as one weight varies.
    """
    results = []
    for weight_val in vary_range:
        weights = base_weights.copy()
        weights[vary_criterion] = weight_val
        remaining = 1 - weight_val
        other_sum = sum(base_weights[c] for c in criteria if c != vary_criterion)
        
        for c in criteria:
            if c != vary_criterion:
                weights[c] = base_weights[c] * remaining / other_sum
        
        ranking = run_trade_study(options, criteria, weights)
        winner = ranking[0][0]
        results.append((weight_val, winner))
    
    return results

def mass_budget_check(allocations, actuals, margin_required=0.10):
    """
    Check mass budget status.
    """
    results = []
    for subsystem in allocations:
        alloc = allocations[subsystem]
        actual = actuals.get(subsystem, 0)
        margin = (alloc - actual) / alloc
        status = "OK" if margin >= margin_required else "WARNING"
        results.append({
            "subsystem": subsystem,
            "allocated": alloc,
            "actual": actual,
            "margin_pct": margin * 100,
            "status": status
        })
    return results

def design_to_cost_allocation(target_cost, subsystem_percentages):
    """
    Allocate cost budget to subsystems.
    """
    return {sub: target_cost * pct for sub, pct in subsystem_percentages.items()}

## Pitfalls

1. **Unstated weights.** Every decision has implicit weights. Make them explicit or you'll argue about scores when you disagree on priorities.

2. **3-point-itis.** Rating everything 3/5 makes all options equal. Force yourself to use 1s and 5s.

3. **Ignoring interactions.** Mass affects power affects cost. Evaluate at the system level, not just subsystem.

4. **Optimizing past the point of value.** Saving $1k on a $300k satellite doesn't matter. Focus on big drivers.

5. **Forgetting schedule.** A perfect design that ships 2 years late loses to a good design that ships now.

6. **Recalculating on every iteration.** Do the trade study once, document it, move on. Don't re-optimize daily.

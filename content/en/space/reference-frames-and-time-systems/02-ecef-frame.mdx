## Why this matters

Ground stations don't move. Cities don't orbit. When you need to point an antenna at a satellite, compute a footprint, or determine if a spacecraft is over a specific country, you need coordinates that rotate with Earth. ECEF (Earth-Centered Earth-Fixed) is that frame — the coordinate system where ground locations have constant coordinates. GPS, Google Maps, and every ground-based radar uses ECEF (specifically WGS84). Understanding ECEF is essential for satellite ground track plotting, coverage analysis, and any operation involving Earth-fixed infrastructure.

### Geodetic to Cartesian conversion

```
Converting Geodetic (lat, lon, alt) to ECEF (X, Y, Z):

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Given: latitude φ, longitude λ, altitude h                        │
│                                                                      │
│  STEP 1: Calculate radius of curvature in prime vertical          │
│                                                                      │
│         N = a / √(1 - e²·sin²φ)                                    │
│                                                                      │
│  STEP 2: Convert to Cartesian                                      │
│                                                                      │
│         X = (N + h) · cos φ · cos λ                                │
│         Y = (N + h) · cos φ · sin λ                                │
│         Z = (N·(1 - e²) + h) · sin φ                               │
│                                                                      │
│  EXAMPLE: San Francisco (37.7749°N, 122.4194°W, 16m)              │
│                                                                      │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  φ = 37.7749° = 0.6592 rad                            │         │
│  │  λ = -122.4194° = -2.1366 rad                         │         │
│  │  h = 16 m                                              │         │
│  │                                                        │         │
│  │  N = 6390825.5 m                                       │         │
│  │                                                        │         │
│  │  X = -2694892.5 m                                      │         │
│  │  Y = -4293467.0 m                                      │         │
│  │  Z =  3854740.1 m                                      │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  REVERSE (ECEF to geodetic) is iterative — use Bowring's method   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### ECI to ECEF transformation

```
Converting Between ECI and ECEF:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  The ONLY difference: rotation about Z-axis by Earth rotation      │
│                                                                      │
│  ECI → ECEF:                                                        │
│                                                                      │
│  ┌                     ┐   ┌                    ┐ ┌      ┐         │
│  │ X_ecef              │   │ cos(θ)  sin(θ)  0 │ │ X_eci│         │
│  │ Y_ecef              │ = │ -sin(θ) cos(θ)  0 │ │ Y_eci│         │
│  │ Z_ecef              │   │ 0       0       1 │ │ Z_eci│         │
│  └                     ┘   └                    ┘ └      ┘         │
│                                                                      │
│  Where θ = GMST (Greenwich Mean Sidereal Time)                     │
│                                                                      │
│  GMST changes by ~360° per sidereal day (23h 56m 4s)              │
│                                                                      │
│  For velocity transformation, also account for ω×r term:           │
│                                                                      │
│  v_ecef = R(θ) · v_eci - ω × r_ecef                               │
│                                                                      │
│  where ω = [0, 0, 7.292115×10⁻⁵] rad/s                            │
│                                                                      │
│  PRECISION NOTES:                                                   │
│  • For &lt;1m accuracy, include polar motion (x_p, y_p)              │
│  • For &lt;1cm accuracy, include Earth orientation parameters (EOP)   │
│  • IERS provides daily EOP bulletins                               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Apply

**Working with ECEF coordinates:**

```python
import numpy as np

# WGS84 ellipsoid parameters
WGS84_A = 6378137.0           # Semi-major axis (m)
WGS84_F = 1 / 298.257223563   # Flattening
WGS84_B = WGS84_A * (1 - WGS84_F)  # Semi-minor axis
WGS84_E2 = 2 * WGS84_F - WGS84_F**2  # Eccentricity squared
OMEGA_EARTH = 7.292115e-5     # Earth rotation rate (rad/s)

def geodetic_to_ecef(lat_deg, lon_deg, alt_m):
    """Convert geodetic (lat, lon, alt) to ECEF (X, Y, Z)."""
    lat = np.radians(lat_deg)
    lon = np.radians(lon_deg)
    
    sin_lat = np.sin(lat)
    cos_lat = np.cos(lat)
    sin_lon = np.sin(lon)
    cos_lon = np.cos(lon)
    
    # Radius of curvature in prime vertical
    N = WGS84_A / np.sqrt(1 - WGS84_E2 * sin_lat**2)
    
    X = (N + alt_m) * cos_lat * cos_lon
    Y = (N + alt_m) * cos_lat * sin_lon
    Z = (N * (1 - WGS84_E2) + alt_m) * sin_lat
    
    return np.array([X, Y, Z])

def ecef_to_geodetic(X, Y, Z, tol=1e-12, max_iter=10):
    """Convert ECEF (X, Y, Z) to geodetic (lat, lon, alt) using Bowring's method."""
    lon = np.arctan2(Y, X)
    
    p = np.sqrt(X**2 + Y**2)
    
    # Initial estimate using Bowring's method
    lat = np.arctan2(Z, p * (1 - WGS84_E2))
    
    for _ in range(max_iter):
        sin_lat = np.sin(lat)
        N = WGS84_A / np.sqrt(1 - WGS84_E2 * sin_lat**2)
        lat_new = np.arctan2(Z + WGS84_E2 * N * sin_lat, p)
        
        if abs(lat_new - lat) < tol:
            break
        lat = lat_new
    
    sin_lat = np.sin(lat)
    cos_lat = np.cos(lat)
    N = WGS84_A / np.sqrt(1 - WGS84_E2 * sin_lat**2)
    
    if abs(cos_lat) > 1e-10:
        alt = p / cos_lat - N
    else:
        alt = abs(Z) - WGS84_B
    
    return np.degrees(lat), np.degrees(lon), alt

def gmst_rad(jd):
    """Calculate Greenwich Mean Sidereal Time (radians)."""
    T = (jd - 2451545.0) / 36525
    gmst_deg = 280.46061837 + 360.98564736629 * (jd - 2451545.0) + \
               0.000387933 * T**2 - T**3 / 38710000
    return np.radians(gmst_deg % 360)

def eci_to_ecef(r_eci, v_eci, jd):
    """Convert ECI state to ECEF state."""
    theta = gmst_rad(jd)
    cos_t = np.cos(theta)
    sin_t = np.sin(theta)
    
    # Rotation matrix
    R = np.array([
        [cos_t, sin_t, 0],
        [-sin_t, cos_t, 0],
        [0, 0, 1]
    ])
    
    r_ecef = R @ r_eci
    
    # Velocity includes Earth rotation effect
    omega = np.array([0, 0, OMEGA_EARTH])
    v_ecef = R @ v_eci - np.cross(omega, r_ecef)
    
    return r_ecef, v_ecef

def satellite_elevation_azimuth(sat_ecef, station_lat, station_lon, station_alt):
    """Calculate satellite elevation and azimuth from ground station."""
    station_ecef = geodetic_to_ecef(station_lat, station_lon, station_alt)
    
    # Vector from station to satellite
    delta = sat_ecef - station_ecef
    
    # Local ENU (East-North-Up) transformation
    lat = np.radians(station_lat)
    lon = np.radians(station_lon)
    
    sin_lat = np.sin(lat)
    cos_lat = np.cos(lat)
    sin_lon = np.sin(lon)
    cos_lon = np.cos(lon)
    
    # ECEF to ENU rotation matrix
    R = np.array([
        [-sin_lon, cos_lon, 0],
        [-sin_lat*cos_lon, -sin_lat*sin_lon, cos_lat],
        [cos_lat*cos_lon, cos_lat*sin_lon, sin_lat]
    ])
    
    enu = R @ delta
    E, N, U = enu
    
    # Elevation and azimuth
    horiz_dist = np.sqrt(E**2 + N**2)
    elevation = np.degrees(np.arctan2(U, horiz_dist))
    azimuth = np.degrees(np.arctan2(E, N)) % 360
    
    return elevation, azimuth

print(f"  Station location: {station['lat']:.4f}°N, {abs(station['lon']):.4f}°W")
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Geodetic vs geocentric latitude.** They differ by up to 0.19° at 45° latitude. GPS uses geodetic.

2. **Altitude reference.** WGS84 altitude is above ellipsoid, not mean sea level (geoid). Difference up to ±100m.

3. **Forgetting Earth rotation for velocity.** ECEF velocity includes Earth rotation. ECI to ECEF velocity transform needs ω×r term.

4. **Mixing datums.** WGS84, NAD83, ITRF differ by cm-m. Mixing them corrupts precision.

5. **Assuming spherical Earth.** Using R_earth = 6371 km instead of WGS84 ellipsoid causes km-level errors at high latitudes.

6. **ECEF for orbit propagation.** Never propagate orbits in ECEF — fictitious forces make equations complex. Use ECI, then transform.

</section>

## Why this matters

Satellite IoT connects sensors in remote areas—ocean buoys, wildlife trackers, agricultural monitors—where terrestrial networks don't reach. The challenge: a battery-powered ground device with milliwatts of power must close a link to a CubeSat 600 km away. This walkthrough shows how L-band and narrow-band waveforms make it possible.

## How it works

### Link budget breakdown

```
IoT Uplink Budget (Ground → CubeSat):

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Parameter                           Value      Unit    Notes       │
│  ═══════════════════════════════════════════════════════════════   │
│                                                                      │
│  TRANSMIT SIDE (IoT Sensor):                                        │
│  ────────────────────────────────────────────────────────────────  │
│  Tx power (100 mW)                   -10.0      dBW    Battery     │
│  Tx antenna gain (whip)               +0.0      dBi    Omni        │
│  Cable/connector loss                 -1.0      dB                 │
│  ────────────────────────────────────────────────────────────────  │
│  EIRP                                -11.0      dBW                │
│                                                                      │
│  PATH:                                                              │
│  ────────────────────────────────────────────────────────────────  │
│  Frequency                           401        MHz    L-band      │
│  Slant range (45° elevation)         800        km                 │
│  Free space loss                    -145.6      dB                 │
│  Atmospheric loss                     -0.5      dB                 │
│  Ionospheric scintillation           -1.0      dB     Margin      │
│  Polarization mismatch               -0.5      dB                 │
│  ────────────────────────────────────────────────────────────────  │
│  Total path loss                    -147.6      dB                 │
│                                                                      │
│  RECEIVE SIDE (CubeSat):                                            │
│  ────────────────────────────────────────────────────────────────  │
│  Rx antenna gain (helix)              +6.0      dBi                │
│  System noise temp                    500        K     High!       │
│  10×log₁₀(T_sys)                      27.0      dB                 │
│  G/T                                 -21.0      dB/K   Poor        │
│  ────────────────────────────────────────────────────────────────  │
│                                                                      │
│  LINK CALCULATION:                                                  │
│  ────────────────────────────────────────────────────────────────  │
│  Boltzmann constant                 +228.6      dBW/K/Hz          │
│  C/N₀ = EIRP - L + G/T - k                                        │
│       = -11.0 - 147.6 + (-21.0) + 228.6                          │
│       = 49.0 dBHz                                                  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Making 49 dBHz work

```
Narrow-band Waveform Strategy:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  C/N₀ = 49 dBHz seems low. Let's check if it closes:               │
│                                                                      │
│  OPTION 1: Standard data link                                       │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  At 9600 bps: BW ≈ 10 kHz = 40 dBHz                        │   │
│  │  C/N = C/N₀ - 10×log₁₀(BW) = 49 - 40 = 9 dB               │   │
│  │  Required for BPSK: ~10 dB at BER 10⁻⁵                     │   │
│  │  Margin: 9 - 10 = -1 dB ❌ DOES NOT CLOSE                  │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  OPTION 2: Narrow-band IoT waveform                                 │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  At 400 bps: BW ≈ 500 Hz = 27 dBHz                         │   │
│  │  C/N = C/N₀ - 10×log₁₀(BW) = 49 - 27 = 22 dB              │   │
│  │  Required for BPSK: ~10 dB                                  │   │
│  │  Margin: 22 - 10 = 12 dB ✓ CLOSES WITH MARGIN              │   │
│  │                                                              │   │
│  │  Message: 24 bytes = 192 bits                               │   │
│  │  Time: 192 / 400 = 0.48 seconds                            │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
│  OPTION 3: Spread spectrum (LoRa-style)                             │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                                                              │   │
│  │  Spreading factor SF=10: 10 dB processing gain             │   │
│  │  Effective C/N₀: 49 + 10 = 59 dBHz                         │   │
│  │  Now 9.6 kbps could work with margin                       │   │
│  │  Trade-off: Uses more spectrum, longer on-air time          │   │
│  │                                                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Apply

**Complete IoT link budget calculation:**

```python
import numpy as np

def iot_cubesat_link_budget():
    """
    L-band IoT uplink budget (sensor → CubeSat).
    """
    # Geometry
    altitude_km = 600
    elevation_deg = 45
    earth_radius_km = 6371
    
    el_rad = np.radians(elevation_deg)
    r_sat = earth_radius_km + altitude_km
    slant_range_km = np.sqrt(
        r_sat**2 - (earth_radius_km * np.cos(el_rad))**2
    ) - earth_radius_km * np.sin(el_rad)
    
    # Frequency
    freq_mhz = 401
    freq_ghz = freq_mhz / 1000
    
    # Free space path loss
    fspl_db = 92.45 + 20 * np.log10(freq_ghz * slant_range_km)
    
    # IoT sensor (transmit)
    p_tx_mw = 100
    p_tx_dbw = 10 * np.log10(p_tx_mw / 1000)  # -10 dBW
    g_tx_dbi = 0.0  # whip antenna
    cable_loss_db = 1.0
    eirp_dbw = p_tx_dbw + g_tx_dbi - cable_loss_db
    
    # Additional losses
    atm_loss_db = 0.5
    iono_loss_db = 1.0
    pol_loss_db = 0.5
    total_add_loss_db = atm_loss_db + iono_loss_db + pol_loss_db
    
    # CubeSat receiver
    g_rx_dbi = 6.0  # deployable helix
    t_sys_k = 500.0  # looking at warm Earth
    g_t_dbk = g_rx_dbi - 10 * np.log10(t_sys_k)
    
    # Link budget
    total_loss_db = fspl_db + total_add_loss_db
    k_dbw = -228.6
    c_n0_dbhz = eirp_dbw - total_loss_db + g_t_dbk - k_dbw
    
    print("L-band IoT Uplink Link Budget")
    print("=" * 60)
    print(f"\n--- GEOMETRY ---")
    print(f"Altitude:        {altitude_km} km")
    print(f"Elevation:       {elevation_deg}°")
    print(f"Slant range:     {slant_range_km:.1f} km")
    
    print(f"\n--- TRANSMIT (IoT Sensor) ---")
    print(f"Tx power:        {p_tx_mw} mW = {p_tx_dbw:.1f} dBW")
    print(f"Antenna gain:    {g_tx_dbi:.1f} dBi (whip)")
    print(f"Cable loss:      -{cable_loss_db} dB")
    print(f"EIRP:            {eirp_dbw:.1f} dBW")
    
    print(f"\n--- PATH ---")
    print(f"Frequency:       {freq_mhz} MHz")
    print(f"Free space loss: {fspl_db:.1f} dB")
    print(f"Additional loss: {total_add_loss_db:.1f} dB")
    print(f"Total loss:      {total_loss_db:.1f} dB")
    
    print(f"\n--- RECEIVE (CubeSat) ---")
    print(f"Antenna gain:    {g_rx_dbi:.1f} dBi")
    print(f"System temp:     {t_sys_k:.0f} K")
    print(f"G/T:             {g_t_dbk:.1f} dB/K")
    
    print(f"\n--- RESULT ---")
    print(f"C/N₀:            {c_n0_dbhz:.1f} dBHz")
    
    # Data rate options
    print(f"\n--- DATA RATE ANALYSIS ---")
    data_rates = [100, 400, 1200, 9600]  # bps
    for rate in data_rates:
        # Assume bandwidth ≈ 1.2 × data rate for BPSK
        bw_hz = 1.2 * rate
        c_n_db = c_n0_dbhz - 10 * np.log10(bw_hz)
        required_cn = 10  # dB for BPSK at BER 10^-5
        margin = c_n_db - required_cn
        status = "✓" if margin > 3 else "⚠" if margin > 0 else "✗"
        print(f"  {rate:5d} bps: C/N = {c_n_db:.1f} dB, "
              f"margin = {margin:.1f} dB {status}")
    
    # Message timing
    print(f"\n--- MESSAGE TIMING ---")
    msg_bytes = 24
    msg_bits = msg_bytes * 8
    best_rate = 400  # bps
    tx_time = msg_bits / best_rate
    print(f"Message size:    {msg_bytes} bytes ({msg_bits} bits)")
    print(f"At {best_rate} bps:      {tx_time:.2f} seconds on-air")
    
    # Contact window
    pass_time_min = 10  # typical LEO pass
    messages_per_pass = int(pass_time_min * 60 / (tx_time + 1))  # +1s guard
    print(f"Pass duration:   ~{pass_time_min} min")
    print(f"Messages/pass:   ~{messages_per_pass} (with guard time)")
    
    return c_n0_dbhz

c_n0 = iot_cubesat_link_budget()

# Power vs range trade-off
print("\n" + "=" * 60)
print("POWER vs RANGE TRADE-OFF")
print("-" * 60)
powers_mw = [50, 100, 250, 500, 1000]
for p in powers_mw:
    eirp = 10 * np.log10(p/1000) + 0 - 1  # dBW
    c_n0_var = eirp - 147.6 + (-21) + 228.6
    margin_400bps = c_n0_var - 27 - 10  # 27 dBHz for 400bps BW
    print(f"  {p:4d} mW: EIRP = {eirp:.1f} dBW, "
          f"C/N₀ = {c_n0_var:.1f} dBHz, "
          f"margin@400bps = {margin_400bps:.1f} dB")
```

<section class="lesson-pitfalls">

## Pitfalls

1. **CubeSat G/T surprise.** -20 dB/K is common—don't assume ground-station performance.

2. **Doppler at LEO.** ±10 kHz at L-band from CubeSat motion; receiver must track.

3. **Short contact windows.** ~10 min pass, not always overhead—plan message timing.

4. **Ionospheric effects.** Faraday rotation, scintillation—worse at low frequencies.

5. **Spectrum constraints.** Only narrow allocations available for sat-IoT; coordinate.

6. **Battery life.** 100 mW × 1 sec = 100 mJ per message; multiply by messages/day.

7. **Collision probability.** Many sensors, one satellite—MAC protocol essential.

</section>

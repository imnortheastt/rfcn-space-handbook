## Why this matters

"Why can't my phone just connect to Starlink?" is a question every engineer hears. The answer involves physics that no amount of software can overcome: smartphones have tiny antennas, limited transmit power, and can't point at the sky. Connecting them to satellites 600 km away requires closing a link budget that's 20+ dB worse than terrestrial cellular. This lesson explains why direct-to-device satellite is fundamentally hard, and how services like T-Mobile/Starlink and AST SpaceMobile approach the problem differently.

### The fundamental asymmetry

```
Direct-to-Device Link Budget Challenge:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  DOWNLINK (Satellite → Phone): Relatively Easy                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Satellite EIRP: 50-60 dBW (with large antenna array)         │ │
│  │  Phone sensitivity: -110 dBm (typical LTE/NR receiver)        │ │
│  │  Free space path loss: ~160 dB @ 2 GHz, 600 km                │ │
│  │                                                                 │ │
│  │  Link margin: 50 dBW + 30 dBm - 160 dB - (-110 dBm)          │ │
│  │             = 80 - 160 + 110 = +30 dB margin ✓                 │ │
│  │                                                                 │ │
│  │  Result: Downlink works with moderate satellite power          │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  UPLINK (Phone → Satellite): The Hard Part                         │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Phone EIRP: 23 dBm (0.2 W, regulatory limit)                 │ │
│  │  Phone antenna gain: ~0 dBi (omni, held any direction)        │ │
│  │  Satellite G/T: 10-15 dB/K (best case)                        │ │
│  │  Free space path loss: ~160 dB                                 │ │
│  │                                                                 │ │
│  │  Link margin: 23 dBm + 10 dB - 160 dB - noise                 │ │
│  │             = Very tight, ~0 dB margin ⚠️                      │ │
│  │                                                                 │ │
│  │  Result: UL is THE limiting factor                             │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  KEY INSIGHT: Every direct-to-device service is constrained by    │
│  uplink. The satellite must have extremely high receive gain,      │
│  or the UE must transmit longer/slower to compensate.              │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## How it works

### Two approaches to direct-to-device

```
Industry Approaches:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  APPROACH 1: Small satellite, narrow service (T-Mobile/Starlink)   │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Strategy: Accept UL limitation, offer text-only service       │ │
│  │                                                                 │ │
│  │  Satellite: Standard Starlink v2 with ~5m antenna array        │ │
│  │  Service: SMS/iMessage, emergency SOS                          │ │
```

### Link budget closure techniques

```
Closing the UL Budget:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  TECHNIQUE 1: Satellite antenna gain                               │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Standard satellite: 2-3m antenna → ~30 dBi                   │ │
│  │  Direct-to-device: 10-64 m² array → 45-50 dBi                 │ │
│  │                                                                 │ │
│  │  +15-20 dB gain improvement                                    │ │
│  │                                                                 │ │
│  │  Trade-off: Satellite cost, launch mass, deployment risk       │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  TECHNIQUE 2: Bandwidth reduction                                  │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Standard cellular: 10-20 MHz channels                         │ │
│  │  D2D satellite: 180 kHz - 1.4 MHz (NB-IoT/LTE-M widths)       │ │
│  │                                                                 │ │
│  │  Narrower BW → lower noise floor → better SNR                 │ │
│  │  10 MHz → 180 kHz = 17 dB noise reduction                     │ │
│  │                                                                 │ │
│  │  Trade-off: Lower throughput                                   │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  TECHNIQUE 3: Time spreading (repetition)                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Transmit same data N times → N× energy received              │ │
│  │  Processing gain: 10 × log₁₀(N) dB                            │ │
│  │                                                                 │ │
│  │  128× repetition = +21 dB gain                                 │ │
│  │  2048× repetition = +33 dB gain                                │ │
│  │                                                                 │ │
│  │  Trade-off: Throughput drops proportionally                    │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  TECHNIQUE 4: Lower orbit                                          │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Path loss ∝ distance²                                        │ │
│  │  600 km vs 36,000 km (GEO) = 36 dB difference                 │ │
│  │                                                                 │ │
│  │  All D2D services use LEO (300-600 km)                        │ │
│  │                                                                 │ │
│  │  Trade-off: More satellites needed, faster handovers           │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Apply

**Direct-to-device link budget calculator:**

```python
import math

def d2d_uplink_budget(freq_ghz, sat_alt_km, elevation_deg,
                       ue_power_dbm, ue_antenna_dbi,
                       sat_antenna_area_m2, sat_noise_temp_k,
                       bandwidth_hz, repetitions):
    """
    Calculate direct-to-device uplink budget.
    """
    earth_radius = 6371
    el_rad = math.radians(elevation_deg)
    
    slant_range = (-earth_radius * math.sin(el_rad) + 
                   math.sqrt((earth_radius * math.sin(el_rad))**2 + 
                            2 * earth_radius * sat_alt_km + sat_alt_km**2))
    
    c = 3e8
    wavelength = c / (freq_ghz * 1e9)
    
    fspl = 20 * math.log10(4 * math.pi * slant_range * 1000 / wavelength)
    
    ae = sat_antenna_area_m2 * 0.6
    sat_antenna_gain = 10 * math.log10(4 * math.pi * ae / wavelength**2)
    
    sat_gt = sat_antenna_gain - 10 * math.log10(sat_noise_temp_k)
    
    ue_eirp = ue_power_dbm + ue_antenna_dbi
    
    body_loss = 4
    polarization_loss = 3
    atmospheric_loss = 0.5
    total_loss = fspl + body_loss + polarization_loss + atmospheric_loss
    
    k_boltz = -228.6
    noise_density = k_boltz + 10 * math.log10(sat_noise_temp_k)
    
    cn0 = ue_eirp + sat_gt - total_loss - noise_density
    
    noise_bw = 10 * math.log10(bandwidth_hz)
    snr_no_rep = cn0 - noise_bw
    
    rep_gain = 10 * math.log10(repetitions)
    snr_with_rep = snr_no_rep + rep_gain
    
    required_snr = 0
    margin = snr_with_rep - required_snr
    
    return {
        "slant_range_km": slant_range,
        "fspl_db": fspl,
        "sat_antenna_gain_dbi": sat_antenna_gain,
        "sat_gt_db_k": sat_gt,
        "ue_eirp_dbm": ue_eirp,
        "total_path_loss_db": total_loss,
        "cn0_dbhz": cn0,
        "snr_no_repetition_db": snr_no_rep,
        "repetition_gain_db": rep_gain,
        "snr_with_repetition_db": snr_with_rep,
        "link_margin_db": margin,
        "link_closes": margin > 0
    }

def compare_d2d_approaches():
    """
    Compare different D2D satellite approaches.
    """
    scenarios = [
        {
            "name": "T-Mobile/Starlink (text)",
            "sat_area_m2": 25,
            "bandwidth_hz": 180000,
            "repetitions": 128,
            "service": "SMS only"
        },
        {
            "name": "AST SpaceMobile (voice/data)",
            "sat_area_m2": 64,
            "bandwidth_hz": 5000000,
            "repetitions": 1,
            "service": "Voice + data"
        },
        {
            "name": "Apple Emergency SOS",
            "sat_area_m2": 10,
            "bandwidth_hz": 50000,
            "repetitions": 512,
            "service": "Emergency only"
        }
    ]
    
    results = []
    for s in scenarios:
        lb = d2d_uplink_budget(
            freq_ghz=2.0,
            sat_alt_km=500,
            elevation_deg=30,
            ue_power_dbm=23,
            ue_antenna_dbi=0,
            sat_antenna_area_m2=s["sat_area_m2"],
            sat_noise_temp_k=300,
            bandwidth_hz=s["bandwidth_hz"],
            repetitions=s["repetitions"]
        )
        
        effective_rate = s["bandwidth_hz"] / s["repetitions"]
        
        results.append({
            "name": s["name"],
            "service": s["service"],
            "sat_gain_dbi": lb["sat_antenna_gain_dbi"],
            "margin_db": lb["link_margin_db"],
            "closes": lb["link_closes"],
            "effective_rate_kbps": effective_rate / 1000
        })
    
    return results

def throughput_vs_coverage(bandwidth_hz, snr_db, repetitions):
    """
    Calculate achievable throughput given coverage constraints.
    """
    snr_linear = 10 ** (snr_db / 10)
    
    capacity_bps = bandwidth_hz * math.log2(1 + snr_linear)
    
    effective_capacity = capacity_bps / repetitions
    
    overhead = 0.7
    throughput = effective_capacity * overhead
    
    return {
        "shannon_capacity_bps": capacity_bps,
        "effective_capacity_bps": effective_capacity,
        "throughput_bps": throughput,
        "throughput_kbps": throughput / 1000
    }
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Assuming DL = UL.** Downlink budget is 30+ dB better. Services limited by UL, not DL.

2. **Ignoring body loss.** Phone held against body loses 3-6 dB. Real-world is worse than spec.

3. **Forgetting about indoor.** D2D satellites require line-of-sight. No signal through buildings.

4. **Over-promising throughput.** "5G from space" sounds great. Reality: text messages for most users.

5. **Ignoring spectrum reality.** D2D needs MNO partnership for spectrum. Pure satellite operators can't do it alone.

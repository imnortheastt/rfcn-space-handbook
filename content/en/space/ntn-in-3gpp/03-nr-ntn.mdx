## Why this matters

NR-NTN is where satellite-based 5G became real. Release 17 (2022) took the studies (TR 38.811, TR 38.821) and converted them into normative specifications. Now chipset vendors like Qualcomm, MediaTek, and Skylo can build NR-NTN modems that work with any compliant satellite network. This is the standard behind T-Mobile/Starlink direct-to-cell, AST SpaceMobile, and next-generation maritime/aviation broadband.

## How it works

### SIB19 structure

```
System Information Block 19 (NTN-specific):

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  SIB19 Contents (broadcast by gNB):                                │
│                                                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  ntn-Config:                                                   │ │
│  │    ├── epochTime              (reference time)                 │ │
│  │    ├── ta-Info                                                │ │
│  │    │   ├── ta-Common          (delay to reference point)      │ │
│  │    │   └── ta-CommonDrift     (rate of change)                │ │
│  │    │                                                           │ │
│  │    ├── ephemerisInfo          (satellite position)            │ │
│  │    │   ├── positionX/Y/Z      (ECEF coordinates, m)           │ │
│  │    │   └── velocityX/Y/Z      (ECEF velocity, m/s)            │ │
│  │    │   OR                                                      │ │
│  │    │   └── orbitalParameters  (Keplerian elements)            │ │
│  │    │                                                           │ │

│                                                                      │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  1. UE acquires SIB19 on DL                                    │ │
│  │  2. UE extracts satellite position from ephemerisInfo          │ │
│  │  3. UE reads its own position from GNSS                        │ │
│  │  4. UE computes:                                               │ │
│  │     - Slant range → TA-Offset                                 │ │
│  │     - Relative velocity → Doppler compensation                │ │
│  │  5. UE applies compensations before UL transmission            │ │
│  │                                                                 │ │

│                                                                      │
```

### Timing advance mechanism

```
NR-NTN Timing Advance (Normative):

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  FORMULA:                                                           │
│                                                                      │
│  N_TA = N_TA,common + N_TA,offset + N_TA,UE                        │
│                                                                      │
│  Where:                                                             │
│    N_TA,common = ta-Common from SIB19 (in units of T_c × 64)       │
│    N_TA,offset = UE-computed from GNSS + ephemeris                 │
│    N_TA,UE = standard NR TA adjustment from MAC-CE                 │
│                                                                      │
│  TIMING GRANULARITY:                                               │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  T_c = 1 / (480,000 × 4096) = 0.509 ns                        │ │
│  │  T_c × 64 = 32.55 ns                                          │ │
│  │  N_TA,common range: 0 to 2^20 - 1 (up to 34 ms)               │ │
│  │                                                                 │ │
│  │  Compare to terrestrial NR:                                    │ │
│  │    N_TA range: 0 to 3846 × 16 (up to ~2 ms)                   │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  UE TA COMPUTATION:                                                 │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  1. Get satellite ECEF position [x_s, y_s, z_s] from SIB19     │ │
│  │  2. Get UE ECEF position [x_u, y_u, z_u] from GNSS             │ │
│  │  3. Get reference ECEF position [x_r, y_r, z_r] from SIB19     │ │
│  │                                                                 │ │
│  │  4. Compute slant ranges:                                      │ │
│  │     d_UE = sqrt((x_s-x_u)² + (y_s-y_u)² + (z_s-z_u)²)         │ │
│  │     d_ref = sqrt((x_s-x_r)² + (y_s-y_r)² + (z_s-z_r)²)        │ │
│  │                                                                 │ │
│  │  5. TA_offset = 2 × (d_UE - d_ref) / c                        │ │
│  │     (factor of 2 for round-trip in transparent mode)           │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### HARQ operation

```
NR-NTN HARQ Configurations:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  HARQ ENABLED (LEO, short RTT):                                    │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Configuration:                                                 │ │
│  │    • nrofHARQ-ProcessesForPDSCH: up to 32                      │ │
│  │    • K1 (DL-to-ACK timing): extended to cover RTT              │ │

│  │    K1_min = ceil(RTT / slot_duration) + processing_time       │ │
│  │                                                                 │ │
│  │  Example (LEO-600, 30° elevation, 30 kHz SCS):                │ │
│  │    RTT = 13 ms                                                 │ │
│  │    Slot = 0.5 ms                                               │ │
│  │    K1_min = ceil(13 / 0.5) + 4 = 30 slots                     │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  HARQ DISABLED (GEO, long RTT):                                    │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │

│  │    • harq-FeedbackDisabled = true                              │ │
│  │    • Higher FEC coding (more redundancy)                       │ │
│  │    • RLC AM for reliable delivery                              │ │
│  │                                                                 │ │
│  │  Rationale:                                                     │ │
│  │    GEO RTT ≈ 540 ms                                            │ │
│  │    HARQ retransmission adds another 540 ms                     │ │
│  │    Better to rely on RLC retransmission                        │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

```python
import math

def nr_ntn_link_budget(freq_ghz, sat_alt_km, elevation_deg, 
                       ue_eirp_dbm, sat_gt_db, bandwidth_mhz):
    """
    Calculate NR-NTN link budget for uplink.
    """
    earth_radius = 6371
    el_rad = math.radians(elevation_deg)
    
    slant_range = (-earth_radius * math.sin(el_rad) + 
                   math.sqrt((earth_radius * math.sin(el_rad))**2 + 
                            2 * earth_radius * sat_alt_km + sat_alt_km**2))
    
    fspl = 32.45 + 20 * math.log10(freq_ghz * 1000) + 20 * math.log10(slant_range)
    
    atmospheric_loss = 0.5 if freq_ghz < 10 else 2.0
    
    received_power = ue_eirp_dbm - fspl - atmospheric_loss + sat_gt_db
    
    noise_figure = 3.0
    k_boltz = -228.6
    thermal_noise = k_boltz + 10 * math.log10(bandwidth_mhz * 1e6) + noise_figure
    
    snr = received_power - thermal_noise
    
    return {
        "slant_range_km": slant_range,
        "fspl_db": fspl,
        "atmospheric_loss_db": atmospheric_loss,
        "received_power_dbm": received_power,
        "thermal_noise_dbm": thermal_noise,
        "snr_db": snr
    }

def compute_ta_from_ephemeris(ue_ecef, sat_ecef, ref_ecef):
    """
    Compute TA-Offset from ECEF positions per NR-NTN spec.
    """
    c = 299792.458
    
    def distance(a, b):
        return math.sqrt(sum((a[i] - b[i])**2 for i in range(3)))
    
    d_ue = distance(sat_ecef, ue_ecef)
    d_ref = distance(sat_ecef, ref_ecef)
    
    ta_offset_ms = 2 * (d_ue - d_ref) / c * 1000
    
    T_c = 1 / (480000 * 4096)
    N_ta_offset = int(ta_offset_ms / 1000 / (T_c * 64))
    
    return {
        "d_ue_km": d_ue,
        "d_ref_km": d_ref,
        "ta_offset_ms": ta_offset_ms,
        "N_ta_offset": N_ta_offset
    }

def harq_configuration(rtt_ms, scs_khz, harq_enabled=True):
    """
    Determine HARQ configuration for NR-NTN.
    """
    slot_ms = 1 / (scs_khz / 15)
    
    processing_slots = 4
    
    if harq_enabled:
        k1_min = math.ceil(rtt_ms / slot_ms) + processing_slots
        
        processes_needed = math.ceil(rtt_ms / slot_ms) + 1
        processes = min(32, max(16, processes_needed))
        
        return {
            "harq_enabled": True,
            "k1_min_slots": k1_min,
            "k1_min_ms": k1_min * slot_ms,
            "processes_needed": processes_needed,
            "processes_configured": processes
        }
    else:
        return {
            "harq_enabled": False,
            "k1_min_slots": None,
            "processes_configured": 0,
            "recommendation": "Use RLC AM with aggressive FEC"
        }

def nr_ntn_throughput_estimate(snr_db, bandwidth_mhz, harq_enabled):
    """
    Estimate NR-NTN throughput based on SNR.
    """
    if snr_db < -3:
        se = 0.15
        modulation = "QPSK 1/8"
    elif snr_db < 0:
        se = 0.4
        modulation = "QPSK 1/3"
    elif snr_db < 3:
        se = 0.8
        modulation = "QPSK 1/2"
    elif snr_db < 6:
        se = 1.2
        modulation = "QPSK 3/4"
    elif snr_db < 10:
        se = 2.0
        modulation = "16QAM 1/2"
    elif snr_db < 15:
        se = 3.0
        modulation = "16QAM 3/4"
    else:
        se = 4.5
        modulation = "64QAM 3/4"
    
    overhead = 0.75 if harq_enabled else 0.70
    
    throughput_mbps = bandwidth_mhz * se * overhead
    
    return {
        "snr_db": snr_db,
        "spectral_efficiency": se,
        "modulation": modulation,
        "throughput_mbps": throughput_mbps
    }
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Forgetting feeder link for transparent.** The UE-gateway path has TWO space hops. Link budget must include both.

2. **Using wrong TA granularity.** NR-NTN uses T_c × 64 units for N_TA,common, not the standard NR T_c × 16.

3. **Ignoring band constraints.** S-band (n256) has only 30 MHz total. Don't plan for 100 MHz carriers.

4. **Assuming HARQ always works.** GEO deployments typically disable HARQ. Plan for RLC-layer retransmissions.

5. **Underestimating UL limitations.** Smartphone EIRP (23 dBm) + omni antenna → UL is the bottleneck. DL can be 10× faster.

</section>

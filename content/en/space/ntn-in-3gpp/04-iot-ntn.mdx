## Why this matters

There are billions of IoT devices that need connectivity outside cellular coverage: shipping containers in mid-ocean, cattle trackers in remote ranches, pipeline sensors in deserts. NR-NTN targets smartphones, but IoT-NTN targets the long tail — devices that transmit kilobytes per day, sleep for hours, and run on a single battery for a decade. Release 17 adapted NB-IoT and LTE-M for satellite, enabling truly global IoT coverage with minimal changes to existing chipsets.

## How it works

### NTN adaptations for NB-IoT

```
Key Protocol Changes for IoT-NTN:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  1. TIMING ADVANCE (Same as NR-NTN):                               │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  N_TA = N_TA,common + N_TA,offset + N_TA,UE                   │ │
│  │                                                                 │ │
│  │  • TA-Common broadcast in SIB-NTN                              │ │
│  │  • UE computes offset from GNSS + ephemeris                    │ │
│  │  • Extended range: up to 40 ms (GEO support)                   │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  2. GNSS INTEGRATION:                                              │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Mandatory for IoT-NTN UEs                                     │ │
│  │  But: GNSS acquisition is power-expensive                      │ │
│  │                                                                 │ │
│  │  Optimization:                                                  │ │
│  │  • Assisted GNSS from network (SUPL over NTN)                 │ │
│  │  • Last-known position + age validity                          │ │
│  │  • Position accuracy: ±5 km sufficient for TA                  │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  3. HARQ OPERATION:                                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  NB-IoT: HARQ typically disabled (as in terrestrial extreme)  │ │
│  │  LTE-M: Extended HARQ RTT timer                                │ │
│  │                                                                 │ │
│  │  Rationale: NB-IoT already relies on repetitions               │ │
│  │  No efficiency gain from HARQ at extreme coverage              │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  4. DISCONTINUOUS RECEPTION (eDRX):                                │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Terrestrial eDRX max: ~43 minutes                             │ │
│  │  NTN eDRX max: Extended to hours                               │ │
│  │                                                                 │ │
│  │  Why longer? Satellite passes over sleeping UE                 │ │
│  │  UE needs to align wake-up with satellite visibility           │ │
│  │                                                                 │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

```
IoT-NTN Power Optimization:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  POWER SAVING MODE (PSM):                                          │
│                                                                      │
│     Active           TAU Timer          PSM Sleep                   │

│  NTN ENHANCEMENT:                                                   │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Problem: Satellite not always visible                         │ │
│  │                                                                 │ │
│  │  Solution: Scheduled wake-up aligned with satellite pass       │ │
│  │                                                                 │ │
│  │    1. UE knows satellite orbit from ephemeris                  │ │
│  │    2. UE computes next visibility window                       │ │
│  │    3. UE sleeps until window approaches                        │ │

│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                 │ │
│  │  Standard eDRX: UE wakes at fixed intervals                    │ │
│  │  NTN eDRX: Intervals can extend to satellite pass period       │ │
│  │                                                                 │ │
│  │  LEO period: ~90 minutes orbit, ~10 min visibility window     │ │
│  │  eDRX could align to 90-minute cycle                           │ │
│  │                                                                 │ │
```

**IoT-NTN link budget and battery life calculator:**

```python
import math

def iot_ntn_link_budget(freq_ghz, sat_alt_km, elevation_deg,
                         ue_power_dbm, ue_antenna_dbi, sat_gt_db,
                         bandwidth_khz, repetitions):
    """
    Calculate IoT-NTN link budget with repetition gain.
    """
    earth_radius = 6371
    el_rad = math.radians(elevation_deg)
    
    slant_range = (-earth_radius * math.sin(el_rad) + 
                   math.sqrt((earth_radius * math.sin(el_rad))**2 + 
                            2 * earth_radius * sat_alt_km + sat_alt_km**2))
    
    fspl = 32.45 + 20 * math.log10(freq_ghz * 1000) + 20 * math.log10(slant_range)
    
    ue_eirp = ue_power_dbm + ue_antenna_dbi
    
    repetition_gain = 10 * math.log10(repetitions)
    
    received_power = ue_eirp - fspl + sat_gt_db + repetition_gain
    
    noise_figure = 3.0
    k_boltz = -228.6
    thermal_noise = k_boltz + 10 * math.log10(bandwidth_khz * 1000) + noise_figure
    
    snr = received_power - thermal_noise
    
    mcl = fspl - ue_antenna_dbi - sat_gt_db - repetition_gain
    
    return {
        "slant_range_km": slant_range,
        "fspl_db": fspl,
        "ue_eirp_dbm": ue_eirp,
        "repetition_gain_db": repetition_gain,
        "received_power_dbm": received_power,
        "snr_db": snr,
        "mcl_db": mcl
    }

def nb_iot_throughput(coverage_class, repetitions):
    """
    Estimate NB-IoT throughput based on coverage class.
    """
    base_rate_bps = 160000
    
    effective_rate = base_rate_bps / repetitions
    
    if coverage_class == "normal":
        overhead = 0.7
    elif coverage_class == "robust":
        overhead = 0.5
    elif coverage_class == "extreme":
        overhead = 0.3
    else:
        overhead = 0.2
    
    throughput_bps = effective_rate * overhead
    
    return {
        "coverage_class": coverage_class,
        "repetitions": repetitions,
        "base_rate_bps": base_rate_bps,
        "effective_rate_bps": effective_rate,
        "throughput_bps": throughput_bps,
        "throughput_kbps": throughput_bps / 1000
    }

def battery_life_estimate(battery_mah, tx_ma, rx_ma, sleep_ua,
                           tx_time_s_per_day, rx_time_s_per_day):
    """
    Estimate IoT device battery life.
    """
    tx_hours_per_day = tx_time_s_per_day / 3600
    rx_hours_per_day = rx_time_s_per_day / 3600
    sleep_hours_per_day = 24 - tx_hours_per_day - rx_hours_per_day
    
    daily_consumption_mah = (
        tx_ma * tx_hours_per_day +
        rx_ma * rx_hours_per_day +
        (sleep_ua / 1000) * sleep_hours_per_day
    )
    
    battery_days = battery_mah / daily_consumption_mah
    battery_years = battery_days / 365
    
    return {
        "tx_hours_per_day": tx_hours_per_day,
        "rx_hours_per_day": rx_hours_per_day,
        "daily_consumption_mah": daily_consumption_mah,
        "battery_days": battery_days,
        "battery_years": battery_years
    }

def satellite_visibility(orbit_km, min_elevation_deg, latitude_deg):
    """
    Estimate LEO satellite visibility for IoT scheduling.
    """
    earth_radius = 6371
    orbit_radius = earth_radius + orbit_km
    
    orbital_period_min = 2 * math.pi * math.sqrt(orbit_radius**3 / 398600.4) / 60
    
    el_rad = math.radians(min_elevation_deg)
    
    earth_angle = math.acos(earth_radius * math.cos(el_rad) / orbit_radius) - el_rad
    
    visibility_fraction = earth_angle / math.pi
    
    visibility_min = orbital_period_min * visibility_fraction
    
    passes_per_day = 1440 / orbital_period_min * 0.7
    
    return {
        "orbital_period_min": orbital_period_min,
        "visibility_per_pass_min": visibility_min,
        "passes_per_day": passes_per_day,
        "total_visibility_min_per_day": visibility_min * passes_per_day
    }
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Ignoring GNSS power.** GNSS acquisition consumes 20-50 mA for 30+ seconds. Budget for it in battery calculations.

2. **Forgetting satellite visibility.** IoT device can't transmit when no satellite is overhead. Schedule transmissions for visibility windows.

3. **Over-optimizing for throughput.** IoT-NTN is about coverage, not speed. 100 bytes/hour is fine for most sensors.

4. **Underestimating repetition impact.** 2048× repetitions means a 100-byte message takes ~10 seconds of airtime. Plan accordingly.

5. **Mixing terrestrial and NTN assumptions.** Terrestrial NB-IoT has 144 dB MCL. NTN needs 170+ dB. Design for the harder case.

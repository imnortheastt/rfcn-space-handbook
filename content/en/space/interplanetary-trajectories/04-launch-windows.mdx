## Why this matters

Mars missions launch roughly every 26 months. Jupiter opportunities come every 13 months. Miss your window, and you wait years for the next one—or burn vastly more fuel. Understanding synodic periods and planetary geometry is essential for mission planning, budget allocation, and knowing why some missions get delayed for years.

## Apply

**Synodic period calculations:**

```python
import numpy as np

def synodic_period(t_earth_years, t_planet_years):
    """
    Calculate synodic period between Earth and another planet.
    
    Returns: synodic period in Earth years
    """
    if t_planet_years > t_earth_years:
        # Outer planet
        return 1 / (1/t_earth_years - 1/t_planet_years)
    else:
        # Inner planet
        return 1 / (1/t_planet_years - 1/t_earth_years)

def days_to_months(days):
    return days / 30.44

# Orbital periods (years)
ORBITAL_PERIODS = {
    'Mercury': 0.241,
    'Venus': 0.615,
    'Earth': 1.0,
    'Mars': 1.881,
    'Jupiter': 11.86,
    'Saturn': 29.46,
    'Uranus': 84.01,
    'Neptune': 164.8,
}

print("Synodic Periods (Launch Window Frequency):")
print("=" * 55)

for planet, t_planet in ORBITAL_PERIODS.items():
    if planet == 'Earth':
        continue
    
    t_syn = synodic_period(1.0, t_planet)
    days = t_syn * 365.25
    months = days_to_months(days)
    
    print(f"{planet:10} Orbital: {t_planet*365.25:7.0f} days  "
          f"Synodic: {days:5.0f} days ({months:.1f} months)")

# Example: Next Mars launch windows
print("\n" + "=" * 55)
print("Mars Launch Windows (approximate):")
print("-" * 55)

# Mars synodic period ~780 days = 26 months
base_year = 2024
mars_synodic_months = 26

for i in range(5):
    year = base_year + (i * mars_synodic_months) // 12
    month = (7 + i * mars_synodic_months) % 12 + 1  # Starting July 2024
    print(f"  Window {i+1}: ~{year} (approximately)")

# Phase angle for Hohmann transfer
def hohmann_phase_angle(r1_au, r2_au):
    """
    Calculate required phase angle for Hohmann transfer.
    
    Returns: phase angle in degrees
    """
    # Transfer time
    a_transfer = (r1_au + r2_au) / 2
    t_transfer = np.pi * np.sqrt(a_transfer**3)  # in units where T_Earth = 2π
    
    # Planet moves during transfer
    omega_planet = 2 * np.pi / np.sqrt(r2_au**3)  # angular velocity
    planet_travel = omega_planet * t_transfer  # radians
    
    # Phase angle: planet must be this far ahead
    phase = np.pi - planet_travel  # want planet at 180° when we arrive
    return np.degrees(phase)

print("\n" + "=" * 55)
print("Hohmann Transfer Phase Angles:")
print("-" * 55)

destinations = [
    ('Mars', 1.524),
    ('Venus', 0.723),
    ('Jupiter', 5.203),
]

for name, r_au in destinations:
    phase = hohmann_phase_angle(1.0, r_au)
    print(f"  {name:10} Phase angle: {phase:+.1f}° "
          f"({'ahead' if phase > 0 else 'behind'})")
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Confusing opposition with launch.** Opposition is closest approach, not optimal launch time.

2. **Ignoring window width.** Early/late in window requires more fuel; plan for margin.

3. **Daily vs period.** Synodic period sets which months; daily rotation sets which hour.

4. **Non-Hohmann trajectories.** Faster trajectories have different phase angles, different windows.

5. **Gravity assist complexity.** Multi-flyby trajectories have much more constrained windows.

6. **Launch site limitations.** Some launch sites can't reach all orbital inclinations.

7. **Weather and delays.** Real launches slip; need multiple attempts within window.

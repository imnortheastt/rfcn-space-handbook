## Why this matters

MODCOD selection is the single most important decision in satellite link design. Choose wrong, and you either waste half your transponder capacity (too conservative) or suffer constant outages (too aggressive). DVB-S2 offers 28 MODCODs; DVB-S2X adds 88 more. Each combination of modulation and code rate has a specific SNR threshold and spectral efficiency. This lesson gives you the complete lookup tables and the intuition to pick the right MODCOD for any link.

## Intuition

### The MODCOD trade-off

```
MODCOD Selection Trade-off:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  Spectral                                                           │
│  Efficiency                                                         │
│  (bps/Hz)    HIGH EFFICIENCY                                        │
│     6 ─      ┌────────────────────────────────┐                     │
│              │ 256APSK 3/4 (S2X)              │ ← Needs ~18 dB     │
│     5 ─      │                                │                     │
│              │ 32APSK 8/9                     │ ← Needs ~14 dB     │
│     4 ─      │                                │                     │
│              │ 16APSK 3/4                     │ ← Needs ~10 dB     │
│     3 ─      │                                │                     │
│              │ 8PSK 2/3                       │ ← Needs ~7 dB      │
│     2 ─      │                                │                     │
│              │ QPSK 3/4                       │ ← Needs ~4 dB      │
│     1 ─      │                                │                     │
│              │ QPSK 1/4                       │ ← Works at -2 dB!  │
│     0 ─      └────────────────────────────────┘                     │
│              LOW SNR REQUIRED    HIGH SNR REQUIRED                  │
│                                                                      │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  TRADE-OFF:                                           │         │
│  │                                                        │         │
│  │  Higher modulation (8PSK→16APSK→32APSK):             │         │
│  │    + More bits per symbol                            │         │
│  │    - Symbols closer together → needs higher SNR      │         │
│  │                                                        │         │
│  │  Higher code rate (1/2→3/4→9/10):                    │         │
│  │    + Less redundancy → more useful data              │         │
│  │    - Less error protection → needs higher SNR        │         │
│  │                                                        │         │
│  │  RULE: Pick the highest efficiency your SNR supports │         │
│  │        with margin for fading                        │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### MODCOD naming convention

```
Understanding MODCOD Names:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  MODCOD = MODulation + CODing rate                                  │
│                                                                      │
│  Example: "16APSK 3/4"                                              │
│            │      │                                                  │
│            │      └── Coding rate: 3/4 of bits are useful data     │
│            │          (1/4 are parity for error correction)         │
│            │                                                         │
│            └── Modulation: 16-point Amplitude-Phase Shift Keying   │
│                (4 bits per symbol)                                   │
│                                                                      │
│  BITS PER SYMBOL BY MODULATION:                                     │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Modulation │ Bits/Symbol │ Constellation Points     │         │
│  │  ───────────┼─────────────┼─────────────────────      │         │
│  │  QPSK       │ 2           │ 4                        │         │
│  │  8PSK       │ 3           │ 8                        │         │
│  │  16APSK     │ 4           │ 16                       │         │
│  │  32APSK     │ 5           │ 32                       │         │
│  │  64APSK     │ 6           │ 64   (S2X only)         │         │
│  │  128APSK    │ 7           │ 128  (S2X only)         │         │
│  │  256APSK    │ 8           │ 256  (S2X only)         │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  SPECTRAL EFFICIENCY FORMULA:                                       │
│                                                                      │
│  η = (bits per symbol) × (code rate)                               │
│                                                                      │
│  Example: 16APSK 3/4                                                │
│  η = 4 × (3/4) = 3.0 bps/Hz (theoretical)                          │
│  Actual: 2.97 bps/Hz (accounting for overhead)                      │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### MODCOD selection flow

```
How to Select a MODCOD:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  STEP 1: Calculate available C/N (or Es/N0)                        │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  From link budget:                                    │         │
│  │  C/N = EIRP - FSPL - Losses + G/T - k - B            │         │
│  │                                                        │         │
│  │  Convert C/N to Es/N0:                               │         │
│  │  Es/N0 = C/N + 10·log₁₀(B/Rs) [dB]                   │         │
│  │  (B = noise bandwidth, Rs = symbol rate)              │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                         │                                            │
│                         ▼                                            │
│  STEP 2: Account for required margin                               │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Implementation loss:    ~0.5-1.0 dB                 │         │
│  │  ACM hysteresis margin:  ~0.5-1.0 dB                 │         │
│  │  Rain fade margin:       (depends on availability)   │         │
│  │                                                        │         │
│  │  Available Es/N0 for MODCOD selection:               │         │
│  │  = Calculated Es/N0 - Total margin                   │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                         │                                            │
│                         ▼                                            │
│  STEP 3: Select highest efficiency MODCOD                          │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Find MODCOD where:                                   │         │
│  │  Threshold Es/N0 < Available Es/N0                   │         │
│  │                                                        │         │
│  │  Among those that work, pick highest efficiency      │         │
│  │                                                        │         │
│  │  Example:                                             │         │
│  │  Available Es/N0 = 11.5 dB                           │         │
│  │                                                        │         │
│  │  Options that work:                                   │         │
│  │  • 16APSK 3/4 (10.21 dB threshold) → 2.97 bps/Hz ✓  │         │
│  │  • 16APSK 4/5 (11.03 dB threshold) → 3.17 bps/Hz ✓  │         │
│  │  • 16APSK 5/6 (11.61 dB threshold) → TOO HIGH ✗     │         │
│  │                                                        │         │
│  │  Select: 16APSK 4/5 (highest efficiency that works)  │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

```python
def select_modcod(es_n0_db, margin_db=1.0, modcods=MODCODS_S2):
    """Select best MODCOD for given Es/N0."""
    available = es_n0_db - margin_db
    candidates = [mc for mc in modcods if mc["threshold"] <= available]
    if not candidates:
        return None
    return max(candidates, key=lambda x: x["eff"])

def calculate_data_rate(symbol_rate_msps, modcod):
    """Calculate data rate in Mbps."""
    return symbol_rate_msps * modcod["eff"]

def cn_to_es_n0(cn_db, bandwidth_mhz, symbol_rate_msps):
    """Convert C/N to Es/N0."""
    return cn_db + 10 * np.log10(bandwidth_mhz / symbol_rate_msps)

acm_min = min(mc["threshold"] for mc in MODCODS_S2)
acm_max = max(mc["threshold"] for mc in MODCODS_S2)
eff_min = min(mc["eff"] for mc in MODCODS_S2)
eff_max = max(mc["eff"] for mc in MODCODS_S2)

print(f"  DVB-S2 ACM dynamic range:")
print(f"    SNR range: {acm_min:.1f} dB to {acm_max:.1f} dB ({acm_max - acm_min:.1f} dB span)")
print(f"    Efficiency: {eff_min:.2f} to {eff_max:.2f} bps/Hz")
print(f"    Ratio: {eff_max/eff_min:.1f}x capacity variation")
```

| SNR Range | Typical MODCODs | Use Case |
|-----------|-----------------|----------|
| < 0 dB | QPSK 1/4 to 2/5 | Deep space, severe fade |
| 0-5 dB | QPSK 1/2 to 5/6 | Rain fade, small terminals |
| 5-10 dB | 8PSK, 16APSK low | Typical VSAT operation |
| 10-15 dB | 16APSK, 32APSK | Clear sky, large terminals |

<section class="lesson-pitfalls">

## Pitfalls

1. **Confusing C/N and Es/N0.** They differ by 10·log(B/Rs). Use correct units for MODCOD lookup.

2. **Forgetting implementation loss.** Real modems are ~0.5-1 dB worse than table thresholds.

3. **Ignoring short frame penalty.** Short frames are ~0.2-0.5 dB worse than normal frames.

4. **Mixing DVB-S2 and S2X tables.** S2X MODCODs require S2X-capable equipment.

5. **HPA backoff for high-order APSK.** 32APSK needs 2-3 dB output backoff, reducing effective EIRP.

6. **Static MODCOD in variable conditions.** Use ACM for links with rain fading.

</section>

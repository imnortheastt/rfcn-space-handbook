## Why this matters

Counter-intuitively, adding a third burn can sometimes use less fuel than the "optimal" Hohmann transfer. The bi-elliptic transfer exploits the Oberth effect—burning when you're moving fastest gives more energy change per unit fuel. For orbit changes with radius ratios greater than 11.94, bi-elliptic wins on delta-v. The trade-off: much longer transfer times.

### The Oberth effect explanation

```
Why Bi-elliptic Can Beat Hohmann:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  OBERTH EFFECT:                                                     │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Kinetic energy: KE = ½mv²                            │         │
│  │                                                        │         │
│  │  Change in KE from adding Δv:                        │         │
│  │  ΔKE = ½m(v + Δv)² - ½mv² = m⋅v⋅Δv + ½m(Δv)²        │         │
│  │                                                        │         │
│  │  The m⋅v⋅Δv term means:                              │         │
│  │  • Higher v → more energy per unit Δv               │         │
│  │  • Burns at periapsis (high v) are more efficient   │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  BI-ELLIPTIC STRATEGY:                                              │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  1. Burn #1 at periapsis (r₁): v is HIGH             │         │
│  │     → Efficient boost to very high apoapsis          │         │
│  │                                                        │         │
│  │  2. Burn #2 at apoapsis (r_b): v is LOW              │         │
│  │     → Cheap retro-burn (small Δv needed)             │         │
│  │                                                        │         │
│  │  3. Burn #3 at periapsis (r₂): v is HIGH             │         │
│  │     → Efficient circularization                      │         │
│  │                                                        │         │
│  │  For large r₂/r₁, the savings at burns #1 and #3    │         │
│  │  outweigh the cost of burn #2                        │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## How it works

### Delta-V equations

```
Bi-elliptic Delta-V Calculation:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  GIVEN:                                                             │
│  • r₁ = initial orbit radius                                        │
│  • r₂ = target orbit radius                                         │
│  • r_b = intermediate apoapsis radius (r_b > r₂)                    │
│  • μ = GM                                                           │
│                                                                      │
│  TRANSFER ELLIPSES:                                                 │
│  • 1st ellipse: a₁ = (r₁ + r_b) / 2                                │
│  • 2nd ellipse: a₂ = (r₂ + r_b) / 2                                │
│                                                                      │
│  DELTA-V EQUATIONS:                                                 │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Burn #1 (at r₁):                                     │         │
│  │  Δv₁ = √(μ(2/r₁ - 1/a₁)) - √(μ/r₁)                  │         │
│  │                                                        │         │
│  │  Burn #2 (at r_b):                                    │         │
│  │  Δv₂ = |√(μ(2/r_b - 1/a₂)) - √(μ(2/r_b - 1/a₁))|    │         │
│  │                                                        │         │
│  │  Burn #3 (at r₂):                                     │         │
│  │  Δv₃ = √(μ/r₂) - √(μ(2/r₂ - 1/a₂))                  │         │
│  │                                                        │         │
│  │  Total: Δv_total = |Δv₁| + |Δv₂| + |Δv₃|            │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Apply

**Bi-elliptic transfer calculations:**

```python
import numpy as np

MU_EARTH = 398600.4418  # km³/s²

def hohmann_delta_v(r1, r2, mu=MU_EARTH):
    """Calculate Hohmann transfer delta-v."""
    a_t = (r1 + r2) / 2
    
    v1_circ = np.sqrt(mu / r1)
    v1_trans = np.sqrt(mu * (2/r1 - 1/a_t))
    delta_v1 = v1_trans - v1_circ
    
    v2_circ = np.sqrt(mu / r2)
    v2_trans = np.sqrt(mu * (2/r2 - 1/a_t))
    delta_v2 = v2_circ - v2_trans
    
    return abs(delta_v1) + abs(delta_v2)

def bielliptic_delta_v(r1, r2, rb, mu=MU_EARTH):
    """
    Calculate bi-elliptic transfer delta-v.
    
    Args:
        r1: Initial orbit radius
        r2: Target orbit radius
        rb: Intermediate apoapsis radius (rb > max(r1, r2))
        mu: Gravitational parameter
    """
    # Semi-major axes
    a1 = (r1 + rb) / 2  # 1st transfer ellipse
    a2 = (r2 + rb) / 2  # 2nd transfer ellipse
    
    # Burn 1: at r1, boost to 1st ellipse
    v1_circ = np.sqrt(mu / r1)
    v1_trans1 = np.sqrt(mu * (2/r1 - 1/a1))
    delta_v1 = abs(v1_trans1 - v1_circ)
    
    # Burn 2: at rb, change to 2nd ellipse
    v_rb_trans1 = np.sqrt(mu * (2/rb - 1/a1))
    v_rb_trans2 = np.sqrt(mu * (2/rb - 1/a2))
    delta_v2 = abs(v_rb_trans2 - v_rb_trans1)
    
    # Burn 3: at r2, circularize
    v2_trans2 = np.sqrt(mu * (2/r2 - 1/a2))
    v2_circ = np.sqrt(mu / r2)
    delta_v3 = abs(v2_circ - v2_trans2)
    
    return delta_v1, delta_v2, delta_v3, delta_v1 + delta_v2 + delta_v3

def bielliptic_time(r1, r2, rb, mu=MU_EARTH):
    """Calculate bi-elliptic transfer time."""
    a1 = (r1 + rb) / 2
    a2 = (r2 + rb) / 2
    
    t1_half = np.pi * np.sqrt(a1**3 / mu)  # Half period of 1st ellipse
    t2_half = np.pi * np.sqrt(a2**3 / mu)  # Half period of 2nd ellipse
    
    return (t1_half + t2_half) / 3600  # hours

def find_optimal_rb(r1, r2, mu=MU_EARTH, rb_range=None):
    """Find optimal rb that minimizes total delta-v."""
    if rb_range is None:
        rb_range = np.linspace(r2 * 1.1, r2 * 10, 100)
    
    best_rb = rb_range[0]
    best_dv = float('inf')
    
    for rb in rb_range:
        _, _, _, total = bielliptic_delta_v(r1, r2, rb, mu)
        if total < best_dv:
            best_dv = total
            best_rb = rb
    
    return best_rb, best_dv

# Example calculations
print("Bi-elliptic Transfer Analysis")
print("=" * 60)

# Compare with Hohmann for different radius ratios
print("\n1. Bi-elliptic vs Hohmann by Radius Ratio")
print("-" * 60)

r1 = 6771  # LEO radius (km)
print(f"  Starting orbit: {r1} km radius")
print(f"")
print(f"  {'R ratio':>8} {'r₂ (km)':>12} {'Hohmann':>10} {'Bi-elliptic':>12} {'Winner':>10}")
print(f"  {'':>8} {'':>12} {'(km/s)':>10} {'(km/s)':>12} {'':>10}")
print("-" * 60)

for ratio in [5, 10, 11.94, 15, 20, 50]:
    r2 = r1 * ratio
    dv_hoh = hohmann_delta_v(r1, r2)
    
    # Try different rb values to find best bi-elliptic
    rb_opt, dv_bie = find_optimal_rb(r1, r2)
    
    winner = "Hohmann" if dv_hoh <= dv_bie else "Bi-elliptic"
    print(f"  {ratio:>8.2f} {r2:>12.0f} {dv_hoh:>10.3f} {dv_bie:>12.3f} {winner:>10}")

# Detailed breakdown for a specific case
print("\n2. Detailed Bi-elliptic Breakdown (R = 50)")
print("-" * 60)
r1 = 6771
r2 = r1 * 50  # ~338,550 km
rb_values = [r2 * 1.5, r2 * 2, r2 * 5, r2 * 10]

print(f"  {'r_b (km)':>15} {'Δv₁':>8} {'Δv₂':>8} {'Δv₃':>8} {'Total':>8} {'Time':>10}")

    print(f"  {rb:>15.0f} {dv1:>8.3f} {dv2:>8.3f} {dv3:>8.3f} {total:>8.3f} {time_h:>10.1f}")

# Practical example: LEO to GEO
print("\n3. Practical Case: LEO (400 km) to GEO")

print(f"")

    rb = r_geo * rb_factor

    time_h = bielliptic_time(r_leo, r_geo, rb)
```

<section class="lesson-pitfalls">

## Pitfalls

1. **R < 11.94 wastes fuel.** Bi-elliptic never beats Hohmann below this threshold; don't use it.

2. **Time cost is severe.** Transfer time can be 3-10× longer; often not worth the delta-v savings.

3. **Three burns = three failure points.** More engine restarts, more risk.

4. **Radiation exposure.** High apoapsis passes through Van Allen belts; bad for electronics and crew.

5. **Optimal r_b is infinite.** In practice, choose r_b based on time constraints, not minimum delta-v.

6. **Ignores perturbations.** Real trajectories need corrections for Moon, Sun gravity.

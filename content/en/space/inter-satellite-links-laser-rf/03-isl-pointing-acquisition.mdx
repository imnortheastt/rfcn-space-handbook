## Why this matters

Before two satellites can exchange data over an ISL, they must first find each other. For optical links with sub-arcsecond beamwidths, this is like shining a laser pointer from New York and hitting a quarter in Los Angeles—while both are moving at 7 km/s. The Pointing, Acquisition, and Tracking (PAT) system is often the most challenging subsystem of a laser terminal. Understanding PAT sequences is essential for terminal design and link scheduling.

```
Point-Ahead Compensation:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │

│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Light travel time: 2000 km / 3×10⁸ m/s ≈ 6.7 ms     │         │

│  │                                                        │         │
│  │  t = 0:           t = 6.7 ms:                         │         │
│  │                                                        │         │
│  │  🛰️ A ────────────→ (photons traveling)               │         │
│  │   ↓ transmit here                                     │         │
│  │                                                        │         │
│  │            🛰️ B                                       │         │
│  │            (was here)                                 │         │
│  │                        🛰️ B                           │         │
│  │                     → (will be here when photons arrive)│        │
│  │                                                        │         │

│  │                                                        │         │
```

## Apply

**PAT system sizing calculations:**

```python
import numpy as np

# Constants
C = 299792458  # m/s

def point_ahead_angle_urad(relative_velocity_km_s):
    """Calculate point-ahead angle."""
    v = relative_velocity_km_s * 1000  # m/s
    return 2 * v / C * 1e6  # μrad

def acquisition_time_spiral(uncertainty_urad, beam_urad, scan_rate_urad_s):
    """Estimate spiral scan acquisition time (worst case)."""
    area_ratio = (uncertainty_urad / beam_urad) ** 2
    scan_time = uncertainty_urad / scan_rate_urad_s
    return scan_time * np.sqrt(area_ratio)

def acquisition_time_staring(uncertainty_urad, beacon_urad, snr_required_db):
    """Check if staring acquisition is feasible."""
    coverage = (beacon_urad / uncertainty_urad) ** 2
    power_penalty = -20 * np.log10(beacon_urad / uncertainty_urad)
    return coverage >= 1, power_penalty

def tracking_bandwidth_required(jitter_freq_hz, rejection_db):
    """Required loop bandwidth for disturbance rejection."""
    # For -3dB rejection at jitter_freq, need BW >> jitter_freq
    # For -20dB rejection, need BW ≈ 10 × jitter_freq
    factor = 10 ** (rejection_db / 20)
    return jitter_freq_hz * factor

def fsm_stroke_required(uncertainty_urad, margin=1.5):
    """FSM mechanical stroke needed."""
    # Optical angle = 2 × mechanical angle for reflection
    mechanical_urad = uncertainty_urad * margin / 2
    return mechanical_urad

# Analysis
print("PAT System Analysis")
print("=" * 60)

# Point-ahead
print("\n1. Point-Ahead Angle Requirements")
print("-" * 60)
scenarios = [
    ('LEO-LEO intra-plane', 0.5),
    ('LEO-LEO inter-plane', 5.0),
    ('LEO-LEO worst case', 15.0),
    ('GEO-LEO', 3.0),
    ('GEO-GEO', 0.1),
]

print(f"  {'Scenario':25} {'v_rel (km/s)':>12} {'θ_pa (μrad)':>12} {'θ_pa (arcsec)':>14}")
print(f"  {'-'*25} {'-'*12} {'-'*12} {'-'*14}")
for name, v_rel in scenarios:
    pa = point_ahead_angle_urad(v_rel)
    arcsec = pa * 0.206
    print(f"  {name:25} {v_rel:>12.1f} {pa:>12.1f} {arcsec:>14.1f}")

# Acquisition time
print("\n2. Acquisition Time Analysis")
print("-" * 60)
uncertainty = 500  # μrad (from ephemeris + attitude errors)
beam = 50  # μrad (communication beam)
beacon = 200  # μrad (wide acquisition beacon)
scan_rate = 100  # μrad/s

print(f"  Uncertainty cone: {uncertainty} μrad")
print(f"  Comm beam: {beam} μrad")
print(f"  Beacon beam: {beacon} μrad")

# Spiral scan
t_spiral = acquisition_time_spiral(uncertainty, beam, scan_rate)
print(f"\n  Spiral scan (narrow beam):")
print(f"    Worst-case time: {t_spiral:.1f} s")

# Staring with beacon
feasible, penalty = acquisition_time_staring(uncertainty, beacon, 10)
print(f"\n  Staring (wide beacon):")
print(f"    Feasible: {feasible}")
print(f"    Power penalty: {penalty:.1f} dB")

# Tracking requirements
print("\n3. Tracking Loop Requirements")
print("-" * 60)
disturbances = [
    ('Reaction wheel jitter', 150, 20),
    ('Solar array slosh', 0.5, 30),
    ('Thermal drift', 0.001, 40),
]

print(f"  {'Disturbance':25} {'Freq (Hz)':>10} {'Rejection':>10} {'BW needed':>12}")
print(f"  {'-'*25} {'-'*10} {'-'*10} {'-'*12}")
for name, freq, rejection in disturbances:
    bw = tracking_bandwidth_required(freq, rejection)
    print(f"  {name:25} {freq:>10.3f} {rejection:>8} dB {bw:>10.1f} Hz")

print(f"  Uncertainty to cover: {uncertainty} μrad")
print(f"  Mechanical stroke needed: ±{stroke_req:.0f} μrad = ±{stroke_req/1000:.2f} mrad")
print(f"  Optical deflection: ±{stroke_req*2:.0f} μrad = ±{stroke_req*2/1000:.2f} mrad")

# Acquisition sequence timeline
print("\n5. Typical Acquisition Sequence Timeline")
print("-" * 60)
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Point-ahead miscalculation.** Forgetting point-ahead or calculating wrong sign causes link loss.

2. **Acquisition timeout.** If scan pattern doesn't cover uncertainty region, link never establishes.

3. **Beacon power tradeoff.** Wide beacon = faster acquisition but less range margin.

4. **Tracking loop instability.** Too high bandwidth causes oscillation; too low misses jitter.

5. **FSM saturation.** If disturbance exceeds FSM range, tracking fails suddenly.

6. **Sun in field of view.** Acquisition sensor saturates; must schedule around Sun angle.

7. **Handover gaps.** Breaking one link to acquire another takes 10-60s—plan routing accordingly.

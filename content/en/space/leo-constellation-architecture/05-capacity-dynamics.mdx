## Why this matters

Unlike GEO systems with fixed footprints, LEO constellation capacity is dynamic—it varies by latitude, time of day, and instantaneous satellite positions. A user in New York at 6 PM competes for capacity with millions of others, while a ship in the Pacific has a satellite nearly to itself. Understanding these dynamics is critical for service planning, SLA design, and network optimization.

## Apply

**Capacity planning calculator:**

```python
import numpy as np

def satellite_capacity_gbps(
    bandwidth_ghz=3.5,
    spectral_efficiency_bps_hz=3.0,
    overhead_fraction=0.15
):
    """Raw satellite throughput capacity."""
    raw = bandwidth_ghz * spectral_efficiency_bps_hz
    usable = raw * (1 - overhead_fraction)
    return usable

def users_per_satellite(
    sat_capacity_gbps,
    avg_user_speed_mbps,
    concurrent_fraction=0.15  # 15% online at once
):
    """Maximum subscribers a satellite can serve."""
    concurrent_capacity_mbps = sat_capacity_gbps * 1000
    concurrent_users = concurrent_capacity_mbps / avg_user_speed_mbps
    total_subscribers = concurrent_users / concurrent_fraction
    return int(total_subscribers)

def system_capacity(
    n_satellites,
    sat_capacity_gbps,
    active_fraction=0.5  # ~half over populated areas
):
    """Total system capacity over populated areas."""
    return n_satellites * sat_capacity_gbps * active_fraction

def congestion_probability(
    subscribers_in_cell,
    sat_capacity_gbps,
    avg_session_mbps=25,
    peak_concurrent_fraction=0.25
):
    """Estimate probability of congestion during peak hours."""
    peak_demand_gbps = (subscribers_in_cell * peak_concurrent_fraction * 
                        avg_session_mbps / 1000)
    
    if peak_demand_gbps < sat_capacity_gbps * 0.7:
        return "Low (< 10%)"
    elif peak_demand_gbps < sat_capacity_gbps * 0.9:
        return "Medium (10-30%)"
    elif peak_demand_gbps < sat_capacity_gbps:
        return "High (30-60%)"
    else:
        return "Very High (> 60%)"

def latitude_density_factor(latitude_deg, inclination_deg=53):
    """Relative satellite density at given latitude."""
    # Simplified model: density peaks at inclination latitude
    if abs(latitude_deg) > inclination_deg:
        # Linear decrease toward poles
        return max(0.1, 1 - (abs(latitude_deg) - inclination_deg) / 30)
    else:
        # Sinusoidal increase from equator to inclination
        return 0.5 + 0.5 * np.sin(np.pi * abs(latitude_deg) / (2 * inclination_deg))

# Analysis
print("LEO Constellation Capacity Analysis")
print("=" * 70)

# Per-satellite capacity
sat_cap = satellite_capacity_gbps()
print(f"\n1. Per-Satellite Capacity")
print("-" * 70)
print(f"  Bandwidth: 3.5 GHz")
print(f"  Spectral efficiency: 3.0 bps/Hz")
print(f"  Usable capacity: {sat_cap:.1f} Gbps")

# System capacity
print(f"\n2. System Capacity (4000 satellites)")
print("-" * 70)
n_sats = 4000
sys_cap = system_capacity(n_sats, sat_cap)
print(f"  Total satellites: {n_sats}")
print(f"  Active over populated areas: ~50%")
print(f"  System capacity: {sys_cap:.0f} Gbps = {sys_cap/1000:.1f} Tbps")

# Subscribers estimate
print(f"\n3. Subscriber Capacity Estimate")
print("-" * 70)
speeds = [100, 50, 25]
for speed in speeds:
    users = users_per_satellite(sat_cap, speed)
    total = users * n_sats * 0.5  # 50% over populated
    print(f"  @ {speed} Mbps average: {users:,} per sat, {total/1e6:.1f}M system-wide")

# Congestion analysis by location
print(f"\n4. Congestion Analysis by Cell Density")
print("-" * 70)
print(f"{'Subscribers/cell':<20} {'Peak demand':<15} {'Congestion risk':<20}")
print("-" * 70)
for subs in [100, 300, 500, 1000, 2000]:
    demand = subs * 0.25 * 25 / 1000  # 25% concurrent, 25 Mbps avg
    risk = congestion_probability(subs, sat_cap)
    print(f"{subs:<20} {demand:.1f} Gbps{'':<7} {risk:<20}")

# Latitude analysis
print(f"\n5. Capacity by Latitude (53° inclination)")
print("-" * 70)
print(f"{'Latitude':<12} {'Sat Density':<15} {'Effective Cap':<15}")
print("-" * 70)
for lat in [0, 20, 35, 45, 53, 60, 70]:
    density = latitude_density_factor(lat)
    eff_cap = sat_cap * density
    print(f"{lat}°{'':<10} {density:.2f}x{'':<12} {eff_cap:.1f} Gbps/sat")

# Time-of-day simulation
print(f"\n6. Demand Variation (US Eastern, typical residential)")
print("-" * 70)
hours = [0, 3, 6, 9, 12, 15, 18, 21]
demand_factors = [0.2, 0.1, 0.15, 0.4, 0.5, 0.6, 0.9, 1.0]

print(f"{'Hour':<10} {'Demand':<15} {'Cell status':<20}")
print("-" * 70)
for hour, factor in zip(hours, demand_factors):
    demand_gbps = 500 * factor * 0.25 * 25 / 1000  # 500 subs
    status = congestion_probability(int(500 * factor), sat_cap)
    bar = "█" * int(factor * 20)
    print(f"{hour:02d}:00{'':<5} {bar:<20} {status:<20}")

# SLA planning
```

## Pitfalls

1. **Marketing vs reality.** "Up to 500 Mbps" means peak, not average during congestion.

2. **Overbooking assumptions.** 20:1 works for casual users; gamers/streamers break the model.

3. **Geography ignored.** Selling to dense urban areas without capacity planning.

4. **Peak hour blindness.** Capacity looks fine at 3 AM; 8 PM is the real test.

5. **Cell boundary effects.** Users at cell edges get worse speeds than cell center.

6. **Weather impact.** Rain fade reduces effective capacity; must plan for 99% availability.

7. **Backhaul bottleneck.** Gateway capacity may limit before satellite capacity does.

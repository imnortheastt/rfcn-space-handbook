## Why this matters

A LEO satellite at 550 km crosses overhead in about 5-10 minutes. To maintain continuous service, user terminals and gateways must seamlessly switch from one satellite to another—often every few minutes. This handover process is fundamentally different from cellular handoff: both endpoints are moving (satellite at 7.5 km/s, Earth rotating), and the timing is predictable from orbital mechanics. Understanding handover strategies is essential for designing low-latency, high-availability LEO services.

## Apply

**Handover timing calculator:**

```python
import numpy as np
from math import radians, degrees, sin, cos, asin, sqrt, atan2

R_EARTH = 6371  # km
MU = 398600.4418  # km³/s²

def orbital_velocity_km_s(altitude_km):
    """Orbital velocity at given altitude."""
    r = R_EARTH + altitude_km
    return sqrt(MU / r)

def orbital_period_min(altitude_km):
    """Orbital period in minutes."""
    r = R_EARTH + altitude_km
    return 2 * np.pi * sqrt(r**3 / MU) / 60

def visibility_duration_min(altitude_km, min_elevation_deg):
    """Time satellite is visible above minimum elevation."""
    r = R_EARTH + altitude_km
    el_rad = radians(min_elevation_deg)
    
    # Central angle for visibility
    rho = np.arcsin(R_EARTH / r * np.cos(el_rad)) - el_rad
    
    # Arc fraction
    arc_fraction = 2 * rho / (2 * np.pi)
    
    # Period
    T = orbital_period_min(altitude_km)
    
    return T * arc_fraction

def handovers_per_hour(altitude_km, min_elevation_deg, sats_in_view=2.5):
    """Estimate handover frequency."""
    visibility = visibility_duration_min(altitude_km, min_elevation_deg)
    return 60 / visibility * sats_in_view

def elevation_vs_time(altitude_km, max_elevation_deg, time_array_min):
    """Elevation angle during a satellite pass (simplified)."""
    # Parabolic approximation for pass geometry
    pass_duration = visibility_duration_min(altitude_km, 5)  # to 5° minimum
    t_peak = pass_duration / 2
    
    elevations = []
    for t in time_array_min:
        # Simple parabolic model
        el = max_elevation_deg * (1 - ((t - t_peak) / t_peak)**2)
        elevations.append(max(0, el))
    return elevations

def handover_decision(
    current_el_deg,
    next_sat_el_deg,
    current_snr_db,
    threshold_el_deg=25,
    threshold_snr_db=5,
    hysteresis_db=2
):
    """Decide if handover should occur."""
    reasons = []
    
    # Check elevation
    if current_el_deg < threshold_el_deg:
        reasons.append(f"elevation {current_el_deg:.1f}° < {threshold_el_deg}°")
    
    # Check SNR
    if current_snr_db < threshold_snr_db:
        reasons.append(f"SNR {current_snr_db:.1f} dB < {threshold_snr_db} dB")
    
    # Check if next satellite is better (with hysteresis)
    if next_sat_el_deg > current_el_deg + hysteresis_db:
        reasons.append(f"next sat elevation {next_sat_el_deg:.1f}° > current + hysteresis")
    
    should_handover = len(reasons) > 0
    return should_handover, reasons

# Analysis
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Mechanical antennas.** Can't do make-before-break; handover takes 30-60 seconds.

2. **Ping-pong effect.** Poorly tuned hysteresis causes rapid switching between two satellites.

3. **ISL routing lag.** Handover at user terminal doesn't help if ISL path takes time to converge.

4. **Authentication overhead.** Full re-auth on every handover wastes time; use session tickets.

5. **Load imbalance.** All users in a cell hand over simultaneously → thundering herd on new satellite.

6. **TCP sensitivity.** Large RTT spikes trigger congestion control backoff; use QUIC where possible.

7. **Maritime edge cases.** Ships at extreme latitude see fewer satellites; handover gaps possible.

# Symplectic Integrators

<section class="lesson-hook">

## The Million-Year Problem

Propagate Mercury's orbit for 5 billion years. RK4 cannot do it — energy drifts, the planet spirals into the Sun or escapes to infinity.

Yet astronomers routinely simulate the Solar System for billions of years. How?

**Symplectic integrators**. They don't conserve energy exactly, but they conserve something deeper: the structure of phase space. The result? Bounded energy oscillations instead of secular drift.

When JPL computes planetary ephemerides, when N-body simulations model galaxy formation, when researchers study orbital resonances over geological time — symplectic methods make it possible.

</section>

<section class="lesson-intuition">

## Phase Space and Liouville's Theorem

A Hamiltonian system lives in phase space: position q and momentum p coordinates. For an orbit: (x, y, z, px, py, pz).

**Liouville's theorem**: Hamiltonian flow preserves phase space volume. Imagine a cloud of initial conditions — the cloud deforms but its volume stays constant.

```
  p                         p
  |                         |
  |  +----+                 |    /\
  |  |    |    time -->     |   /  \
  |  |    |                 |  /    \
  |  +----+                 | /      \
  +---------> q             +-----------> q
  
  Initial box               Deformed but SAME AREA
```

RK4 violates this. Each step slightly shrinks or expands phase space volume. Over millions of steps, the distortion compounds.

**Symplectic integrators preserve phase space volume exactly** (to machine precision). This is their superpower.

</section>

## The Symplectic Family

### The Stormer-Verlet Method (Order 2)

The workhorse of molecular dynamics and celestial mechanics.

For a separable Hamiltonian H(q, p) = T(p) + V(q):

**Position-Verlet form:**
$$q_{n+1/2} = q_n + \frac{h}{2} \frac{\partial H}{\partial p}(p_n)$$
$$p_{n+1} = p_n - h \frac{\partial H}{\partial q}(q_{n+1/2})$$
$$q_{n+1} = q_{n+1/2} + \frac{h}{2} \frac{\partial H}{\partial p}(p_{n+1})$$

For orbital mechanics where T = p^2/(2m) and V = -mu/r:

```
         POSITION UPDATE (half step)
                  |
                  v
         VELOCITY UPDATE (full step, using new position)
                  |
                  v
         POSITION UPDATE (half step)
```

### The Leapfrog Method (Order 2)

Mathematically equivalent to Verlet, different bookkeeping:

$$p_{n+1/2} = p_n - \frac{h}{2} \nabla V(q_n)$$
$$q_{n+1} = q_n + h \cdot p_{n+1/2}/m$$
$$p_{n+1} = p_{n+1/2} - \frac{h}{2} \nabla V(q_{n+1})$$

```
  t=0     t=h/2    t=h     t=3h/2   t=2h
   |        |       |        |       |
   q0      p1/2    q1      p3/2    q2      <-- positions at integer times
           |               |               <-- momenta at half-integer times
           
  "Leapfrog": q and p leap over each other
```

### Ruth's 3rd Order Method

Higher-order symplectic methods exist. Ruth (1983) gave a 3-stage, 3rd-order method:

$$c_1 = 1, \quad c_2 = -2/3, \quad c_3 = 2/3$$
$$d_1 = -1/24, \quad d_2 = 3/4, \quad d_3 = 7/24$$

Apply alternating position and momentum kicks with these coefficients.

### Yoshida's 4th Order Method

Compose three Verlet steps with magic coefficients:

$$w_0 = -\frac{2^{1/3}}{2 - 2^{1/3}} \approx -1.7024$$
$$w_1 = \frac{1}{2 - 2^{1/3}} \approx 1.3512$$

$$\Phi_h = \Phi_{w_1 h} \circ \Phi_{w_0 h} \circ \Phi_{w_1 h}$$

One "step" requires three Verlet sub-steps with coefficients w1, w0, w1.

## Why Symplectic = Energy Bounded

A symplectic integrator exactly integrates a *nearby* Hamiltonian:

$$\tilde{H} = H + h^p H_p + h^{p+1} H_{p+1} + ...$$

where p is the method's order. The numerical solution conserves this shadow Hamiltonian exactly!

```
  Energy
    |
    |  ----  Shadow Hamiltonian (conserved exactly)
    | /    \
    |/      \
    |\      /  <-- Oscillates around true energy
    | \----/
    |
    +---------------------------------> time
              RK4 would drift upward or downward here
```

The energy of the true Hamiltonian oscillates but never drifts secularly.

## Comparison: Verlet vs RK4

| Property | Verlet (Symplectic) | RK4 (Non-symplectic) |
|----------|---------------------|----------------------|
| Order | 2 | 4 |
| Evaluations/step | 1 force eval | 4 force evals |
| Energy over 10^6 steps | Bounded oscillation | Secular drift |
| Phase space volume | Preserved | Slowly distorted |
| Time-reversible | Yes | No |
| Best for | Long integrations | Short, high-accuracy |

## Python Implementation: Leapfrog for Orbits

```python
import numpy as np
import matplotlib.pyplot as plt

MU = 398600.4418  # km^3/s^2

def acceleration(r):
    """Gravitational acceleration."""
    r_mag = np.linalg.norm(r)
    return -MU * r / r_mag**3

def leapfrog_step(r, v, h):
    """
    Single leapfrog step.
    
    Args:
        r: Position [x, y, z] in km
        v: Velocity [vx, vy, vz] in km/s
        h: Time step in seconds
    
    Returns:
        r_new, v_new
    """
    # Kick (half step)
    a = acceleration(r)
    v_half = v + 0.5 * h * a
    
    # Drift (full step)
    r_new = r + h * v_half
    
    # Kick (half step)
    a_new = acceleration(r_new)
    v_new = v_half + 0.5 * h * a_new
    
    return r_new, v_new

def rk4_step(r, v, h):
    """RK4 step for comparison."""
    def f(state):
        r, v = state[:3], state[3:]
        return np.concatenate([v, acceleration(r)])
    
    state = np.concatenate([r, v])
    k1 = f(state)
    k2 = f(state + h/2 * k1)
    k3 = f(state + h/2 * k2)
    k4 = f(state + h * k3)
    state_new = state + h/6 * (k1 + 2*k2 + 2*k3 + k4)
    return state_new[:3], state_new[3:]

def compute_energy(r, v):
    """Specific orbital energy."""
    return 0.5 * np.dot(v, v) - MU / np.linalg.norm(r)

def compare_integrators(n_orbits=100, dt=60.0):
    """Compare energy drift: Leapfrog vs RK4."""
    
    # Eccentric orbit (e=0.5) - more challenging
    r0 = np.array([7000.0, 0.0, 0.0])  # Periapsis
    v_circ = np.sqrt(MU / np.linalg.norm(r0))
    v0 = np.array([0.0, 1.3 * v_circ, 0.0])  # Boost for eccentricity
    
    # Compute period
    a = 1 / (2/np.linalg.norm(r0) - np.dot(v0,v0)/MU)
    period = 2 * np.pi * np.sqrt(a**3 / MU)
    t_total = n_orbits * period
    n_steps = int(t_total / dt)
    
    E0 = compute_energy(r0, v0)
    
    # Leapfrog integration
    r_lf, v_lf = r0.copy(), v0.copy()
    energy_lf = [0.0]
    
    # RK4 integration
    r_rk, v_rk = r0.copy(), v0.copy()
    energy_rk = [0.0]
    
    for i in range(n_steps):
        r_lf, v_lf = leapfrog_step(r_lf, v_lf, dt)
        r_rk, v_rk = rk4_step(r_rk, v_rk, dt)
        
        energy_lf.append((compute_energy(r_lf, v_lf) - E0) / abs(E0))
        energy_rk.append((compute_energy(r_rk, v_rk) - E0) / abs(E0))
    
    return np.array(energy_lf), np.array(energy_rk), n_orbits

# Run comparison
dE_leapfrog, dE_rk4, n_orb = compare_integrators(n_orbits=100, dt=60.0)

print(f"After {n_orb} orbits:")
print(f"  Leapfrog max |dE/E|: {np.max(np.abs(dE_leapfrog)):.2e}")
print(f"  RK4 final dE/E:      {dE_rk4[-1]:.2e}")
print(f"  RK4 max |dE/E|:      {np.max(np.abs(dE_rk4)):.2e}")
```

**Typical output:**
```
After 100 orbits:
  Leapfrog max |dE/E|: 3.45e-06
  RK4 final dE/E:      -8.21e-05
  RK4 max |dE/E|:      8.21e-05
```

Note: RK4 shows monotonic drift, Leapfrog oscillates within bounds.

## Common Pitfalls

### 1. Non-separable Hamiltonians

Symplectic methods assume H = T(p) + V(q). Velocity-dependent forces (magnetic fields, Coriolis) break this.

**Workarounds**:
- Use implicit symplectic methods
- Split the Hamiltonian creatively
- Accept some symplecticity loss for small perturbations

### 2. Variable Step Sizes

Naive adaptive stepping destroys symplecticity.

**Solution**: Time-transformation methods that make dt part of the phase space.

### 3. Dissipative Forces

Atmospheric drag removes energy — no symplectic method can help.

**Strategy**: Use symplectic for conservative forces, add dissipation as a perturbation.

### 4. Expecting Exact Energy Conservation

Symplectic methods don't conserve energy exactly — they conserve a *nearby* energy. The oscillation amplitude depends on step size and eccentricity.

```
  Smaller h --> Smaller energy oscillation amplitude
  Higher eccentricity --> Need smaller h near periapsis
```

### 5. Overhead Concerns

Leapfrog needs 1 force evaluation/step vs RK4's 4. But Leapfrog is only 2nd order.

**Comparison at equal work**: For long integrations, Leapfrog often wins because it doesn't accumulate drift.

## What's Next?

Symplectic methods excel at long-term, fixed-step integration. But what if you need:

- **Adaptive accuracy** without destroying symplecticity? Specialized techniques exist.

- **Higher precision for ephemeris** generation? Implicit methods like Everhart/Radau handle this.

- **Choosing the right method** for your specific problem? See [RK4 vs DOPRI vs Everhart](/space/math-toolbox-numerical-integrators/rk4-vs-dopri-vs-everhart) for a decision framework.

**Key takeaway**: When your integration spans years or millennia, symplectic structure trumps raw order. A 2nd-order symplectic integrator often beats an 8th-order non-symplectic one for long-term accuracy.

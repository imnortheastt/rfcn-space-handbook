## Why this matters

Before phased arrays, there were motorized dishes—and they're still everywhere. GEO satellite earth stations, radio telescopes, radar systems, and many maritime/aviation terminals use mechanical tracking. Understanding azimuth-elevation mounts, servo control loops, and tracking algorithms is essential for operating traditional ground stations and understanding the baseline that ESAs are replacing.

## How it works

### Tracking modes

```
Satellite Tracking Modes:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  1. PROGRAM TRACK (open loop):                                      │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  [Ephemeris] ──► [Compute Az/El] ──► [Motors]        │         │
│  │                                                        │         │
│  │  • Uses predicted satellite position                 │         │
│  │  • No feedback from signal                           │         │
│  │  • Good for: acquisition, LEO passes                 │         │
│  │  • Accuracy limited by ephemeris quality             │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  2. STEP TRACK (closed loop):                                       │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  [Measure signal] ──► [Step Az/El] ──► [Maximize]   │         │
│  │                                 ↑____________│        │         │
│  │                                                        │         │
│  │  • Small steps, measure signal after each            │         │
│  │  • Hill-climbing to maximize received power          │         │
│  │  • Good for: GEO tracking, fine pointing             │         │
│  │  • Slow but robust                                   │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  3. MONOPULSE (closed loop, fast):                                  │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  Four-horn feed compares signal in quadrants         │         │
│  │                                                        │         │
│  │       ┌───┬───┐                                      │         │
│  │       │ A │ B │  Δaz ∝ (A+C) - (B+D)                │         │
│  │       ├───┼───┤  Δel ∝ (A+B) - (C+D)                │         │
│  │       │ C │ D │                                      │         │
│  │       └───┴───┘                                      │         │
│  │                                                        │         │
│  │  • Instantaneous error signal                        │         │
│  │  • No stepping needed                                │         │
│  │  • Fast enough for LEO tracking                      │         │
│  │  • More complex feed system                          │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  4. CONICAL SCAN:                                                   │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  • Feed nutates (wobbles) in small circle            │         │
│  │  • Signal modulation indicates pointing error        │         │
│  │  • Simpler than monopulse but slower                 │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Apply

**Mechanical tracking calculations:**

```python
import numpy as np

def satellite_angular_velocity(altitude_km, elevation_deg):
    """
    Approximate angular velocity of LEO satellite as seen from ground.
    
    Args:
        altitude_km: Satellite altitude
        elevation_deg: Current elevation angle
    
    Returns:
        Angular velocity in deg/s
    """
    R_earth = 6371  # km
    orbital_velocity = 7.5  # km/s typical for LEO
    
    # Slant range
    el_rad = np.radians(elevation_deg)
    slant_range = np.sqrt((R_earth + altitude_km)**2 - 
                          (R_earth * np.cos(el_rad))**2) - R_earth * np.sin(el_rad)
    
    # Approximate angular velocity (simplified)
    # Faster near horizon, slower at zenith
    angular_vel = np.degrees(orbital_velocity / (slant_range + altitude_km))
    
    return angular_vel

def pointing_error_to_loss_db(error_deg, beamwidth_deg):
    """
    Calculate gain loss due to pointing error.
    
    For parabolic dish: G(θ) ≈ G_max × exp(-2.76 × (θ/θ_3dB)²)
    """
    x = error_deg / beamwidth_deg
    loss_db = -12 * x**2  # Approximation for Gaussian beam
    return loss_db

def motor_sizing(dish_diameter_m, max_velocity_deg_s, max_acceleration_deg_s2):
    """
    Estimate motor torque requirements.
    """
    # Moment of inertia approximation for dish
    # Treat as thin disk: I = 0.25 × m × r²
    # Mass estimate: ~10 kg/m² for typical dish
    area = np.pi * (dish_diameter_m/2)**2
    mass = area * 10  # kg
    radius = dish_diameter_m / 2
    inertia = 0.25 * mass * radius**2  # kg⋅m²
    
    # Torque = I × α
    alpha_rad = np.radians(max_acceleration_deg_s2)
    torque = inertia * alpha_rad
    
    return {
        "mass_kg": mass,
        "inertia_kg_m2": inertia,
        "torque_nm": torque,
    }
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Keyhole at zenith.** Az-El mounts fail for overhead passes; need X-Y or accept dropout.

2. **Wind loading.** Large dishes need stow position and wind speed limits; tracking degrades in wind.

3. **Backlash.** Gear backlash in motors causes tracking hysteresis; use anti-backlash gears or direct drive.

4. **Ice and snow.** Dish accumulation changes balance and blocks signal; heating systems needed.

5. **Cable wrap.** Continuous rotation requires slip rings or cable wrap limits; plan for unwrap time.

6. **Encoder resolution.** Poor encoder resolution = poor tracking; typical need: 0.001° or better.

7. **LEO vs GEO confusion.** GEO tracking hardware usually can't handle LEO speeds; verify specs.

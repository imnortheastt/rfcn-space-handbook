## Why this matters

Electronic steering is what makes flat-panel satellite antennas possible. Instead of physically rotating a dish, we compute the right phase shift for each antenna element to point the beam wherever we want—in microseconds. Understanding beamforming algorithms, phase shifter technologies, and the tradeoffs between analog and digital approaches is essential for designing next-generation satellite terminals and base stations.

## Intuition

### From physics to algorithm

```
The Beamforming Problem:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  GIVEN:                                                             │
│  • N antenna elements at known positions                            │
│  • Desired beam direction (θ, φ) in spherical coordinates          │
│  • Operating frequency f (wavelength λ)                            │
│                                                                      │
│  FIND:                                                              │
│  • Phase shift φₙ for each element n                               │
│  • (Optional) Amplitude weight wₙ for each element                 │
│                                                                      │
│  SOLUTION (uniform linear array):                                   │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  φₙ = -k × dₙ × sin(θ)                                │         │
│  │                                                        │         │
│  │  where:                                                │         │
│  │    k = 2π/λ (wavenumber)                              │         │
│  │    dₙ = position of element n                        │         │
│  │    θ = desired steering angle                         │         │
│  │                                                        │         │
│  │  For 2D planar array:                                 │         │
│  │  φₘₙ = -k × (xₘ × sin(θ)cos(φ) + yₙ × sin(θ)sin(φ)) │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## How it works

### Analog vs digital beamforming

```
Beamforming Architectures:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  ANALOG BEAMFORMING:                                                │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  ▭ ──► [φ₁] ──┐                                       │         │
│  │  ▭ ──► [φ₂] ──┼──► [Combiner] ──► [ADC] ──► DSP     │         │
│  │  ▭ ──► [φ₃] ──┤                                       │         │
│  │  ▭ ──► [φ₄] ──┘                                       │         │
│  │                                                        │         │
│  │  • Single RF chain (cheaper)                          │         │
│  │  • Phase shifts in analog domain                     │         │
│  │  • Limited to single beam at a time                  │         │
│  │  • Lower power consumption                           │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  DIGITAL BEAMFORMING:                                               │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  ▭ ──► [LNA] ──► [ADC] ──┐                           │         │
│  │  ▭ ──► [LNA] ──► [ADC] ──┼──► DSP (compute φ)       │         │
│  │  ▭ ──► [LNA] ──► [ADC] ──┤                           │         │
│  │  ▭ ──► [LNA] ──► [ADC] ──┘                           │         │
│  │                                                        │         │
│  │  • RF chain per element (expensive)                  │         │
│  │  • Phase shifts computed digitally                   │         │
│  │  • Multiple simultaneous beams possible              │         │
│  │  • Maximum flexibility, highest power                │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
│  HYBRID BEAMFORMING:                                                │
│  ┌───────────────────────────────────────────────────────┐         │
│  │                                                        │         │
│  │  ▭ ▭ ▭ ▭ ──► [Analog BF] ──► [ADC] ──┐               │         │
│  │  ▭ ▭ ▭ ▭ ──► [Analog BF] ──► [ADC] ──┼──► DSP       │         │
│  │  ▭ ▭ ▭ ▭ ──► [Analog BF] ──► [ADC] ──┤               │         │
│  │  ▭ ▭ ▭ ▭ ──► [Analog BF] ──► [ADC] ──┘               │         │
│  │                                                        │         │
│  │  • Subarrays with analog BF                          │         │
│  │  • Digital BF between subarrays                      │         │
│  │  • Best of both worlds (cost vs flexibility)         │         │
│  │  • Common in 5G massive MIMO and satcom              │         │
│  │                                                        │         │
│  └───────────────────────────────────────────────────────┘         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Apply

**Beamforming algorithm implementation:**

```python
import numpy as np

def compute_steering_vector(element_positions, theta_deg, phi_deg, frequency_ghz):
    """
    Compute steering vector for arbitrary array geometry.
    
    Args:
        element_positions: Nx3 array of (x,y,z) positions in meters
        theta_deg: Elevation angle from broadside (0=up)
        phi_deg: Azimuth angle (0=x-axis)
        frequency_ghz: Operating frequency
    
    Returns:
        Complex steering vector (phase shifts)
    """
    c = 299792458
    wavelength = c / (frequency_ghz * 1e9)
    k = 2 * np.pi / wavelength
    
    # Convert angles to radians
    theta = np.radians(theta_deg)
    phi = np.radians(phi_deg)
    
    # Unit vector in desired direction
    u = np.array([
        np.sin(theta) * np.cos(phi),
        np.sin(theta) * np.sin(phi),
        np.cos(theta)
    ])
    
    # Phase shift for each element
    phases = -k * np.dot(element_positions, u)
    
    # Return complex steering vector
    return np.exp(1j * phases)

def create_ula(n_elements, spacing_wavelengths):
    """Create uniform linear array positions."""
    positions = np.zeros((n_elements, 3))
    positions[:, 0] = np.arange(n_elements) * spacing_wavelengths
    return positions

def create_upa(nx, ny, spacing_wavelengths):
    """Create uniform planar array positions."""
    positions = []
    for i in range(nx):
        for j in range(ny):
            positions.append([i * spacing_wavelengths, j * spacing_wavelengths, 0])
    return np.array(positions)

def array_factor(element_positions, weights, theta_scan, phi_scan, frequency_ghz):
    """
    Compute array factor (radiation pattern) for given weights.
    """
    theta_vals = np.linspace(-90, 90, 361)
    af = np.zeros(len(theta_vals), dtype=complex)
    
    c = 299792458
    wavelength = c / (frequency_ghz * 1e9)
    k = 2 * np.pi / wavelength
    
    for i, theta in enumerate(theta_vals):
        theta_rad = np.radians(theta)
        u = np.array([np.sin(theta_rad), 0, np.cos(theta_rad)])
        phases = k * np.dot(element_positions, u)
        af[i] = np.sum(weights * np.exp(1j * phases))
    
    return theta_vals, np.abs(af)

# Example: 8-element linear array beamforming
print("Electronic Beam Steering Example")
print("=" * 60)

freq = 12.0  # GHz
wavelength = 299792458 / (freq * 1e9)
n_elements = 8
spacing = 0.5  # wavelengths

# Create array
positions = create_ula(n_elements, spacing * wavelength)

print(f"\nArray Configuration:")
print(f"  Elements: {n_elements}")
print(f"  Frequency: {freq} GHz")
print(f"  Spacing: {spacing}λ = {spacing * wavelength * 1000:.2f} mm")

# Compute steering vectors for different angles
print(f"\n1. Phase Shifts for Different Steering Angles")
print("-" * 60)
print(f"  {'Angle':>8} | {'Phase shifts (degrees)':>45}")
print("-" * 60)

for theta in [0, 15, 30, 45]:
    sv = compute_steering_vector(positions, theta, 0, freq)
    phases = np.angle(sv, deg=True)
    phase_str = " ".join([f"{p:6.1f}" for p in phases])
    print(f"  {theta:>6}° | {phase_str}")

# Phase shifter resolution impact
print(f"\n2. Phase Shifter Resolution Impact")
print("-" * 60)

def quantize_phases(phases, bits):
    """Quantize phases to given resolution."""
    levels = 2 ** bits
    step = 360 / levels
    return np.round(phases / step) * step

theta_target = 30  # degrees
sv_ideal = compute_steering_vector(positions, theta_target, 0, freq)
phases_ideal = np.angle(sv_ideal, deg=True) % 360

print(f"  Target angle: {theta_target}°")
print(f"  {'Bits':>6} {'Levels':>8} {'Max error':>12} {'Pointing error':>15}")

for bits in [2, 3, 4, 5, 6]:
    phases_quant = quantize_phases(phases_ideal, bits)
    max_error = np.max(np.abs(phases_ideal - phases_quant))
    # Approximate pointing error
    pointing_err = max_error / 30  # rough approximation
    print(f"  {bits:>6} {2**bits:>8} {max_error:>10.1f}° {pointing_err:>13.2f}°")

# Beamwidth calculation
print(f"\n3. Beamwidth Analysis")
print("-" * 60)

for n in [4, 8, 16, 32]:
    pos = create_ula(n, spacing * wavelength)
    aperture = (n - 1) * spacing * wavelength
    beamwidth = np.degrees(0.886 * wavelength / aperture)
    gain = 10 * np.log10(n) + 5  # approximate
    print(f"  {n:>3} elements: BW = {beamwidth:>5.1f}°, Gain ≈ {gain:.1f} dBi")

# Steering speed requirements
print(f"\n4. LEO Tracking Update Rate")
print("-" * 60)

sat_altitude_km = 550
orbital_velocity_km_s = 7.5
angular_rate_deg_s = np.degrees(orbital_velocity_km_s / sat_altitude_km)

for bw in [0.5, 1.0, 2.0, 5.0]:
    update_period_ms = (bw / angular_rate_deg_s) * 1000
    print(f"  Beamwidth {bw:>4.1f}°: update every {update_period_ms:>6.0f} ms")
```

## Pitfalls

1. **Phase shifter resolution.** 4-bit (16 levels) minimum for good beam pointing; 6-bit for low sidelobes.

2. **Quantization lobes.** Coarse phase resolution creates additional sidelobes that waste power.

3. **Calibration drift.** Temperature changes can shift phase by degrees; continuous cal needed.

4. **Mutual coupling.** Elements affect each other; ignore this and beams point wrong.

5. **Processing latency.** Digital BF needs fast DSP; latency matters for LEO tracking.

6. **Power consumption.** Digital BF eats power; thermal management critical.

7. **Complexity vs reliability.** More digital = more compute, more failure modes.

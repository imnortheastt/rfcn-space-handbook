## Why this matters

CDMA lets everyone transmit simultaneously on the same frequency, separated by unique codes — like multiple conversations at a party, each in a different language. SCPC is the opposite: one dedicated carrier per link, no sharing. Both have their place. GPS uses CDMA (weak signals, graceful degradation). International TV uplinks use SCPC (maximum throughput, guaranteed bandwidth). Knowing when to use each separates competent satellite engineers from excellent ones.

## How it works

### CDMA spreading and despreading

```
CDMA Signal Processing:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  SPREADING (Transmitter):                                           │
│                                                                      │
│  Data bit:    ┌───────────────────────────┐  (1 bit period)        │
│               │            1              │                         │
│               └───────────────────────────┘                         │
│                            ×                                        │
│  Spreading    ┌─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┐  (chip sequence)       │
│  code:        │+│-│+│+│-│+│-│-│+│-│+│+│-│+│  (PN sequence)         │
│               └─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┘                         │
│                            =                                        │
│  Spread       ┌─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┬─┐                         │
│  signal:      │+│-│+│+│-│+│-│-│+│-│+│+│-│+│                         │
│               └─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┴─┘                         │
│                                                                      │
│  Bandwidth expansion: If 14 chips per bit, BW expands 14×          │
│  This is the "spread" in spread spectrum.                          │
│                                                                      │
│  ─────────────────────────────────────────────────────────────────  │
│                                                                      │
│  DESPREADING (Receiver):                                            │
│                                                                      │
│  Received = User_A (wanted) + User_B (interference) + noise        │
│                                                                      │
│  Correlate with User_A's code:                                     │
│                                                                      │
│  User_A contribution:  code × code = +1 (correlates perfectly)    │
│  User_B contribution:  code × other_code ≈ 0 (orthogonal)          │
│  Noise contribution:   code × noise ≈ 0 (random, averages out)    │
│                                                                      │
│  Result: User_A's signal emerges cleanly!                          │
│                                                                      │
│  This is "processing gain": SNR improves by ~10×log10(chips/bit)  │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### CDMA capacity limit

```
CDMA Self-Interference:

┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  CDMA has a "soft" capacity limit:                                 │
│                                                                      │
│  As more users transmit, interference increases:                   │
│                                                                      │
│  Total interference = (N-1) users × average power per user        │
│                                                                      │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │                                                                │ │
│  │  Users │ Interference │ Eb/N0  │ BER     │ Quality           │ │
│  │  ──────┼──────────────┼────────┼─────────┼────────────────── │ │
│  │    1   │     Low      │ 15 dB  │ 10^-9   │ Excellent         │ │
│  │   10   │    Medium    │ 10 dB  │ 10^-6   │ Good              │ │
│  │   20   │    High      │  6 dB  │ 10^-4   │ Acceptable        │ │
│  │   30   │   Very High  │  3 dB  │ 10^-2   │ Degraded          │ │
│  │   40+  │   Overload   │ &lt;0 dB  │ Fails   │ System collapse   │ │
│  │                                                                │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                      │
│  KEY INSIGHT: CDMA "gracefully degrades" as load increases.        │
│  TDMA/FDMA: works or doesn't. CDMA: works progressively worse.    │
│                                                                      │
│  This is why GPS works with weak signals:                          │
│  • Few simultaneous satellites (4-12)                              │
│  • Long spreading codes (1023 chips)                               │
│  • Plenty of processing gain margin                                │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## Apply

**CDMA and SCPC analysis tools:**

```python
import numpy as np

def cdma_processing_gain(chip_rate_mcps, data_rate_kbps):
    """
    Calculate CDMA processing gain.
    
    Args:
        chip_rate_mcps: Chip rate in Mcps (megachips per second)
        data_rate_kbps: Data rate in kbps
    
    Returns:
        Processing gain in dB
    """
    chips_per_bit = (chip_rate_mcps * 1e6) / (data_rate_kbps * 1e3)
    processing_gain_db = 10 * np.log10(chips_per_bit)
    
    return {
        "chips_per_bit": chips_per_bit,
        "processing_gain_db": processing_gain_db
    }

def cdma_capacity(bandwidth_mhz, data_rate_kbps, required_eb_n0_db,
                  voice_activity_factor=1.0, sectorization_gain=1.0):
    """
    Estimate CDMA system capacity (simplified model).
    
    Args:
        bandwidth_mhz: System bandwidth
        data_rate_kbps: Per-user data rate
        required_eb_n0_db: Required Eb/N0 for desired BER
        voice_activity_factor: Fraction of time users transmit (1.0 for data)
        sectorization_gain: Gain from antenna sectorization
    
    Returns:
        Estimated number of simultaneous users
    """
    chip_rate_mcps = bandwidth_mhz * 0.8  # Approximate
    pg = cdma_processing_gain(chip_rate_mcps, data_rate_kbps)
    
    processing_gain_linear = 10**(pg["processing_gain_db"]/10)
    required_eb_n0_linear = 10**(required_eb_n0_db/10)
    
    num_users = (processing_gain_linear / required_eb_n0_linear) * \
                voice_activity_factor * sectorization_gain
    
    return {
        "processing_gain_db": pg["processing_gain_db"],
        "estimated_users": int(num_users),
        "chips_per_bit": pg["chips_per_bit"]
    }

def scpc_link_efficiency(carrier_bw_mhz, symbol_rate_msps, 
                         bits_per_symbol, code_rate):
    """
    Calculate SCPC link spectral efficiency.
    """
    gross_rate_mbps = symbol_rate_msps * bits_per_symbol
    net_rate_mbps = gross_rate_mbps * code_rate
    spectral_efficiency = net_rate_mbps / carrier_bw_mhz
    
    return {
        "gross_rate_mbps": gross_rate_mbps,
        "net_rate_mbps": net_rate_mbps,
        "spectral_efficiency_bps_hz": spectral_efficiency
    }

def scpc_transponder_fill(transponder_bw_mhz, carriers):
    """
    Calculate transponder utilization with multiple SCPC carriers.
    
    Args:
        transponder_bw_mhz: Total transponder bandwidth
        carriers: List of (carrier_bw_mhz, guard_mhz) tuples
    
    Returns:
        Utilization metrics
    """
    total_used = sum(bw + guard for bw, guard in carriers)
    utilization = total_used / transponder_bw_mhz
    num_carriers = len(carriers)
    
    return {
        "num_carriers": num_carriers,
        "total_used_mhz": total_used,
        "remaining_mhz": transponder_bw_mhz - total_used,
        "utilization": utilization
    }

def compare_cdma_scpc_tdma(num_users, data_rate_kbps, activity_factor,
                           transponder_bw_mhz):
    """
    Compare access methods for a given scenario.
    """
    per_user_bw_scpc = data_rate_kbps / 1000 * 1.5  # Assume 1.5× overhead
    scpc_total = num_users * per_user_bw_scpc
    scpc_efficiency = (num_users * data_rate_kbps / 1000) / transponder_bw_mhz
    
    tdma_overhead = 0.05
    tdma_efficiency = activity_factor * (1 - tdma_overhead)
    
    cdma_cap = cdma_capacity(transponder_bw_mhz, data_rate_kbps, 6,
                             activity_factor, 1.0)
    cdma_efficiency = min(1.0, num_users / cdma_cap["estimated_users"])
    
    return {
        "scpc": {"total_bw_needed": scpc_total, "efficiency": scpc_efficiency},
        "tdma": {"efficiency": tdma_efficiency},
        "cdma": {"max_users": cdma_cap["estimated_users"], 
                 "load_factor": cdma_efficiency}
    }
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Near-far problem in CDMA.** A strong nearby transmitter can drown out weak distant ones. Power control is essential — without it, CDMA fails.

2. **Assuming CDMA always saves bandwidth.** CDMA's processing gain comes at cost of bandwidth expansion. It wins when interference tolerance matters more than raw throughput.

3. **Ignoring SCPC's simplicity.** SCPC modems are cheaper, more reliable, and need no network timing. For dedicated point-to-point links, they often beat shared access.

4. **Forgetting guard bands for SCPC.** Real filters have roll-off. Budget 5-10% between carriers or you'll get adjacent channel interference.

5. **CDMA capacity overestimation.** Theoretical capacity assumes perfect power control and orthogonal codes. Real systems achieve 60-70% of theoretical max.

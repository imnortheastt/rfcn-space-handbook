## Why this matters

The lunar south pole—Artemis's target—is never directly visible from Earth. Without relay infrastructure, astronauts there would have no communication with mission control during critical EVAs. Multiple nations are designing lunar communication architectures: NASA's LunaNet, ESA's Moonlight, and commercial providers. Understanding these systems reveals how cislunar networks differ from Earth-centric satellite constellations and why relay satellites are essential for sustained lunar presence.

## Apply

**Lunar relay coverage analysis:**

```python
import numpy as np

def fspl_db(distance_km, freq_ghz):
    """Free Space Path Loss in dB."""
    return 20 * np.log10(distance_km) + 20 * np.log10(freq_ghz) + 92.45

def link_cn0(eirp_dbw, distance_km, freq_ghz, g_over_t_dbk, losses_db=0):
    """Calculate C/N0 for a link."""
    k_dbw = -228.6
    fspl = fspl_db(distance_km, freq_ghz)
    return eirp_dbw - fspl - losses_db + g_over_t_dbk - k_dbw

def data_rate_from_cn0(cn0_dbhz, ebn0_required_db):
    """Calculate achievable data rate from C/N0."""
    return 10 ** ((cn0_dbhz - ebn0_required_db) / 10)

# Surface user parameters
print("Lunar Surface Communications Analysis")
print("=" * 60)

# Define user types
users = {
    "Astronaut EVA": {"tx_w": 2, "gain_dbi": 0, "freq_ghz": 2.0},
    "Rover": {"tx_w": 20, "gain_dbi": 6, "freq_ghz": 8.0},
    "Lander": {"tx_w": 50, "gain_dbi": 30, "freq_ghz": 26.0},
    "Fixed Base": {"tx_w": 100, "gain_dbi": 40, "freq_ghz": 32.0},
}

# Relay satellite parameters
relay_gt = 10  # dB/K (S-band relay)
relay_gt_ka = 20  # dB/K (Ka-band relay)

# Distances by orbit type
relay_orbits = {
    "LLO (300 km)": [300, 1500],  # [min, max] km
    "ELFO perilune": [700, 2000],
    "ELFO apolune": [6000, 10000],
    "Gateway": [3000, 70000],
}

print("\n1. Surface User EIRP")
print("-" * 60)

print(f"  {'User':<20} {'TX(W)':<8} {'Gain(dBi)':<10} {'EIRP(dBW)':<10}")
print("-" * 60)

for name, params in users.items():
    tx_dbw = 10 * np.log10(params["tx_w"])
    eirp = tx_dbw + params["gain_dbi"]
    print(f"  {name:<20} {params['tx_w']:<8} {params['gain_dbi']:<10} {eirp:.1f}")

# Link budget for each combination
print("\n2. Surface-to-Relay C/N₀ (dBHz)")
print("-" * 60)

# Use astronaut for demo
astro = users["Astronaut EVA"]
astro_eirp = 10 * np.log10(astro["tx_w"]) + astro["gain_dbi"]

print(f"  Astronaut EVA (EIRP = {astro_eirp:.1f} dBW, {astro['freq_ghz']} GHz)")
print()
print(f"  {'Relay Orbit':<18} {'Min Dist':<12} {'Max Dist':<12}")
print("-" * 60)

ebn0_voice = 6  # dB for voice

for orbit, (d_min, d_max) in relay_orbits.items():
    cn0_max = link_cn0(astro_eirp, d_min, astro["freq_ghz"], relay_gt)
    cn0_min = link_cn0(astro_eirp, d_max, astro["freq_ghz"], relay_gt)
    rate_max = data_rate_from_cn0(cn0_max, ebn0_voice) / 1000  # kbps
    rate_min = data_rate_from_cn0(cn0_min, ebn0_voice) / 1000
    print(f"  {orbit:<18} {cn0_max:.1f} dBHz    {cn0_min:.1f} dBHz")
    print(f"  {'':18} ({rate_max:.0f} kbps)   ({rate_min:.0f} kbps)")

# Lander high-rate link
print("\n3. Lander High-Rate Link (Ka-band)")
print("-" * 60)

lander = users["Lander"]
lander_eirp = 10 * np.log10(lander["tx_w"]) + lander["gain_dbi"]

print(f"  Lander: EIRP = {lander_eirp:.1f} dBW @ {lander['freq_ghz']} GHz")
print()

ebn0_data = 3  # dB for high-rate LDPC

for orbit, (d_min, d_max) in relay_orbits.items():
    cn0_max = link_cn0(lander_eirp, d_min, lander["freq_ghz"], relay_gt_ka)
    cn0_min = link_cn0(lander_eirp, d_max, lander["freq_ghz"], relay_gt_ka)
    rate_max = data_rate_from_cn0(cn0_max, ebn0_data) / 1e6  # Mbps
    rate_min = data_rate_from_cn0(cn0_min, ebn0_data) / 1e6
    print(f"  {orbit:<18} {rate_max:.1f} Mbps    {rate_min:.2f} Mbps")

# Coverage comparison
print("\n4. South Pole Coverage by Architecture")
print("-" * 60)

coverage = [
    ("Direct to Earth", "0-50%", "Only when rim is visible"),
    ("1 ELFO satellite", "~60%", "Half of 12-hour orbit"),
    ("2 ELFO satellites", "~90%", "Overlapping coverage"),
    ("4 ELFO satellites", "~99%", "Full redundancy"),

    ("Gateway only", "~20-30%", "NRHO visibility limited"),
]

print(f"  {'Architecture':<20} {'Coverage':<10} {'Notes':<30}")
print("-" * 60)

for arch, cov, notes in coverage:

# Comparison with Earth GEO

print("-" * 60)

comparisons = [
    ("", "Earth Mobile", "Lunar EVA"),
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Assuming direct Earth visibility.** South pole crater floors often have zero Earth line-of-sight.

2. **Treating Gateway as primary relay.** NRHO visibility to south pole is only 20-30%. Gateway is backup, not primary.

3. **Underestimating EVA constraints.** Astronaut terminals are 2 W omni—can't close high-rate links.

4. **Ignoring lunar terrain masking.** Mountains and crater walls block signals even when relay is overhead.

5. **Single-point-of-failure designs.** Crew safety requires redundant communication paths.

6. **Forgetting thermal environment.** Lunar day/night cycles affect antenna pointing and electronics.

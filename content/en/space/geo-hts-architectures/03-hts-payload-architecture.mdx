## Why this matters

An HTS satellite is more than spot beams and frequency reuse—it's a complete system routing hundreds of gigabits between gateways and user beams. The payload architecture determines flexibility, capacity, and resilience. Understanding bent-pipe vs regenerative, beam hopping, and gateway diversity separates knowing HTS marketing from engineering reality.

## How it works

### Bent-pipe vs regenerative

```
Payload Processing Architectures:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  BENT-PIPE (Transparent):                               │
│  ┌─────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │   Input ──▶ LNA ──▶ Frequency ──▶ HPA ──▶ Output│    │
│  │            │       Converter       │            │    │
│  │            ▼       (down/up)       ▼            │    │
│  │         Amplify              Amplify            │    │
│  │                                                  │    │
│  │   • No signal demodulation                      │    │
│  │   • Frequency translation only                  │    │
│  │   • Simple, proven, reliable                    │    │
│  │   • Noise accumulates end-to-end               │    │
│  │   • Limited flexibility                         │    │
│  │                                                  │    │
│  │   Example: ViaSat-1, early HTS                  │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
│  REGENERATIVE (OBP - On-Board Processing):              │
│  ┌─────────────────────────────────────────────────┐    │
│  │                                                  │    │
│  │   Input ──▶ LNA ──▶ Demod ──▶ Switch ──▶ Mod ──▶│    │
│  │            │         │         │         │      │    │
│  │            ▼         ▼         ▼         ▼      │    │
│  │         Amplify   Decode    Route    Encode     │    │
│  │                    bits     bits      bits      │    │
│  │                                                  │    │
│  │   ──▶ HPA ──▶ Output                            │    │
│  │        │                                        │    │
│  │        ▼                                        │    │
│  │     Amplify                                     │    │
│  │                                                  │    │
│  │   • Full signal regeneration                   │    │
│  │   • Bit-level switching                        │    │
│  │   • Noise doesn't accumulate                   │    │
│  │   • Complex, expensive, power-hungry           │    │
│  │   • Maximum flexibility                        │    │
│  │                                                  │    │
│  │   Example: ViaSat-3, SES-17                    │    │
│  │                                                  │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

## Apply

**HTS payload sizing:**

```python
import numpy as np

def hts_capacity_estimate(num_user_beams, bw_per_beam_mhz, spectral_eff):
    """
    Estimate HTS total capacity
    
    spectral_eff: typical 2-4 bits/s/Hz
    """
    capacity_gbps = num_user_beams * bw_per_beam_mhz * spectral_eff / 1000
    return capacity_gbps

def gateway_count(total_capacity_gbps, capacity_per_gw_gbps, redundancy_ratio=1.3):
    """
    Estimate number of gateways needed
    
    redundancy_ratio: N+M where ratio = (N+M)/N
    """
    active_gws = np.ceil(total_capacity_gbps / capacity_per_gw_gbps)
    total_gws = np.ceil(active_gws * redundancy_ratio)
    return int(active_gws), int(total_gws)

def beam_hopping_gain(peak_to_avg_ratio):
    """
    Capacity gain from beam hopping
    
    peak_to_avg_ratio: traffic demand variability (typically 2-5)
    """
    # Beam hopping allows sharing power based on demand
    # Equivalent capacity gain vs fixed allocation
    return np.sqrt(peak_to_avg_ratio)

# Example: ViaSat-3 class HTS
print("HTS Payload Design Example:")
print("=" * 50)

num_beams = 300
bw_per_beam = 250  # MHz (after frequency reuse)
spectral_eff = 3.5  # bits/s/Hz

capacity = hts_capacity_estimate(num_beams, bw_per_beam, spectral_eff)
print(f"User beams: {num_beams}")
print(f"Bandwidth per beam: {bw_per_beam} MHz")
print(f"Spectral efficiency: {spectral_eff} bits/s/Hz")
print(f"Total capacity: {capacity:.0f} Gbps")

# Gateway planning
gw_capacity = 10  # Gbps per gateway
active, total = gateway_count(capacity, gw_capacity)
print(f"\nGateway infrastructure:")
print(f"  Capacity per gateway: {gw_capacity} Gbps")
print(f"  Active gateways: {active}")
print(f"  Total with redundancy: {total}")

# Beam hopping benefit
peak_avg = 3.0
gain = beam_hopping_gain(peak_avg)
print(f"\nBeam hopping:")
print(f"  Peak-to-average traffic ratio: {peak_avg}")
print(f"  Effective capacity gain: {gain:.1f}×")
print(f"  Equivalent fixed capacity: {capacity * gain:.0f} Gbps")
```

## Pitfalls

1. **Marketing capacity.** Advertised Gbps assumes best conditions; practical throughput lower.

2. **Gateway bottleneck.** Each gateway needs fiber backhaul—availability and cost vary by location.

3. **Beam hopping timing.** Terminals must synchronize to hopping schedule; adds complexity.

4. **OBP power consumption.** Regenerative payloads consume significantly more DC power.

5. **Technology risk.** OBP technology fixes at launch; bent-pipe adapts to ground upgrades.

6. **Fill factor.** Capacity utilization rarely exceeds 30-50% in practice.

7. **Interference coordination.** Multiple HTS in same slot creates complex frequency sharing.

## Why this matters

High-throughput satellites (HTS) need bandwidth—lots of it. Ku, Ka, Q, V, and W bands (12-110 GHz) offer massive spectrum allocations that lower bands simply can't provide. Starlink, OneWeb, Viasat, and SES's O3b mPOWER all operate primarily in Ka-band. Understanding these higher frequencies—their capacity advantages and rain fade challenges—is essential for designing modern satellite broadband systems.

## Apply

**High-frequency band comparison and rain fade calculator:**

```python
import numpy as np

C = 299792458  # Speed of light (m/s)

def wavelength_cm(frequency_ghz):
    """Calculate wavelength in centimeters."""
    return (C / (frequency_ghz * 1e9)) * 100

def fspl_db(frequency_ghz, distance_km):
    """Free space path loss in dB."""
    return 20 * np.log10(distance_km) + 20 * np.log10(frequency_ghz) + 92.45

def rain_attenuation_itu_r838(frequency_ghz, rain_rate_mm_hr, 
                               elevation_deg=30, polarization='H'):
    """
    ITU-R P.838 rain attenuation model (simplified).
    Returns specific attenuation in dB/km.
    """
    f = frequency_ghz
    
    # Simplified ITU-R P.838 coefficients (horizontal polarization)
    if f < 2.9:
        k = 0.0000387 * f ** 0.912
        alpha = 0.912
    elif f < 54:
        k = 0.0000998 * f ** 1.276
        alpha = 1.021 + 0.0048 * f
    else:
        k = 0.0002 * f ** 1.175
        alpha = 1.1
    
    # Specific attenuation (dB/km)
    gamma_r = k * (rain_rate_mm_hr ** alpha)
    
    return gamma_r

def effective_path_length_km(rain_rate_mm_hr, elevation_deg, altitude_km=5):
    """
    ITU-R P.618 effective path length through rain (simplified).
    """
    # Rain height (simplified model)
    h_r = 4.0  # km, typical mid-latitude rain height
    
    # Slant path length
    L_s = (h_r - altitude_km * 0) / np.sin(np.radians(elevation_deg))
    L_s = max(L_s, 0)
    
    # Horizontal reduction factor (simplified)
    L_g = L_s * np.cos(np.radians(elevation_deg))
    r = 1 / (1 + 0.045 * L_g)
    
    return L_s * r

def total_rain_fade_db(frequency_ghz, rain_rate_mm_hr, elevation_deg=30):
    """
    Total rain fade for slant path.
    """
    gamma = rain_attenuation_itu_r838(frequency_ghz, rain_rate_mm_hr)
    L_eff = effective_path_length_km(rain_rate_mm_hr, elevation_deg)
    return gamma * L_eff

# Band definitions
hf_bands = [
    ("Ku-band", 14.0, 12.0, 18.0),
    ("Ka-band DL", 20.0, 17.7, 20.2),
    ("Ka-band UL", 28.5, 27.5, 30.0),
    ("Q-band", 42.0, 33.0, 50.0),
    ("V-band", 55.0, 40.0, 75.0),
    ("W-band", 85.0, 75.0, 110.0),
]

print("High-Frequency Satellite Band Analysis")
print("=" * 75)

# Basic properties
print("\n1. Band Properties Comparison")
print("-" * 75)
print(f"{'Band':<12} {'Center':>8} {'λ':>10} {'FSPL @36Mm':>12} {'FSPL @550km':>12}")
print(f"{'':<12} {'(GHz)':>8} {'(cm)':>10} {'GEO (dB)':>12} {'LEO (dB)':>12}")

for name, center, low, high in hf_bands:

    fspl_geo = fspl_db(center, 36000)

    print(f"{name:<12} {center:>8.1f} {wl:>10.3f} {fspl_geo:>12.1f} {fspl_leo:>12.1f}")

# Rain fade comparison
```

<section class="lesson-pitfalls">

## Pitfalls

1. **Ignoring rain fade.** Ka-band can lose 20+ dB in heavy rain; design for 99.5% availability requires significant margin.

2. **Assuming fixed throughput.** ACM means throughput varies with weather; capacity planning must account for rain statistics.

3. **Gateway bottleneck.** Q/V feeder links need site diversity or the gateway becomes single point of failure.

4. **Elevation angle.** Low elevation = longer slant path through rain = more fade. Tropical locations doubly impacted.

5. **Atmospheric absorption.** 60 GHz oxygen absorption (V-band) creates "windows" and "walls" in spectrum.

6. **Terminal size assumptions.** Higher frequency needs higher antenna gain; "same size dish" doesn't mean same performance.

7. **Interference.** Ka-band increasingly congested; coordination with other operators essential.

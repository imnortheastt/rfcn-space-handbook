<section class="lesson-hook">

## Tai sao dieu nay quan trong

AMF xu ly di dong. Nhung khi dien thoai cua ban can **gui du lieu** — luot web, xem video, hoac upload file — do la lanh tho cua **SMF**.

SMF quan ly **PDU session**: cac duong ong du lieu giua thiet bi va internet. No quyet dinh du lieu di dau, xu ly nhu the nao (QoS), va dieu khien UPF thuc su di chuyen goi tin.

</section>

## Truc giac

### SMF lam gi

```
Vai tro SMF:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│                    ┌─────────┐                          │
│                    │   PCF   │ (policy)                 │
│                    └────▲────┘                          │
│                         │ Npcf                          │
│                         │                                │
│  ┌─────────┐      ┌─────┴─────┐      ┌─────────┐       │
│  │   AMF   │◄────►│    SMF    │◄────►│   UDM   │       │
│  └─────────┘ Nsmf └─────┬─────┘ Nudm └─────────┘       │
│                         │                                │
│                         │ N4 (PFCP)                      │
│                         │                                │
│                    ┌────▼────┐                          │
│                    │   UPF   │──────► Internet          │
│                    └─────────┘                          │
│                                                          │
│  SMF: Dieu khien session (signaling)                    │
│  UPF: Chuyen tiep goi tin (data plane)                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**Diem quan trong:** SMF chi la control plane. No bao UPF lam gi nhung khong bao gio cham vao du lieu nguoi dung.

### Trach nhiem chinh

| Trach nhiem | Y nghia |
|-------------|---------|
| **Thiet lap PDU session** | Tao duong du lieu cho UE |
| **Chon va dieu khien UPF** | Chon UPF, cai dat quy tac chuyen tiep |
| **Cap phat dia chi IP** | Gan IPv4/IPv6 cho UE |
| **Quan ly QoS** | Cau hinh QoS flow va quy tac |
| **Thuc thi policy** | Ap dung quyet dinh PCF |
| **Tich hop tinh cuoc** | Giao tiep voi CHF de tinh cuoc |
| **Xu ly che do SSC** | Hanh vi duy tri session |

### Tu 4G sang 5G quan ly session

```
4G (Ket hop):
┌─────────────────────────────────────────────────────────┐
│  MME: Signaling session (GTP-C den SGW/PGW)             │
│  SGW: Neo user plane                                     │
│  PGW: Cap phat IP, policy, tinh cuoc                    │
└─────────────────────────────────────────────────────────┘

5G (Tach biet):
┌─────────────────────────────────────────────────────────┐
│  AMF: Chi di dong (khong quan ly session)               │
│  SMF: Signaling session, QoS, cap phat IP              │
│  UPF: Chuyen tiep goi tin (tach khoi control)          │
└─────────────────────────────────────────────────────────┘
```

**Loi ich:** Scale SMF cho dung luong session doc lap voi dung luong di dong AMF.

<section class="lesson-apply">

## Ap dung

**Tham so cau hinh SMF:**

| Tham so | Muc dich | Vi du |
|---------|----------|-------|
| Danh sach DNN | Mang du lieu ho tro | "internet", "ims", "enterprise" |
| Pool IP | Dai IPv4/IPv6 | 10.0.0.0/8, fd00::/48 |
| S-NSSAI | Slice ho tro | SST=1, SST=2/SD=001 |
| Danh sach UPF | UPF da biet | Instance UPF voi dia chi N4 |
| 5QI mac dinh | QoS mac dinh | 9 (best effort) |
| DNS server | Cho UE | DNS chinh, phu |

**Chi so dung luong:**

| Chi so | Mo ta |
|--------|-------|
| PDU session hoat dong | Session dong thoi |
| Ty le thiet lap session | Thiet lap/giay |
| Giao dich N4 | Message PFCP/giay |
| Su dung pool IP | IP da dung vs kha dung |

**Xu ly su co:**

| Van de | Kiem tra |
|--------|----------|
| Session that bai | DNN cau hinh, UPF truy cap duoc |
| Khong cap IP | Can pool, timeout DHCP |
| QoS khong ap dung | Policy PCF, quy tac cai trong UPF |
| UPF khong truy cap duoc | N4 heartbeat, PFCP association |
| Thiet lap cham | Do tre N4, NRF discovery |

<section class="lesson-pitfalls">

## Cam bay

1. **SMF xu ly user traffic (sai).** SMF chi la control plane. Neu ban debug van de data path, nhin vao UPF.

2. **Mot SMF moi DNN.** Nhieu SMF co the phuc vu cung DNN. Lap ke hoach phan phoi tai.

3. **Kich co pool IP.** Danh gia thap kich co pool IP va session se that bai. Giam sat su dung.

4. **Bo qua N4 heartbeat.** Mat PFCP association nghia la mat dieu khien UPF. Tinh chinh timer heartbeat.

5. **QoS khong co PCF.** SMF co the gan QoS, nhung PCF cung cap policy. Cau hinh tinh chi hoat dong cho truong hop don gian.

6. **Khong khop che do SSC.** Nhu cau ung dung va che do SSC phai phu hop. TCP qua SSC mode 2 ngat ket noi.

7. **Phuc tap lua chon UPF.** Trien khai edge can logic lua chon UPF can than. Test kich ban di dong.

8. **Thieu DNS trong session.** UE can DNS server. Cau hinh trong SMF hoac policy PCF.

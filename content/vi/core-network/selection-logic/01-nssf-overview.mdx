## Tai sao dieu nay quan trong

Network slicing la tinh nang dot pha cua 5G cho nha mang. Cac slice khac nhau cho dich vu khac nhau — eMBB, URLLC, mMTC — moi slice co QoS rieng.

NSSF quyet dinh slice nao UE su dung. Lua chon slice sai, va nguoi dung roi vao mang sai voi QoS sai.

</section>

```
Single Network Slice Selection Assistance Information:

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  S-NSSAI = SST + SD (tuy chon)                          │
│                                                          │
│  ┌───────────────────────────────────┐                  │
│  │ SST (Slice/Service Type)          │                  │
│  │ 8 bits (0-255)                    │                  │
│  │                                    │                  │

│  │ 1 = eMBB (enhanced Mobile BB)     │                  │
│  │ 2 = URLLC (Ultra-Reliable Low Lat)│                  │
│  │ 3 = mMTC (massive Machine Type)   │                  │
│  │ 4 = V2X (Vehicle-to-Everything)   │                  │
│  │ 5-127 = Danh rieng                │                  │
│  │ 128-255 = Rieng nha mang          │                  │
│  └───────────────────────────────────┘                  │
│                                                          │
│  ┌───────────────────────────────────┐                  │
│  │ SD (Slice Differentiator)         │                  │
│  │ 24 bits (tuy chon)                │                  │
│  │                                    │                  │
│  │ Dung cho:                         │                  │
│  │ - Slice rieng doanh nghiep        │                  │
│  │ - Phan biet dich vu               │                  │
│  │ - Bien the theo vung              │                  │
│  └───────────────────────────────────┘                  │
│                                                          │
│  Vi du S-NSSAIs:                                        │
│  ┌─────────────────────────────────────────────────┐    │
│  │ SST=1, SD=khong    → eMBB mac dinh              │    │
│  │ SST=1, SD=0xABC123 → eMBB doanh nghiep          │    │
│  │ SST=2, SD=khong    → URLLC mac dinh             │    │
│  │ SST=128, SD=0x001  → Slice gaming nha mang      │    │
│  └─────────────────────────────────────────────────┘    │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

<section class="lesson-apply">

## Ap dung

**Cac endpoint dich vu NSSF:**

| Endpoint | Method | Muc dich |
|----------|--------|----------|
| `/network-slice-info` | POST | Lua chon slice |
| `/nssai-availability/{nfId}` | PUT | Dang ky kha nang AMF |
| `/nssai-availability/{nfId}` | DELETE | Huy dang ky AMF |
| `/nssai-availability/{nfId}` | PATCH | Cap nhat kha nang AMF |

**Go loi lua chon slice:**

```bash
# Kiem tra log NSSF cho quyet dinh lua chon
kubectl logs -l app=nssf | grep NSSelection

# Xac minh dang ky NSSAI availability cua AMF
curl -X GET http://nssf:8080/nnssf-nssaiavailability/v1/nssai-availability

# Kiem tra lua chon slice
curl -X POST http://nssf:8080/nnssf-nsselection/v1/network-slice-info \
  -H "Content-Type: application/json" \
  -d '{
    "nfType": "AMF",
    "nfId": "test-amf",
    "sliceInfoRequestForRegistration": {
      "requestedNssai": [{"sst": 1}],
      "tai": {"plmnId": {"mcc": "001", "mnc": "01"}, "tac": "0001"}
    }
  }'
```

**Cau hinh S-NSSAI pho bien:**

| Truong hop | SST | SD | Mo ta |
|------------|-----|-----|-------|
| Broadband tieu dung | 1 | - | eMBB mac dinh |
| Rieng doanh nghiep | 1 | 0xABC123 | eMBB rieng |
| IoT cong nghiep | 2 | 0x000001 | URLLC cho nha may |
| Dong ho thong minh | 3 | - | mMTC cho dien luc |
| Gaming | 128 | 0x00GAME | Gaming do tre thap |

<section class="lesson-pitfalls">

## Cam bay

1. **AMF khong dang ky NSSAI availability.** NSSF khong the dinh tuyen neu khong biet AMF nao phuc vu slice nao.

2. **Thieu SD trong S-NSSAI.** SST=1 va SST=1+SD=abc la cac slice khac nhau. Quen SD gay lua chon slice sai.

3. **Bo qua slice rieng TAI.** Slice co the chi kha dung tai mot so tracking area. NSSF phai kiem tra TAI.

4. **Anh xa NSSAI roaming.** PLMN khach co the dung gia tri S-NSSAI khac. NSSF phai anh xa dung.

5. **Subscribed NSSAI cu.** Cap nhat goi cuoc UDM khong truyen den NSSF. Kiem tra tich hop NSSF-UDM.

6. **Khong co slice du phong.** Neu tat ca slice yeu cau khong kha dung, co nen co mac dinh? Cau hinh hanh vi du phong.

7. **Dinh tuyen lai vong.** AMF-1 dinh tuyen den AMF-2, AMF-2 dinh tuyen nguoc lai AMF-1. Kiem tra cau hinh target AMF.

8. **NSSF khong duoc truy van.** AMF co the bo qua NSSF neu nghi co the xu ly cuc bo. Xac minh cau hinh AMF.

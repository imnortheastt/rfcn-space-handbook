## Tại sao điều này quan trọng

Switch kết nối thiết bị cùng mạng. Nhưng làm sao packet đi từ điện thoại của bạn đến server ở bên kia thế giới?

Routing. Router đọc địa chỉ IP đích, tra cứu bảng định tuyến, và forward packet về phía đích — từng hop một, qua các mạng.

Mọi mạng viễn thông đều phụ thuộc vào routing. Hiểu nó là bắt buộc.

</section>

<section class="lesson-intuition">

## Trực giác

### Router làm gì

Router kết nối các mạng khác nhau. Nó nhận packet trên một interface và forward ra interface khác, dựa trên địa chỉ IP đích.

```
┌─────────────────────────────────────────────────────────┐
│                        Router                            │
│                                                          │
│  eth0 (10.0.0.0/24)              eth1 (192.168.1.0/24)  │
│         │                                  │             │
└─────────┼──────────────────────────────────┼─────────────┘
          │                                  │
          ▼                                  ▼
    Network A                           Network B
    (10.0.0.x)                         (192.168.1.x)
```

**Insight quan trọng:** Router hoạt động ở Layer 3. Nó đọc địa chỉ IP, không phải MAC. Mỗi hop qua router thay đổi Layer 2 header nhưng giữ nguyên Layer 3 header.

### Bảng định tuyến

Mỗi router duy trì **bảng định tuyến** — danh sách đích và cách đến đó.

```
Destination        Next Hop         Interface    Metric
10.0.0.0/24        Connected        eth0         0
192.168.1.0/24     Connected        eth1         0
172.16.0.0/16      10.0.0.254       eth0         10
0.0.0.0/0          192.168.1.1      eth1         100
```

**Các trường chính:**
- **Destination:** Network prefix (địa chỉ IP + subnet mask)
- **Next hop:** Gửi packet cho đích này đến đâu
- **Interface:** Interface vật lý/logic nào dùng
- **Metric:** Chi phí route này (thấp hơn tốt hơn)

### Longest prefix match

Nếu packet match nhiều route thì sao?

```
Bảng định tuyến:
10.0.0.0/8       → Next hop A
10.0.0.0/16      → Next hop B
10.0.0.0/24      → Next hop C

Packet destination: 10.0.0.5
```

**Trả lời:** Router dùng **longest prefix match**. Route cụ thể nhất (mask dài nhất) thắng.

Trong trường hợp này: `/24` cụ thể hơn `/16` cụ thể hơn `/8`. Packet đi đến Next hop C.

**Tại sao quan trọng:** Đây là cách default route hoạt động. `0.0.0.0/0` match mọi thứ nhưng thua bất kỳ route cụ thể hơn.

<section class="lesson-mechanism">

## Cách hoạt động

### Quy trình forward packet

```
┌─────────────────────────────────────────┐
│       Packet đến trên interface          │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│    Giảm TTL (Time to Live)               │
│    Nếu TTL = 0, drop và gửi ICMP         │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│    Trích xuất địa chỉ IP đích            │
└───────────────────┬─────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────┐
│    Tra cứu trong bảng định tuyến         │
│    (longest prefix match)                │
└───────────────────┬─────────────────────┘
                    │
           ┌────────┴────────┐
           │                 │
           ▼                 ▼
    ┌──────────┐      ┌──────────────┐
    │  Match   │      │ Không match  │
    └────┬─────┘      └──────┬───────┘
         │                   │
         ▼                   ▼
┌─────────────────┐  ┌─────────────────┐
│ Forward qua     │  │ Drop packet,    │
│ next hop/iface  │  │ gửi ICMP        │
└─────────────────┘  │ unreachable     │
                     └─────────────────┘
```

### TTL và routing loop

**TTL (Time to Live)** ngăn packet chạy vòng mãi nếu routing bị misconfigure.

1. Sender đặt TTL ban đầu (thường 64 hoặc 128)
2. Mỗi router giảm TTL đi 1
3. Nếu TTL về 0, router drop packet và gửi ICMP Time Exceeded

**Traceroute** khai thác điều này: nó gửi packet với TTL=1, TTL=2, TTL=3... và thu thập ICMP reply để map đường đi.

### Connected route

Khi bạn configure địa chỉ IP trên interface, **connected route** tự động được thêm.

```
Interface eth0: 10.0.0.1/24

Bảng định tuyến nhận:
10.0.0.0/24    Connected    eth0
```

Nghĩa là: "Tôi có thể reach 10.0.0.0/24 trực tiếp qua eth0, không cần next hop."

### Static route

Route được configure thủ công, không đổi trừ khi bạn đổi.

```
ip route add 172.16.0.0/16 via 10.0.0.254
```

**Use case:**
- Mạng đơn giản với topology ổn định
- Configure default gateway
- Backup route (floating static route với metric cao hơn)

### Dynamic route

Route học từ routing protocol (OSPF, BGP, IS-IS). Router tự động học và cập nhật route dựa trên trạng thái mạng.

Chi tiết hơn trong bài tiếp theo.

### Route selection

Khi nhiều source cung cấp route đến cùng đích, router dùng **administrative distance** để chọn.

| Source | Administrative Distance |
|--------|------------------------|
| Connected | 0 |
| Static | 1 |
| OSPF | 110 |
| IS-IS | 115 |
| BGP (external) | 20 |
| BGP (internal) | 200 |

Thấp hơn tốt hơn. Connected route luôn thắng learned route.

### ARP và next hop resolution

Khi router biết next hop IP, nó cần MAC address của next hop để build Layer 2 header.

**ARP (Address Resolution Protocol):**
1. Router kiểm tra ARP cache cho next hop MAC
2. Nếu không có, gửi ARP request: "Ai có 10.0.0.254?"
3. Next hop reply: "10.0.0.254 là AA:BB:CC:DD:EE:FF"
4. Router cache mapping và forward packet

```
Packet gốc:
┌────────────────────────────────────────────────────┐
│ L2: Src=RouterMAC, Dst=NextHopMAC                  │
│ L3: Src=OriginalSrcIP, Dst=FinalDstIP              │
│ Payload                                            │
└────────────────────────────────────────────────────┘
```

**Insight quan trọng:** L2 address thay đổi ở mỗi hop. L3 address giữ nguyên end-to-end.

<section class="lesson-apply">

## Áp dụng

**Đọc bảng định tuyến:**

```
$ ip route show
default via 192.168.1.1 dev eth0 proto static metric 100
10.0.0.0/8 via 10.255.255.1 dev tun0 proto static metric 50
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.100
```

- `default` = `0.0.0.0/0` — match mọi thứ
- `proto kernel` = connected route
- `proto static` = configure thủ công
- `metric` = chi phí (thấp hơn thắng cho cùng đích)

**Troubleshooting routing:**

1. **Packet không đến đích:**
   - Kiểm tra bảng định tuyến: `ip route show`
   - Route cho đích có tồn tại không?
   - Next hop có reachable không?

2. **Asymmetric routing:**
   - Đường đi khác đường về
   - Phổ biến với nhiều uplink
   - Có thể break stateful firewall

3. **Black hole:**
   - Route tồn tại nhưng next hop unreachable
   - Packet drop im lặng
   - Kiểm tra next hop connectivity

4. **Routing loop:**
   - Packet bounce giữa router
   - TTL cuối cùng expire
   - Kiểm tra conflicting route

**Command hữu ích:**

```bash
# Xem bảng định tuyến
ip route show

# Xem route đến đích cụ thể
ip route get 8.8.8.8

# Trace đường đến đích
traceroute 8.8.8.8

# Xem ARP cache
ip neigh show

# Thêm static route
ip route add 10.0.0.0/8 via 192.168.1.254
```

<section class="lesson-pitfalls">

## Cạm bẫy

1. **Nhầm switching với routing.** Switching = Layer 2, MAC address, cùng mạng. Routing = Layer 3, IP address, mạng khác nhau.

2. **Quên return path.** Packet cần route về. A reach được B không có nghĩa B reach được A.

3. **Bỏ qua metric.** Hai route đến cùng đích? Route với metric thấp hơn thắng. Metric misconfigure gây suboptimal path.

4. **Không kiểm tra next hop reachability.** Route đến 10.0.0.0/8 via 192.168.1.254 vô dụng nếu bạn không reach được 192.168.1.254.

5. **Giả định route symmetric.** Internet routing thường asymmetric. Packet của bạn có thể đi đường khác reply.

6. **Quên ARP.** Không có ARP entry = không forward, dù route tồn tại. ARP timeout có thể gây outage ngắn.

7. **TTL exhaustion trong production.** Nếu TTL quá thấp và path quá dài, packet không bao giờ đến. TTL chuẩn 64 xử lý hầu hết trường hợp.
